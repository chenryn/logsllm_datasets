### Feature Request

**Current Behavior:**
- Currently, to initialize a class, one must use the `constructor` method.

**Expected/Desired Behavior:**
- It would be beneficial to introduce an alternative, such as `$class`, for initializing classes.

**Reproduction of the Problem:**
- Please refer to this Plunker for a demonstration: [http://plnkr.co/edit/JK1FbSomU15KgKU64Tr1?p=preview](http://plnkr.co/edit/JK1FbSomU15KgKU64Tr1?p=preview)

**Expected Behavior:**
- The current requirement is to use the `constructor` method for initialization. 

**Motivation / Use Case for Changing the Behavior:**
- The primary motivation for this change is to provide easier access to static data and potentially simplify class initialization.

**Environment Details:**
- **Angular Version:** 2.0.0-rc.4
- **Browser:** Chrome
- **Language:** TypeScript

**Additional Information:**
- For further reference, see issue #4707.
- [Plunker Link](http://plnkr.co/edit/JK1FbSomU15KgKU64Tr1?p=preview)
