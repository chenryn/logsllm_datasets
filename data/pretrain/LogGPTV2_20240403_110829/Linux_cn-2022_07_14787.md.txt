### 数年后 Ubuntu Unity 桌面起死回生

Ubuntu 的 Unity 桌面已经很久没有更新了。最后一次出现在 Ubuntu 17.04 中的是 Unity 7.5 版本。随后，Canonical 公司放弃了对 Unity 的开发支持。然而，一些喜欢它的开发者们创建了 UnityX 项目，使得今年我们见到了 Unity 7.6，它被搭载在非官方的 Ubuntu Unity 合成版中。当然，你也可以在普通的 Ubuntu 上安装它。这是另一个完全不像 Windows 的少数选项。

**[消息来源：The Register](https://www.theregister.com/2022/07/01/ubuntu_unity_desktop_updated_after/)**

**老王点评**：Unity 本身是否优秀或流行并不重要，关键在于开源世界是自由的，你随时都有再造一个轮子的自由。这一点非常重要。

![](/data/attachment/album/202207/02/182130ikxi5ese3ibix5p4.jpg)

### 软件自由保护协会抨击 GitHub

软件自由保护协会（SFC）专注于保护自由和开源软件，宣布已停止使用微软的 GitHub 进行项目托管，并敦促其他软件开发者也这样做。他们在其网站上发布了一份声明，呼吁自由和开源软件开发者们自愿切换到不同的代码托管服务。SFC 与 GitHub 的决裂最终是由 GitHub Copilot 导致的。他们认为，GitHub 决定发布一个基于 FOSS 代码的营利性产品，这是“无法容忍的”。Copilot 的训练数据来自 GitHub 上的公共存储库中的代码，但并未说明这些代码所附带的软件许可证和版权持有人信息。

**[消息来源：The Register](https://www.theregister.com/2022/06/30/software_freedom_conservancy_quits_github/)**

**老王点评**：确实，Copilot 有意无意地忽视了用来训练的代码的许可证问题。因此，你可能会在得到的建议中采用各种许可证的代码，即使只是几行。这从根本上打击了开源许可证的严肃性。

![](/data/attachment/album/202207/02/182144d2zyyanlmhw7lpwp.jpg)

### 无视谷歌、Mozilla 反对，万维网联盟发布了 DID 规范

去中心化标识符（DID）规范描述了一种部署全球唯一标识符的方法，而无需依赖一个中心化的机构作为验证实体。它们旨在使个人和组织能够使用自己信任的系统生成自己的标识符，这些新的标识符使实体能够通过使用数字签名等加密证明来证明对其的控制。DID 的格式类似 `did:方法:特定标识`。然而，谷歌和 Mozilla 提出了反对意见，指出 DID 中的“方法”未定义，因此无法评估 DID 将如何运作，也无法确定如何处理互操作性。他们还担心依靠工作量证明机制的区块链来处理 DID 的道德问题。

**[消息来源：The Register](https://www.theregister.com/2022/07/01/w3c_overrules_objections/)**

**老王点评**：虽然这一举措看起来很去中心化，但我怀疑由中心化组织提出的去中心化规范是否真的有用？
