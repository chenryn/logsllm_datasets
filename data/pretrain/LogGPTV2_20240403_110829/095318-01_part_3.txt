### 3.4.4 删除目录
可以使用 `rmdir` 命令来删除空目录。例如：
```bash
$rmdir dirname
```
**注意：** 在执行删除操作时，请确保目标目录为空，即不包含任何文件或子目录。如果尝试删除非空目录，将导致错误。

同时删除多个空目录也是可行的，只需在命令后列出所有需要删除的目录名即可：
```bash
$rmdir dirname1 dirname2 dirname3
```
如果这些目录确实为空，它们会被成功移除，并且命令行不会显示额外信息。

### 3.4.5 改变所在目录
通过 `cd` (change directory) 命令，用户可以在终端中切换当前工作目录到任意有权限访问的位置。其基本语法为：
```bash
$cd [directory]
```
这里的 `[directory]` 可以是相对路径也可以是绝对路径。例如：
```bash
$cd /usr/local/bin
```
这会将当前目录更改为 `/usr/local/bin`。如果从该位置想要移动到另一个相对位置如 `/usr/home/amrood`，则可以这样操作：
```bash
$cd ../../home/amrood
```

### 3.4.6 重命名目录
使用 `mv` (move) 命令不仅可以移动文件和目录，还能用来重命名它们。重命名一个目录的基本格式如下：
```bash
$mv olddir newdir
```
示例说明如何将名为 `mydir` 的目录改名为 `yourdir`：
```bash
$mv mydir yourdir
```

### 3.5 Linux 文件权限与访问模式
为了提高数据安全性，Linux 系统为每个文件设置了三种主要类型的权限：
- **所有者权限**：控制文件拥有者对该文件的操作能力。
- **组权限**：定义了与文件相关联的用户组成员可进行的操作。
- **其他用户权限**：指定了不属于上述两类用户的人员对文件的访问级别。

#### 3.5.1 查看文件权限
利用 `ls -l` 命令可以查看指定文件或目录的详细权限设置。例如：
```bash
$ls -l /home/amrood
-rwxr-xr-- 1 amrood users 1024 Nov 2 00:10 myfile
drwxr-xr--- 1 amrood users 1024 Nov 2 00:10 mydir
```
输出中的第一列描述了权限配置：
- 第一个字符表示类型（`-` 表示常规文件；`d` 表示目录）。
- 接下来的九个字符分为三组，每组代表一类用户的权限（读r、写w、执行x）。

#### 3.5.2 文件与目录访问模式
对于普通文件而言，权限包括读取(r)、写入(w)及执行(x)。而对于目录来说，虽然也支持相同的权限标签，但含义略有不同：
- **读取**：允许查看目录内容列表。
- **写入**：允许在目录内创建或删除条目。
- **执行**：赋予进入并遍历目录的能力。

#### 3.5.3 更改权限
权限可以通过 `chmod` (change mode) 命令调整。此命令支持两种表达方式：符号法和八进制数字法。

**符号法**：简单直观，适用于初学者。比如给 `testfile` 添加外部用户的写权限并移除所有者的执行权限：
```bash
$chmod o+w u-x testfile
```

**数字法**：基于二进制逻辑，更加精确。每个权限对应一个数值（无=0, 执行=1, 写=2, 读=4）。组合起来形成完整的权限集。例如设置 `testfile` 的权限为所有者全权、同组只读、其他人仅可执行：
```bash
$chmod 741 testfile
```

#### 3.5.4 更改所有者与所属组
文件的所有权信息可通过 `chown` 和 `chgrp` 命令更改。前者用于修改文件拥有者身份，后者则是变更文件关联的主要用户组。

- 修改文件所有者：
  ```bash
  $chown newowner filename
  ```
  
- 修改文件所属组：
  ```bash
  $chgrp newgroup filename
  ```

#### 3.6 Linux 环境变量
环境变量是在操作系统层面存储的数据片段，可用于保存各种配置信息。它们通常由系统启动脚本、用户登录shell以及应用程序动态设置。定义一个新的环境变量及其值的方法如下所示：
```bash
$export TEST="Linux Programming"
```
然后可以通过简单的echo语句来打印这个变量的内容：
```bash
$echo $TEST
Linux Programming
```
每次打开新的shell会话时，都会重新加载默认的环境变量配置文件如`/etc/profile`或用户特定的`.profile`等。
