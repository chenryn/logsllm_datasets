### 问题描述
我想每周将生产数据库的数据转移到一个分析服务器上进行分析。请问你们通常采用什么方法来转移生产数据以供分析使用？我已经阅读了一些相关资料，但对其中涉及的开销和设置过程还不太清楚。

### 可选方案
1. **复制（Replication）**：
   - 生产数据库同时作为发布者和分发者。
   - 使用推送机制，每周更新我的分析服务器上的数据库。

2. **SQL Server Integration Services (SSIS)**：
   - 尽管我没有经验，但据我了解这是标准做法之一。

3. **手动方法**：
   - 使用Windows任务计划程序定期压缩备份文件，并通过FTP连接将其传输到分析服务器。

4. **镜像（Mirroring）**：
   - 我没有经验，但它主要用于高可用性场景。

### 环境信息
- 数据库：SQL Server 2008
- 操作系统：Windows Server 2008 R2
- 数据库大小：约36GB
- 仅需要数据部分

### 其他需求
- 数据库可以是只读的。
- 每周运行一次该过程。

### 补充信息
- 我不介意数据库是只读的。
- 我希望每周执行一次这个过程。

### 其他建议
- **日志传送（Log Shipping）**：
  - 日志传送与您的“手动”方法类似，但它利用了SQL内置组件来完成工作，并且适用于事务日志备份。
  - 关于日志传送与数据库镜像的详细比较，请参阅以下链接中的文章：[SQL Server 2008 Log Shipping](http://blogs.technet.com/josebda/archive/2009/04/02/sql-server-2008-log-shipping.aspx)

如果您有其他推荐的方法或建议，欢迎分享！
