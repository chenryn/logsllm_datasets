### Windows API 监控

**创建时间**: 2022-03-30T14:26:51.873Z  
**描述**: 监控与各种代码注入类型相关的 Windows API 调用可能会生成大量数据，并且除非在特定情况下针对已知的恶意调用序列进行收集，否则可能不会直接用于防御，因为良性使用 API 函数可能很常见且难以与恶意行为区分开来。Windows API 调用如 CreateRemoteThread、SuspendThread/SetThreadContext/ResumeThread 以及那些可以用于修改另一个进程内存的调用（如 VirtualAllocEx/WriteProcessMemory）可用于此技术。[1]  
**引用来源**: [1] Hosseini, A. (2017年7月18日). 十种进程注入技术：常见和趋势进程注入技术的技术调查。检索于2017年12月7日。[链接](https://www.endgame.com/blog/technical-blog/ten-process-injection-techniques-technical-survey-common-and-trending-process)
