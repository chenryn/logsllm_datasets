### 定理1的证明

在步骤P2中，客户端定义了r个多项式f1, ..., fr，每个多项式的次数为t。定义s个多项式Fc = fjWjc，其中1 ≤ c ≤ s，且Wjc是数据库中的单词，如步骤S2所定义。注意这些多项式Fc也是次数最多为t的多项式，并且Fc(0) = ∑1≤j≤r fj(0)Wjc = ∑1≤j≤r ββWjc = Wβc。

假设服务器i是响应的服务器之一。在步骤S1中，它从客户端接收[f1(αi), ..., fr(αi)]。在步骤S3中，它计算Ric = ∑1≤j≤r fj(αi)Wjc，对于1 ≤ c ≤ s，并在步骤S4中将这些值返回给客户端。

客户端在步骤C1中初始化H为{(G, ǫ)}，其中G是响应的服务器编号集合。我们声称，在经过x次C4循环迭代后，H将是{(G, B(xw)β)}的集合，其中B(xw)β是由数据库块Bβ的前xw位组成的字符串。我们通过归纳法进行证明：已经证明了x=0的情况成立。假设x=c-1时成立，对于某个1 ≤ c ≤ s。现在考虑第c次迭代。

客户端调用EASYRECOVER(S, w, t, h, H, [R1c, ..., Rℓc], [α1, ..., αℓ])，其中Rjc = Fc(αj)，对于每个j ∈ G。

### 定理2的证明

与上述类似，我们将证明在C4循环的x次迭代后，H将是一个包含元素(G(x)h, B(xw)β)的集合，其中G(x)h是所有回复并且回复前x个单词诚实的服务器编号集合，B(xw)β如前所定义。对于x=0，这是显然成立的。假设对于x=c-1成立，对于某个1 ≤ c ≤ s。现在考虑第c次迭代。我们可以假设客户端在步骤C6中调用了HARDRECOVER，正如我们之前所指出的，当EASYRECOVER产生答案时，它产生的答案与HARDRECOVER相同。

HARDRECOVER将在步骤H2中生成一组多项式，其中一个必然是Fc（如上所述）。这个集合可能包含多项式数量（在k中），但我们将在第3.4节中看到，该集合包含除所需之外的其他元素的概率可以任意小，确保整个协议在（概率）多项式时间内运行。当H3循环遇到元素Fc时，Wi将被设置为Fc(0) = Wβc（如上所述）。根据归纳假设，步骤H5将找到元素(G(c-1)h, B((c-1)w)β) ∈ H。但也有Γ(Fc) ⊇ Gh，因此Γ(Fc) ∩ G(c-1)h ⊇ Gh，所以|Γ(Fc) ∩ G(c-1)h| ≥ h。然后步骤H5将添加(Γ(Fc) ∩ G(c-1)h, B((c-1)w)β || Wβc) = (G(c)h, B(cw)β)到H′，证明完成。

### 事实3的证明

我们从以下引理开始：

**引理：** 存在一个仅依赖于S是否为环或域的多项式PS(k)，使得在步骤H2中输出的候选多项式{φi}的集合大小最多为PS(k)。

**引理的证明：** 对于域S，[7]中的算法通过构造一个具有如下性质的双变量多项式Q(x, y)来工作：对于任何满足(y - φ(x))是Q的因式的单变量多项式φ，多项式Q在y上的次数为t(h-1 + h(kt)^2 + 4(h^2 - kt))^2 / (2(h^2 - kt))。通过一些简单的代数运算，可以很容易看出，如果1 ≤ t < k且√kt < h ≤ k，该值被限制在2k^2以内。

对于环S = Zpq，我们分别对模p和q进行操作，并使用中国剩余定理组合结果，匹配因子使用Γ(φ)。这可能会将可能结果的数量上限增加到(2k^2)^2。

注意，这两个界限都不是紧的。

每次调用HARDRECOVER最多会在步骤H2中生成PS(k)个多项式。每个这样的多项式φ都有|Γ(φ)| ≥ h。但由于h（诚实服务器的数量）大于回复的总服务器数量的一半，因此Γ(φ)中的某些元素必须是诚实服务器的编号。对于每个φ，至少有一个这样的元素，而对于除了正确多项式以外的每个φ，最多只能有t个（否则，φ将在超过t个地方与正确多项式一致，因此它确实是正确的多项式）。关键观察类似于[16]中的观察，即如果拜占庭服务器不知道诚实服务器编号j的αj值，则他们只有很小的机会生成与正确多项式在某个诚实的αj处一致的错误多项式。

设Z为{αj : 服务器j不诚实}的集合，Y为{αj : 服务器j诚实}的集合。我们希望限定在步骤H2中返回的≤ sPS(k)个多项式中，以及在每个这些多项式与正确多项式一致的≤ t个I\Z点中，最终的≤ stPS(k)个点与集合Y有非平凡交集的概率。但|I\Z| = |I| - (ℓ - h)，且|Y| = h，因此该概率最多为hstPS(k) / (|I| - ℓ + h)，如要求所示。
