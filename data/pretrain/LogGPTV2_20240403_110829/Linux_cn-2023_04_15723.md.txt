> 年龄并不是为开源做贡献的障碍。

我叫 Neil Naveen，今年 14 岁，是一名初中生，已有七年的编程经验。我使用 Golang 编程已有两年时间。

然而，编程并不是我的唯一爱好。我还练习了四年的巴西柔术，并参加过多次比赛。我对编程和柔术充满热情，因为它们教会了我许多重要的生活课程。

### CodeCombat

我在 [CodeCombat](https://codecombat.com) 上开始了我的编程之旅，这个平台教会了我许多基础编程技巧。在我的编程历程中，最激动人心的时刻之一是在 CodeCombat 主办的多人竞技场中，我在大约 50,000 名玩家中排名第 16 位。当时我只有 11 岁，这对我来说是一个不可思议的成就，也给了我继续探索和学习新事物的信心。

### LeetCode

在 CodeCombat 之后，我转向了 [LeetCode](https://leetcode.com/neilnaveen)。通过解决这个网站上精心设计的问题，我进一步磨练了我的算法编程技能。

### Coding Game

当我 13 岁时，我开始参与 [Coding Game](https://www.codingame.com/profile/0fa733a2c7f92a829e4190625b5b9a485718854) 的机器人编程。这里的竞争更加激烈，因此我必须采用更高效的算法。例如，在创建终极井字游戏人工智能时，我使用了极小化极大算法（Minimax）和蒙特卡洛树搜索（Monte Carlo Tree Search），使我的代码快速高效。

### GitHub CLI

有一天，我看到父亲在使用一个名为 [GitHub CLI](https://github.com/cli/cli) 的开源工具，我被它深深吸引。GitHub CLI 是一个允许用户直接从命令行与 GitHub API 交互的工具，而无需访问 GitHub 网站。

后来，我注意到父亲正在审查一个旨在检测依赖项漏洞的机器人的拉取请求（PR）。于是，我思考了 GitHub CLI 和这个机器人，并想知道 GitHub CLI 是否本身被安全机器人监控。事实证明它并没有。

于是我创建了一个修复程序，并包含了对 GitHub CLI 的安全审计。令我高兴的是，我的贡献被接受了，并被合并到了项目中。能够为像 GitHub CLI 这样受欢迎的工具做出贡献，并帮助保护它，对我来说是一个非常宝贵的机会。这是我的 PR 链接：[GitHub CLI 安全审计 PR](<链接>)。

### 提交你的代码

我希望我的故事能激励其他年轻人去探索并为开源世界做出贡献。年龄并不是障碍。每个人都应该勇于探索和贡献。如果你想看看我的网站，请访问 [neilnaveen.dev](https://neilnaveen.dev)。你也可以查看我的 [LeetCode 个人资料](https://leetcode.com/neilnaveen/)。如果你感兴趣，可以观看我在 [CloudNativeSecurityCon](https://www.youtube.com/watch?v=K6NRUGol-rE) 的演讲记录。

我很感激迄今为止所获得的机会，并对未来充满期待。谢谢阅读我的故事！
