The comma and right parenthesis emitted from JSX-transforming a line like that
in the title will result in the following:
    // source JSX
    var ExampleCommaIssue = React.createClass({
        render: function () {
            return ;
        }
    });
    // transforms to 
    ... React.createElement("div", {style: ({'margin':'42px'}, )className: "test"}); ...
    // instead of
    ... React.createElement("div", {style: ({'margin':'42px'}), className: "test"}); ...
Notably, this only seems to be a problem if some attribute follows the one
with the paren-hugged object literal.
Also, I did not test to see if arbitrary values in parentheses also result in
this.
