### Appendix

#### A. Symbol Synchronization (λ)

For a very large number of samples \(N_r\), when no authentication signal is embedded, \(\lambda = \sigma^2_s \cdot e^{j\phi f_c}\), where \(f_c\) is the constant frequency offset, and \(\phi = \frac{2\pi N_f}{F_s}\) [33]. When FEAT is utilized to embed the authentication signal, we define \(M\) sets of frames with the frequency offsets \(f_c + f_k\), where \(f_c = f_t + f_r\), and \(f_k = f_m\) for \(m = 1, 2, \ldots, M\). Assuming that the authentication symbols are statistically independent and identically distributed, \(\lambda\) is given by:

\[
\lambda = \frac{\sigma^2_s}{M} \sum_{m=1}^{M} e^{j\phi (f_c + f_m)} = \frac{\sigma^2_s}{M} e^{j\phi f_c} \sum_{m=1}^{M} e^{j\phi f_m}
\]

For \(M = 2\), \(\lambda\) is given by:

\[
\lambda = \sigma^2_s \cdot e^{j\phi f_c} \cdot \cos(\phi f_a)
\]

where \(\phi = \frac{2\pi N_f}{F_s}\). The maximum value of \(\lambda\) is achieved when \(f_a = 0\), indicating the highest synchronization accuracy without an embedded authentication signal. Conversely, when \(f_a = \frac{F_s}{4\pi N_f}\), the absolute value of \(\lambda\) is 0, making synchronization difficult.

To achieve sufficient robustness for synchronization and authentication, \(f_a\) should be sufficiently larger than 0 but smaller than \(\frac{F_s}{4\pi N_f}\).

#### B. Frame Synchronization (ψ)

For a very large number of samples \(N_r\), when no authentication signal is embedded, each frame has the same frequency offset, and the relative frequency offset between consecutive frames is zero, so \(\psi = \sigma^2_s\). When FEAT is utilized, we define \(M^2\) sets of relative frequency offsets between two consecutive frames—\(M\) sets of frequency offset 0, \(M-1\) sets of frequency offset \(\frac{2f_a}{M-1}\), and so on. Hence, \(\psi\) is given by:

\[
\psi = \frac{\sigma^2_s}{M^2} \sum_{m=1}^{M-1} \sum_{l=0}^{N_o-1} e^{j2\pi \frac{2m f_a l}{(M-1)F_s}}
\]

Approximating, we get:

\[
\psi \approx \frac{\sigma^2_s}{M^2} \left( M^2 + \sum_{m=1}^{M-1} (M - m) \sin\left(\frac{4\pi m f_a}{(M-1)F_s}\right) \right)
\]

#### C. Theoretical Limits on EFO (\(f_a\))

In the presence of a very large number of samples for synchronization, the performance of the proposed algorithm depends mainly on \(f_a\). From equation (5), the absolute value of \(\lambda\) decreases with increasing \(f_a\), reducing the probability of detecting the peak of \(\Lambda\). Similarly, from equation (6), the absolute value of \(\psi\) decreases with increasing \(f_a\), reducing the probability of detecting the peak of \(\Psi\).

To achieve sufficient robustness, \(f_a\) should be within theoretical bounds. For \(M = 2\), \(\lambda\) is given by:

\[
\lambda = \sigma^2_s \cdot e^{j\phi f_c} \cdot \cos(\phi f_a)
\]

The maximum value of \(\lambda\) is achieved when \(f_a = 0\), and it becomes 0 when \(f_a = \frac{F_s}{4\pi N_f}\). Therefore, \(f_a\) needs to be sufficiently larger than 0 but smaller than \(\frac{F_s}{4\pi N_f}\).

Additionally, existing standards like IEEE 802.11g [2] allow a limited margin for the carrier frequency offset (CFO) due to inaccurate oscillators. For instance, the absolute value of CFO should be less than 25 ppm of the carrier frequency. Thus, we need to ensure that \(f_t + f_a \leq F_o\), where \(F_o\) is the allowed frequency offset.
