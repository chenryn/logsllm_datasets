### C++ 之父呼吁改变编程语言以提升安全性

尽管许多现代编程语言已支持类型安全，确保变量仅访问其数据类型定义的内存部分，但 C++ 在这方面一直面临挑战。今年 1 月中旬，官方的 C++ “方向小组” 发表了一份声明，旨在解决 C++ 的安全性问题。这份由 C++ 创始人 Bjarne Stroustrup 共同撰写的声明，呼吁对 C++ 语言本身进行修改，以应对安全问题。“安全方面的改进不仅需要在工具层面，还需要在语言、编译器和库层面实现。” 他们建议将多个功能打包成配置文件，这不仅能提高安全性，还能提供灵活性。这些配置文件可以专门用于支持嵌入式计算、性能敏感的应用程序，或特定领域如汽车、航空航天、航空电子、核能或医疗应用。

**[消息来源：The Newstack](https://thenewstack.io/can-c-be-saved-bjarne-stroustrup-on-ensuring-memory-safety/)**

### 全新 Windows 11 向第三方发送用户数据

网络安全专家分析了全新笔记本上刚安装的 Windows 11 产生的 DNS 流量，发现该系统不仅联系了 Windows Update、MSN 和必应服务器，还联系了 Steam、McAfee 以及各种市场研究公司和广告服务。微软对此回应称，这种数据流动在现代操作系统中很常见，用于“帮助用户保持安全、更新，并确保系统按预期运行”。

**[消息来源：Tom's Hardware](https://www.tomshardware.com/news/windows-11-sends-user-data-to-third-party-services)**

> **老王点评：虽然在意料之中，但被这样抓住现行还是有点吃惊。**

### 开源语音助手 Mycroft 资金耗尽

Mycroft 是一个基于 Linux 设备（包括树莓派）的开源语音助手。然而，在 Kickstarter 众筹后，该项目已经没有资金继续履行众筹活动的奖励，甚至无法维持基本运营。他们遇到了合作伙伴无法提供稳定硬件的问题，转而使用现成零件，随后又遭遇供应链中断。此外，他们还经历了多次塑料注塑模具、定制印刷电路板和设备认证的过程，这些费用远远超出了众筹捐款总额。更糟糕的是，他们还面临着非执业专利实体的持续诉讼。

**[消息来源：Kickstarter](https://www.kickstarter.com/projects/aiforeveryone/mycroft-mark-ii-the-open-voice-assistant/posts/3729060)**

> **老王点评：做事不易，对困难的重视也不足。如果项目夭折，确实非常可惜。**
