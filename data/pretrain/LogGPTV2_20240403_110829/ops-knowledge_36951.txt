**常用的性能监控工具及其使用方法**

1. **top**

   top 是一个在 Linux 系统中广泛使用的实时性能监控工具，它能够显示系统的整体运行状况，包括 CPU 使用率、内存占用情况以及各个进程的状态等。通过在终端输入 `top` 命令即可启动该程序；若想查看多核处理器上每个核心的具体负载，则可以在 top 运行时按下数字键 1。

2. **vmstat**

   vmstat 是一款强大的命令行工具，专门用来检查虚拟内存状态、CPU 活动、磁盘 I/O 以及其他相关指标。此命令对于快速诊断系统瓶颈非常有用。
   
   - **基本语法**:
     ```bash
     $ vmstat [选项]
     ```
     
   - **常用选项**:
     - `-a`: 显示所有信息（等同于同时启用 `-s`, `-m`, 和 `-S all`）。
     - `-f`: 报告自启动以来发生的 fork, vfork, 和 clone 调用次数。
     - `-m`: 展示有关内存使用及分配的额外详情。
     - `-n`: 包含文件 I/O、块设备 I/O、中断和上下文切换频率的数据。
     - `-p <pid>`: 针对特定进程 ID 输出统计数据。
     - `-s`: 列出各种系统统计量。
     - `-d`: 提供磁盘活动概况。
     - `-t`: 添加时间戳至输出结果中。
     - `-V`: 打印版本号并退出。

3. **sar (System Activity Reporter)**

   sar 是一套全面的系统性能测量与报告工具集，支持长期收集关于 CPU 效能、内存消耗、网络流量及磁盘 I/O 的历史记录，并以易于理解的方式呈现给用户。
   
   - **基础格式**:
     ```bash
     $ sar [选项] [采样间隔] [样本数量]
     ```
     
   - **主要参数**:
     - `-A`: 生成一份包含所有可用数据类型的综合报告。
     - `-b`: 关注缓冲区访问模式。
     - `-c`: 记录 CPU 利用率。
     - `-d`: 分析磁盘传输速率。
     - `-g`: 汇总页面交换频率。
     - `-I <irq#>`: 监控指定中断请求线路上的活动。
     - `-q`: 展现平均队列长度和系统负荷。
     - `-r`: 描述物理 RAM 及交换空间的利用情况。
     - `-u`: 聚焦于 CPU 占用比例。
     - `-v`: 涵盖进程创建/销毁率、内存页错误计数等多项指标。

以上三种工具各有侧重但又互为补充，合理运用它们可以帮助运维人员更好地理解和优化其管理下的服务器环境。
