### 问题描述
我和朋友共享一个 Windows Server 2012 R2 Datacenter 虚拟服务器上的账户。我们使用这个服务器来托管一个应用程序，每天最多只需要使用几次，因此共用一个账户对我们来说是可行的。我们通过远程桌面协议 (RDP) 进行连接。

问题是，当我的朋友已经登录时，如果我尝试登录，系统会强制他注销。我记得在以前使用旧版本的 Windows Server（由其他人配置）时，系统总是会询问我是否要继续登录并断开当前已登录用户的连接，或者停止操作。

请问，在我们的服务器上是否可以设置为在断开当前已登录用户之前进行提示？

### 解答
在 Windows Server 2012 R2 中，如果多个用户尝试使用同一个账户同时登录，系统默认会强制断开第一个用户的连接，而不会进行提示。这是因为 RDP 会话在同一时间只能有一个活动实例。

如果你希望在断开当前已登录用户之前进行提示，你需要使用不同的用户账户。这样，当第二个用户尝试登录时，系统会提示是否断开当前已登录用户的连接。

### 解决方案
1. **创建新用户账户**：在服务器上创建一个新的用户账户，并确保该账户具有所需的权限。
2. **配置 RDP 设置**：
   - 打开“远程桌面服务管理器”。
   - 导航到“RD 会话主机配置”。
   - 右键点击“RDP-Tcp”，选择“属性”。
   - 在“会话”选项卡中，找到“限制每个用户只能使用一个会话”选项，并取消勾选它。

3. **测试配置**：使用不同的用户账户分别登录，检查系统是否会在断开当前已登录用户之前进行提示。
