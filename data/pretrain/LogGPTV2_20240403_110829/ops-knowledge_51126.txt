### 问题描述
我正在尝试配置我的 Windows Server 2008，以便开发人员可以在不登录服务器或不具备管理员权限的情况下查看其状态。然而，Windows 2008 对于远程非管理员用户的权限设置并不包括枚举或查询服务的功能。这导致任何试图与远程服务控制管理器 (SCM) 进行通信的工具（如 Win32_Service、sc.exe 和 services.msc）都会失败。

### 问题
如何设置权限，使得这些用户至少可以列出服务并查看它们是否正在运行？

### 解决方案
经过一番搜索后，我发现了一篇 MSDN 文章，该文章提供了一个解决方案，尽管其中的安全描述符是针对 Windows Server 2003 的，但似乎在 Windows Server 2008 上也能正常工作。根据文章中的指导，您可以使用 `sc sdshow SCMANAGER` 命令来获取当前的服务控制管理器的安全描述符。这样，在对比现有设置与所需设置时，您可以确保不会意外锁定某些必要的功能。

### 操作步骤
1. **获取当前安全描述符**：
   - 打开命令提示符（以管理员身份运行）。
   - 输入以下命令并按回车键：
     ```shell
     sc sdshow SCMANAGER
     ```
   - 记录下输出的安全描述符。

2. **修改安全描述符**：
   - 根据 MSDN 文章中的说明，修改安全描述符以添加适当的权限。
   - 使用 `sc sdset SCMANAGER` 命令将新的安全描述符应用到服务控制管理器。例如：
     ```shell
     sc sdset SCMANAGER "D:(A;;CCLCSWRPWPDTLOCRRC;;;SY)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;BA)(A;;CCLCSWLOCRRC;;;AU)(A;;CCLCSWRPWPDTLOCRRC;;;PU)"
     ```

3. **验证更改**：
   - 非管理员用户现在应该能够通过 `sc query` 或 `services.msc` 等工具查看和查询服务状态。
   - 确保没有意外地移除其他必要的权限。

### 注意事项
- 在进行任何更改之前，请备份当前的安全描述符，以防需要恢复。
- 如果您不确定某个权限的具体含义，请查阅相关文档或咨询有经验的管理员。
- 测试更改的效果，确保所有预期的功能都能正常工作。
