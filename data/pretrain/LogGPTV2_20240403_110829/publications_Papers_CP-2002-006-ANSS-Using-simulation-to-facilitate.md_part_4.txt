**工作流设计存储**

METEOR 设计器（Cardoso, 2002）将工作流设计保存为 XML 文档，并存储在 XML 模型库（Arpinar et al., 2001）中。这样的设计可以从库中检索并转换为 JSIM 模拟规范。由于两个系统都将其模型表示为带有 AND 和 XOR 节点的有向图，因此这种转换较为简单。基于节点类型，从工作流节点到模拟节点的默认映射被自动执行。使用 JSIM 模型设计工具可以在实际运行模拟之前调整此默认设置。

**任务计算示例**

1. 任务周期时间 = 等待时间 + 劳动时间 = 1.0 小时 + 1.5 小时 = 2.5 小时
2. 任务成本 = 固定成本 + 劳动费率 * 劳动时间 = $0.6 + $20 * 1.5 小时 = $30.6
3. 任务质量 = 95%

假设在实例执行期间，质量度量反复低于设定阈值，因为技术人员感染了大肠杆菌，导致样品受到污染。这种情况显著降低了 DNA 的纯度，使任务的质量因子低于 95%。作为响应，WfMS 检测到样本质量不足，并向监控人员发送警告消息。

**案例研究：基因组测序工作流**

本节通过一个正在进行中的基因组测序工作流来说明工作流适应的过程。该工作流由 LSDIS 实验室与遗传学系教授 Jonathan Arnold 博士共同开发。基因组测序工作流由多个顶层任务组成（详见 Cardoso, 2002），主要任务包括准备样本、制备克隆体、测序和序列处理。我们的案例研究重点关注测序子工作流。此任务分为两个阶段：第一阶段用于培养单个细胞；第二阶段使用 DNA 测序机读取每个单个 DNA 片段的生物化学字母 (A, G, C 或 T)。工作流如图2所示。
