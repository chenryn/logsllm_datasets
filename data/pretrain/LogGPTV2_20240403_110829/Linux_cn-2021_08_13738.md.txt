### Torvalds 认为 GPLv2 和公司的参与对 Linux 非常重要

![](/data/attachment/album/202108/31/181104qcqpp776apjpeiae.jpg)

Linux 创始人 Linus Torvalds 在最近的一次[采访](https://www.zdnet.com/article/linus-torvalds-on-linuxs-30th-birthday/)中表示，他认为在 1992 年初将 Linux 转向使用 GPLv2 许可证是一个至关重要的决定。他回忆说：“这不是最初的许可证，但我相信这是 Linux 变得如此广泛的一个重要原因。”此外，他还强调了公司参与的重要性，尽管这听起来可能有些陈词滥调，但在开源社区的一些角落里，商业参与常常被视为负面因素。自 Linux 项目早期以来，一直有大公司对其表现出持续的兴趣。

> 尽管 Linux 的成功有一定的时代背景因素，但至少可以说明 Linux 在某些方面做得非常正确。

### 谷歌使用 AI 生成高保真图像

![](/data/attachment/album/202108/31/181123ppq3p1id7vc3qczu.jpg)

谷歌大脑的研究人员[分享](https://petapixel.com/2021/08/30/googles-new-ai-photo-upscaling-tech-is-jaw-dropping/)了他们在图像超级分辨率领域取得的新突破。通过 SR3 和 CDM 方法，他们能够从低分辨率图像生成高分辨率图像，在提升人像和自然图像质量方面效果显著。这些模型的性能已经达到了超分辨率和类条件 ImageNet 生成基准的最先进水平。

> 未来，我们或许可以从低画质照片和视频中生成高画质内容。

### 研究发现 40% 的 GitHub Copilot 建议中存在安全漏洞

![](/data/attachment/album/202108/31/181140ydadzid2cbv767dl.jpg)

研究人员为 GitHub Copilot 创建了 89 种不同的场景来推荐代码，共生成了 1600 多个程序。在审查这些程序时[发现](https://www.techradar.com/news/github-autopilot-highly-likely-to-introduce-bugs-and-vulnerabilities-report-claims)，几乎 40% 的程序都存在某种形式的安全漏洞。由于 Copilot 是基于 GitHub 存储库中的公开代码进行训练的，研究人员推测，生成的脆弱代码可能是系统模仿存储库中错误代码行为的结果。

> 这就像差生抄作业一样，它也不知道所抄的作业是否正确。
