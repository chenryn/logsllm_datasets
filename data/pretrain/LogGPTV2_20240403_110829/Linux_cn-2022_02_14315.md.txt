### 大部分 Rust 程序员认为 Rust 很有挑战性，但值得

根据 [Rust 社区年度调查](https://blog.rust-lang.org/2022/02/15/Rust-Survey-2021.html) 的结果，81% 的 Rust 用户每周都会使用这种语言，而 75% 的用户将其用于生产环境。在工作中使用 Rust 的人中，83% 认为其具有挑战性，但只有 13% 的受访者认为该语言拖累了团队。相反，82% 的人表示 Rust 有助于实现团队目标。仅有 1% 的受访者认为这些挑战不值得，而 79% 的受访者认为绝对值得。此外，90% 的受访者表示未来有可能再次使用 Rust，其中 96% 的人选择它是因为它可以“构建相对正确且无错误的软件”，而 92% 的人则因其性能而选择了它。

**老王点评**：尽管 Rust 难度较大，但它显然已准备好在生产环境中取代 C 语言。

![](/data/attachment/album/202202/28/164321po1dd8zifvyorvdw.jpg)

### FSF 公布五篇探讨 GitHub Copilot 的论文

自由软件基金会 (FSF) 将微软的 GitHub Copilot 称为“服务即软件替代物（SaaSS）”，并认为其对自由软件运动和保障用户及开发者自由的能力提出了严重问题。因此，他们向社区征文讨论这一新生事物。现在，[FSF 公布了其中五篇论文](https://www.fsf.org/news/publication-of-the-fsf-funded-white-papers-on-questions-around-copilot)，虽然不一定同意或反对这些观点，但认为它们推动了对该问题的讨论。

顺便提一下，Copilot 的意思不是“复制许多”，而是指飞机上的“副驾驶”或“自动驾驶仪”，尽管它确实是由 AI 从代码库中“复制”出许多代码。

**老王点评**：这种技术生成的代码版权和知识产权界定仍需进一步讨论，但我认为这是一种有益的进步。

![](/data/attachment/album/202202/28/164341lc1szc7rhccqyjtm.jpg)

### 中国手机品牌几乎统治欧洲最大的通信展

下周，世界移动通信大会 (MWC) 将在巴塞罗那举办。从目前的情况来看，展会中的主要硬件发布似乎都来自中国品牌，如 OPPO、荣耀、TCL、小米的子品牌 Poco 和华为，而非前几年常见的欧洲、美国甚至韩国的品牌。截至去年，IDC 报告显示，全球最受欢迎的五大智能手机品牌中有三个（小米、Oppo 和 vivo）总部位于中国，尽管第一和第二的位置仍由三星和苹果占据。

**老王点评**：尽管苹果和谷歌不太重视这个展会，但这反映了中国手机制造业的繁荣。
