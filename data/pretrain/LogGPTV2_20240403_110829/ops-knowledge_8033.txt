### 问题描述
我保存了一个包含显示信息的事件日志文件，现在想在另一台计算机上打开这个文件并查看完整的事件描述。据我所知，这些信息应该保存在 `LocalMetaData.mta` 文件中，但我不知道如何使用这个文件。请问如何“导入” `.mta` 文件以便在另一台计算机上获取完整的事件日志描述？

### 解答

#### 方法一：使用 XML 格式保存和导入事件日志
1. **保存事件日志**：
   - 如果你的系统是 Windows Server 2008 或更高版本，你可以右键点击“应用程序”日志，选择“另存为...”选项。
   - 在保存对话框中，选择 XML 作为输出格式，并勾选“导出显示信息”选项。
   - 点击“保存”以保存文件。

2. **导入事件日志**：
   - 将保存的 XML 文件复制到另一台计算机上。
   - 打开事件查看器（Event Viewer），选择“操作”菜单中的“打开保存的日志”，然后浏览并选择你保存的 XML 文件。
   - 这样，你就可以在另一台计算机上查看带有完整事件描述的日志了。

#### 方法二：处理 Windows Server 2003 的限制
- 在 Windows Server 2003 上，你只能将事件日志保存为文本或 CSV 格式，无法直接保存为 XML 格式。
- 如果你需要在 Windows Server 2003 上保存和传输事件日志，可以考虑以下方法：
  1. **通过远程连接访问日志**：
     - 使用 Windows Server 2008 或更高版本的事件查看器，通过远程连接访问 Windows Server 2003 的事件日志。
     - 在 Windows Server 2008 的事件查看器中，添加一个远程计算机的连接，然后从该远程计算机导出事件日志为 XML 格式。
  2. **手动转换**：
     - 如果上述方法不可行，你可以尝试手动将文本或 CSV 格式的日志转换为 XML 格式。这可能需要编写一些脚本或使用第三方工具来完成。

#### 关于 `LocalMetaData.mta` 文件
- `LocalMetaData.mta` 文件通常用于存储本地元数据，但它并不直接用于导入或导出事件日志。
- 如果你已经保存了带有显示信息的 XML 文件，那么你不需要单独处理 `LocalMetaData.mta` 文件。只需按照上述方法导入 XML 文件即可。

希望这些步骤能帮助你在另一台计算机上成功查看完整的事件日志描述。如果你还有其他问题，请随时提问。
