After taking a look at the source code, I see that :
Using cudacodec VideoReader needs the HAVE_NVCUVID envvar to be set :  
https://github.com/opencv/opencv/blob/3.4.1/modules/cudacodec/src/video_reader.cpp#L49-L54
This envvar is set only if CUDA_nvcuvid_LIBRARY is set :  
https://github.com/opencv/opencv/blob/3.4.1/cmake/OpenCVDetectCUDA.cmake#L36-L38
And this variable is set only when we are on WIN32 :  
https://github.com/opencv/opencv/blob/3.4.1/cmake/FindCUDA.cmake#L792-L794
So I think that there is no way to enable NVCUVID. Do I miss something ? If
not, I think that it should be fixed.
Furthermore, I don't want to use the CuvidVideoSource, but I want to use the
shared FFMPEG lib that already can use cuvid, with the FFmpegVideoSource. And
I think that there is no need for OpenCV to link to cuvid to use this ffmpeg
backend. So the FFMPEG backend should be available event if HAVE_NVCUVID is
not defined. Is it ?
I can do these changes, but I think that having a confirmation before working
on could be interresting.
