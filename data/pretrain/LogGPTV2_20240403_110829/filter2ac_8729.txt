**ATT&CK ID:** M1025  
**ATT&CK Technique Name:** Privileged Process Integrity

In Windows 8.1, Windows Server 2012 R2, and later versions, the Local Security Authority (LSA) can be configured to run as a Protected Process Light (PPL) by setting the registry key `HKLM\SYSTEM\CurrentControlSet\Control\Lsa\RunAsPPL`. This configuration ensures that all Security Support Provider (SSP) DLLs must be signed by Microsoft. [1] [6]
