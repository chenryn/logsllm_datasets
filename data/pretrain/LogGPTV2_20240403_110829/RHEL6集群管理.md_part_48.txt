    ```
    ::: note
    ::: admonition_header
    **注意**
    :::
    ::: admonition
    ::: para
    如果在卷组中有多个逻辑卷，那么 `lv_name`{.literal}
    资源中的逻辑卷名称（`lv_name`{.literal}）为空白或未指定。另外请注意在
    HA-LVM 配置中，一个卷组只能被单一服务使用。
    :::
    :::
    :::
4.  ::: para
    编辑 `/etc/lvm/lvm.conf`{.filename} 文件的 `volume_list`{.literal}
    字段，使其包括您的 root 卷组名称，同时使用 @ 覆盖在
    `/etc/cluster/cluster.conf`{.filename}
    文件中列出的主机名。这里的主机名是您在其中编辑 `lvm.conf`{.filename}
    文件的机器名称。注：这个字符串[*必须*]{.emphasis}与
    `cluster.conf`{.filename} 文件中给出的节点名称映射。下面是
    `/etc/lvm/lvm.conf`{.filename} 文件中的条目示例：
    :::
    ``` screen
    volume_list = [ "VolGroup00", "@neo-01" ]
    ```
    ::: para
    这个标签将用来激活共享卷组或逻辑卷。[*不要*]{.emphasis}包括任何要使用
    HA-LVM 共享的卷组名称。
    :::
5.  ::: para
    在所有集群节点中更新 `initrd`{.literal} 设备：
    :::
    ``` screen
    # dracut -H -f /boot/initramfs-$(uname -r).img $(uname -r)
    ```
6.  ::: para
    重启所有节点以保证使用正确的 `initrd`{.literal} 设备。
    :::
:::
