## Description
The closure compiler style JSDoc allows the generics like `Array`, not only
`Array.`.  
But the above environment shows the argument styled with its style as `any` in
the intellisense.
### problem case
    /**
     * @params {Array} v
     * @return {string}
     */
    function bar(v) { // 
      return v.join();
    }
    var b = bar(["1", "2"]); // Intellisense shows this signature as `bar(v: any)`;
### non-problem case
It's ok `Array.` or `T[]`which is allowed in http://usejsdoc.org/tags-
type.html
    /**
     * @params {Array.} v
     * @return {string}
     */
    function foo(v) {
      return v.join();
    }
    var b = foo(["1", "2"]); // Intellisense shows this signature as `foo(v: any)`;
### What I expect
I want Salsa interprets the closure compiler style's generics annotation.  
By any chance, I should report this to
https://github.com/Microsoft/TypeScript?
