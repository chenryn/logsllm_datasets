User: How to configure firewall rules on CentOS server to allow specific ports and IP addresses?
Assistant: Configuring firewall rules on a CentOS server can be achieved using the iptables command. The following are the specific steps: 1. Confirm that the iptables service has been installed and started: ``` systemctl status iptables.service ``` 2. List the current firewall rules: ``` iptables -L ``` 3. Add to allow specific IP addresses Rules for accessing specific ports, for example, to allow an IP address of 192.168.1.100 to access SSH port 22: ``` iptables -A INPUT -p tcp -s 192.168.1.100 --dport 22 -j ACCEPT ``` 4. Add to allow specific IP Rules for address access to a specific port range, for example to allow the IP address 192.168.1.100 to access the Redis port range 6379-6380: ``` iptables -A INPUT -p tcp -s 192.168.1.100 --dport 6379:6380 -j ACCEPT `` ` 5. Add a rule to allow a specific network segment to access a specific port, for example, to allow the 10.0.0.0/8 network segment to access HTTP port 80: ``` iptables -A INPUT -p tcp -s 10.0.0.0/8 --dport 80 - j ACCEPT ``` 6. Save the firewall rules: ``` service iptables save ``` 7. Restart the iptables service: ``` systemctl restart iptables.service ``` The above steps are to configure the firewall rules on the CentOS server to allow Specific ports and IP addresses.
