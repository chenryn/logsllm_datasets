Currently the .gitignore generated when you type `flutter create -t module
foo` inside `foo` contains the line ".android". My testing suggests excluding
this from a git checkout causes problems for other people trying to check out
your project.
You can test this by:
  1. Start with some existing android project and follow the instructions here: https://github.com/flutter/flutter/wiki/Add-Flutter-to-existing-apps
  2. Create a repository with both the android app and the flutter module. Commit and push.
  3. Somewhere else, check out that repository and try to load in android studio.
