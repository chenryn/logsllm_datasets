![](/data/attachment/album/202106/08/174726rhrc8fvi5fbvltz2.jpg)

在 LWN.net 发布的 Linux 内核报告中，常年可以看到一个贡献者 —— “Yue Hai Bing”。自 2018 年底以来，TA 对每个版本都会贡献大量的补丁，帮助 Linux 内核不断迭代。为了找到这个来自华为的优秀贡献者，我开始联系朋友圈中的华为同事，了解相关信息。每次提到 “Yue Haibing”，大家总是神秘一笑，不再多言。

这让我更加好奇：到底是谁，能如此频繁地在 Linux 内核中贡献代码，却从未在各种会议和聚会上见到 TA？直到 openEuler 的问世，我们与 openEuler 社区的合作逐渐增多，终于得知，原来 Yue HaiBing 背后竟然是华为的自动内核缺陷发现机器人 HULK Robot！正是 HULK Robot 完成了如此高频且优质的内核代码贡献。没想到，我以为的大神背后竟然是一个机器人！

为了了解更多关于 HULK Robot 的信息，我采访了 HULK Robot 的架构师魏勇军先生，并与他聊了聊 HULK Robot 的故事。

![](/data/attachment/album/202106/08/135544nmich0a061c600if.jpg)
*魏勇军（左）接受老王（右）的专访*

### HULK Robot 的由来

提及 HULK Robot 的诞生，魏勇军表示，其实 HULK Robot 很早便已在社区应用。早在 2017 年，HULK Robot 的前身就已经进入 Linux 内核社区并发挥作用。当时，HULK Robot 以 “Wei Yongjun” 的身份提交代码修复补丁。其提交代码的频率之高、质量之优，让 Linux 内核的核心维护者 Greg Kroah-Hartman 都为之惊叹。在 2017 年北京举办的首次 LC3 大会上，Greg Kroah-Hartman 还特别提到了 “Wei Yongjun”。

社区的积极反馈让魏勇军意识到，这个机器人对 Linux 内核社区是有价值的，可以帮助 Linux 内核变得更好。于是，HULK Robot 正式立项。尽管说起来容易做起来难，但魏勇军找到了思路。华为多年来虽然在做自己的服务器操作系统，但始终坚持“上游优先”，积累了大量的 Linux 内核维护经验。HULK Robot 可以通过整理过去的华为开发者经验、测试用例，并进行对比验证，确保这些修改是正确且有价值的，再将其迁移至 HULK Robot 中，让机器人自动扫描更多代码。

随着华为对 Linux 内核的不断贡献，HULK Robot 成为了一个拥有丰富测试集和探测能力的机器人。

### HULK Robot 如何提交代码？

HULK Robot 随着不断发展，能够为社区贡献更多的代码。从 Linux 内核的统计报告来看，从 Linux 5.1 开始，一直到最新的 5.10 和 5.12，HULK Robot 的提交能力逐渐增强，为社区贡献的代码也越来越多，在内核社区历史 Reported-By 榜单上排名第二。

魏勇军表示，HULK Robot 背后是一套非常复杂的分布式系统，本质上是一个庞大的测试集和一系列先进的自动化测试、问题检测手段。与其他 Linux 内核测试机器人不同的是，华为因开发 openEuler 发行版，在内核维护上付出了大量时间和精力，积累了丰富的经验，因此可以在其中贡献更多代码。

对于魏勇军来说，HULK Robot 当前的重点已不仅仅是准备更多的测试集，而是进一步优化整个内核检测流程。目前每次版本变更时，HULK Robot 都需要运行一次完整的测试集，费时费力。他希望 HULK Robot 能够更加智能，当代码更改后，只检测涉及相应功能的测试集，而不是整个庞大的测试集。此外，他还计划加入更多的 DevOps 能力，让更多人参与到 HULK Robot 的开发工作中，更好地为内核贡献代码。这些改进可以使 HULK Robot 更高效地为社区提供贡献。

正因为 HULK Robot 的存在，整个社区得以拥有一个更加稳健和安全的操作系统，整个社区都从中受益。

### 社区、开源、机器人

长期以来，机器人在开源项目的维护中广泛应用，但在开源项目本身的维护中参与较少。借此机会，我与魏勇军交流了看法。

魏勇军表示，Linux 内核的维护风格非常社区化，社区维护者默认信任每一个提交贡献者都是为了社区的长期发展而贡献代码。因此，社区维护者并不拒绝机器人参与内核维护。如果机器人的贡献对整个社区的长期发展有益，社区支持更多的自动化机器人参与贡献。

也许在未来，我们可以看到 HULK Robot 开源，让整个社区一起参与贡献，更好地维护 Linux 内核。

### 用技术赋能社区

一直以来，Linux 社区都因为资深维护者的流失而担忧，担心社区不再有新鲜血液注入，最终失去活力。

HULK Robot 为此提供了新的思路：将资深维护者的经验固化为代码，使知识得以传承。HULK Robot 的自动化运行机制为社区提供了源源不断的贡献和补丁，使系统变得更加好用和安全。从某种角度来看，HULK Robot 解放了 Linux 内核的贡献者，让他们可以更专注于更重要的事情，将繁琐复杂的工作交给机器人完成，从而降低了对社区新鲜血液的需求。

HULK Robot 不仅为上游提供了新鲜血液，更是 openEuler 长期发展不可或缺的护道人。正是 HULK Robot 背后的默默工作，才使得 openEuler 能够更加稳固地运行，为广大开发者提供一个简单、好用、实用的 Linux 发行版。

我们需要新鲜血液吗？当然需要，但如果我们能把门槛降低，这不也是一件好事吗？
