User: How to configure and manage firewall rules on Linux system?
Assistant: Common firewall software on Linux systems include iptables and firewalld. The specific configuration and management steps are as follows: 1. iptables Step 1: View iptables rules You can use the following command to view the current iptables rules: ``` iptables -L -n # View all rules iptables -L INPUT -n # View the rules of the INPUT chain ``` Step 2: Configure iptables rules You can use the following command to add iptables rules: ``` iptables -A INPUT -p tcp --dport 22 -j ACCEPT # Allow ssh connection iptables -A INPUT -p tcp --dport 80 - j ACCEPT # Allow http connection iptables -A INPUT -j DROP # Do not allow other connections``` Step 3: Save iptables rules``` iptables-save > /etc/sysconfig/iptables #Save iptables rules to a file`` ` 2. firewalld Step 1: View firewalld rules You can use the following command to view the current firewalld rules: ``` firewall-cmd --list-all # View all rules firewall-cmd --zone=public --list-all # View Rules in the public area``` Step 2: Configure firewalld rules You can use the following command to add firewalld rules: ``` firewall-cmd --permanent --add-port=22/tcp # Allow ssh connection firewall-cmd --permanent - -add-port=80/tcp # Allow http connection firewall-cmd --set-default-zone=drop # Set default connection rejection``` Step 3: Overload firewalld rules``` firewall-cmd --reload # Heavy load firewalld rules ```
