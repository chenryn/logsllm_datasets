    - 处理：动词将不会被处理，并将返回给发送应用程序，主要返回代码为 NAP_UNEXPECTED_SYSTEM_ERROR。

61. **SNASW-4-MS_LOG_13: 异常 - %lu - 尝试发送警报时内存不足 [字符]**
    - 说明：系统无法分配所需内存来发送警报。
    - 处理：警报将不会被发送，但其错误代码将被记录。

62. **SNASW-4-MS_LOG_14: 异常 - %lu - MDS_MU 处理时内存不足 [字符]**
    - 说明：系统无法为处理 MDS_MU 分配所需的内存。
    - 处理：MDS_MU 将不会被处理。如果 MDS_MU 来源于本地节点，则包含 MDS_MU 的 SEND_MDS_MU 动词将返回给发送应用程序，并指出错误。如果 MDS_MU 不来源于本地节点，则错误消息将返回给原始应用程序。

63. **SNASW-4-MS_LOG_15: 异常 - %lu - 在 MDS_MU 处理中检测到相关器冲突 [字符]**
    - 说明：在处理 MDS_MU 时检测到相关器冲突。
    - 处理：MDS_MU 将不会被处理。如果 MDS_MU 来源于本地节点，则包含 MDS_MU 的 SEND_MDS_MU 动词将返回给发送应用程序，并指出错误。如果 MDS_MU 不来源于本地节点，则错误消息将返回给原始应用程序。如果有其他应用程序受到影响，也可能发送错误消息。

64. **SNASW-4-MS_LOG_16: 异常 - %lu - 存储空间不足，无法将警报数据传递给注册的警报处理器 [字符]**
    - 说明：存储空间不足，无法将警报传递给注册的警报处理器。
    - 处理：警报将不会被发送，但会被记录。

65. **SNASW-4-MS_LOG_17: 异常 - %lu - 以下警报未能发送到焦点 [字符]**
    - 说明：系统收到发送失败的警报，但没有可用的保持警报功能来存储警报，直到建立新的焦点。
    - 处理：警报将不会被发送，但会被记录。

66. **SNASW-4-MS_LOG_18: 异常 - %lu - 警报发送错误未与发送警报队列中的条目关联 [字符]**
    - 说明：系统无法将警报发送错误与发送警报队列中的警报关联起来。发送警报队列太小，原始警报已被删除，或者之前的错误阻止了警报被保留在队列中。
    - 处理：警报将不会被发送到焦点。

67. **SNASW-3-MS_LOG_22: 问题 - %lu - 收到错误消息，报告我们的 MDS_MUs 格式无效 [字符]**
    - 说明：MS 功能收到一条错误消息，报告我们节点发送的 MDS_MU 格式错误。
    - 处理：错误消息将被记录，但无法采取进一步措施。忽略该消息可能会导致更多意外情况。

68. **SNASW-4-MS_LOG_23: 异常 - %lu - 由于内存不足，MS 功能无法发送请求 [字符]**
    - 说明：MS 功能无法获取空间来发送消息。
    - 处理：消息将不会被发送。如果是请求焦点服务的消息，MS 将通过其他方式追求焦点服务。如果是撤销先前焦点的消息，消息将不会被发送。节点仍将在焦点的控制范围内。

69. **SNASW-4-MS_LOG_24: 异常 - %lu - MS 功能收到交付失败的错误消息 [字符]**
    - 说明：MS 功能收到一条错误消息，假设这是由于发送失败造成的。
    - 处理：收到的错误消息将被记录，以便检查发送失败类型。如果发送失败影响焦点表，MS 功能将采取适当的恢复措施。

70. **SNASW-4-MS_LOG_27: 异常 - %lu - MDS 创建序列中的内存故障 [字符]**
    - 说明：MDS 在创建序列中无法获取所需内存。
    - 处理：MDS 创建将失败。

71. **SNASW-4-MS_LOG_28: 异常 - %lu - 存储空间不足，无法处理收到的 MS 功能信息 [字符]**
    - 说明：存储空间不足，无法处理收到的 MS 功能信息。
    - 处理：发生此故障的类别将被重置。

72. **SNASW-4-MS_LOG_29: 异常 - %lu - 在 MDS_MU 头部中发现解析错误 [字符]**
    - 说明：MDS 收到无法正确解析的 MDS_MU。
    - 处理：将引发警报。如果 MDS_MU 来源于本地应用程序，则 SEND_MDS_MU 将返回 NAP_PARAMETER_CHECK 和 NAP_INVALID_MDS_MU_FORMAT 返回代码。

73. **SNASW-4-MS_LOG_31: 异常 - %lu - 存储空间不足，无法为错误关联排队警报/解决方案 [字符]**
    - 说明：存储空间不足，无法为错误关联排队警报或解决方案。
    - 处理：警报或解决方案将不会被排队。如果收到关于发送此警报或解决方案失败的错误消息，MDS 将无法关联它。如果警报或解决方案成功发送，则不会出现进一步的问题。

74. **SNASW-4-MS_LOG_32: 异常 - %lu - MDS 无法获取内存来发送 TP_ENDED [字符]**
    - 说明：MDS 无法获取缓冲区来发送 TP_ENDED for SEND_TP。
    - 处理：TP_ENDED 将不会被发送。

75. **SNASW-4-MS_LOG_33: 异常 - %lu - MDS 无法获取 SNASVCMG 会话列表条目的内存 [字符]**
    - 说明：MDS 无法获取 SNASVCMG 会话控制块的内存。
    - 处理：控制块将不会被获取。

76. **SNASW-4-MS_LOG_34: 异常 - %lu - MDS 事务程序中收到意外的 what_received [字符]**
    - 说明：MDS 事务程序收到意外的 what_received。
    - 处理：事务程序将重新初始化。正在发送的 MDS_MU 将返回给原发者（如果在原发节点检测到错误），否则原发者将通过错误消息通知发送失败。

77. **SNASW-4-MS_LOG_36: 异常 - %lu - 以下本地发起的 MSCAPS 消息未能发送 [字符]**
    - 说明：系统收到本地发起的 MS 功能消息的发送失败。
    - 处理：MS 功能将把发送失败视为与伙伴节点的连接丢失。如果影响焦点关系，MS 功能将采取适当的恢复措施。

78. **SNASW-4-MS_LOG_37: 异常 - %lu - MDS 无法为链接控制块分配空间 [字符]**
    - 说明：系统无法分配资源来为收到的 PU_STATUS 消息设置链接控制块。
    - 处理：MS 将把活动链接视为未知。通过该链接收到的任何消息将被返回为不可送达。指定 PU_STATUS 中链接的任何 TRANSFER_MS_DATA 或 SEND_MDS_MUs 将返回 link_name 未知。

### 通道日志

79. **SNASW-4-CH_LOG_0: 异常 - %lu - LU 类型 012 或 3 格式错误 [字符]**
    - 说明：LU 类型 012 或 3 格式错误。这可能表示互操作性问题。
    - 处理：包含错误的 PIU 将被丢弃。

80. **SNASW-4-CH_LOG_1: 异常 - %lu - LU 类型 012 或 3 会话因存储空间不足而异常结束 [字符]**
    - 说明：LU 类型 012 或 3 会话因存储空间不足而异常结束。
    - 处理：会话将被停用，并返回指定的错误代码。

81. **SNASW-4-CH_LOG_2: 异常 - %lu - LU 类型 012 或 3 会话在接收数据时异常结束 [字符]**
    - 说明：LU 类型 012 或 3 会话在接收数据时异常结束。
    - 处理：会话将被停用，并返回指定的错误代码。
