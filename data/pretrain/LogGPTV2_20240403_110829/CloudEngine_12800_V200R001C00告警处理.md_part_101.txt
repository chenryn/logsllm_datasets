原因1：单板内存使用率超过设定的过载门限。
原因2：单板存储介质使用率超过预警阈值。
原因3：单板存储介质使用率超过预警极限。
处理步骤
l 原因1：内存使用率低于设定的过载门限。
a. 执行display memory threshold命令查看内存使用率及其过载门限值。
n 如果内存使用率高于过载门限值，则请执行步骤2。
n 如果内存使用率低于过载门限值，则请执行步骤4。
b. 减少不需要的业务和配置，降低内存使用率，检查告警是否恢复。
n 如果不再产生告警，则此告警问题解决。
n 如果继续产生告警，则请执行步骤3。
c. 执行set memory threshold threshold-value [ restore restore-threshold-value ]
[ slot slot-id ]，命令调整过载门限值，检查告警是否恢复。
n 如果不再产生告警，则此告警问题解决。
n 如果继续产生告警，则请执行步骤4。
d. 请收集日志文件，并联系技术支持人员。
l 原因2：单板存储介质使用率超过预警阈值。
a. 在用户视图下执行cd命令进入相应目录，执行dir命令查看存储空间使用率。
n 如果存储空间使用率高于过载门限值，则请执行步骤2。
n 如果存储空间使用率低于过载门限值，则请执行步骤3。
b. 删除不需要的文件，降低存储空间使用率，检查告警是否恢复。
n 如果不再产生告警，则此告警问题解决。
n 如果继续产生告警，则请执行步骤3。
c. 请收集日志文件，并联系技术支持人员。
l 原因3：单板存储介质使用率超过预警极限。
a. 在用户视图下执行cd命令进入相应存储介质目录，执行dir命令查看存储介质
的存储空间使用率。
n 如果存储介质的存储空间使用率高于满载门限值，则请执行步骤2。
n 如果存储介质的存储空间使用率低于满载门限值，则请执行步骤3。
b. 删除不需要的文件，降低存储介质的存储空间使用率，检查告警是否恢复。
n 如果不再产生告警，则此告警问题解决。
n 如果继续产生告警，则请执行步骤3。
c. 请收集日志文件，并联系技术支持人员。
