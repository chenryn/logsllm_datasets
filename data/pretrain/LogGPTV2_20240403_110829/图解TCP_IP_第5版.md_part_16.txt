## 第3章 数据链路
### 73.3.2 以太网的分类
以太网根据所使用的通信电缆类型及传输速度的不同，发展出了多种不同的标准。"10BASE"中的“10”、“100”、“1G”和“10G”分别代表了10Mbps、100Mbps、1Gbps以及10Gbps的传输速率。而紧随其后的字符如“5”、“2”、“T”、“F”等则表示了所采用的传输介质。

当传输速率一致但传输介质不同时，可以通过允许更换传输介质的中继器或集线器进行连接；而在传输速率不同的情形下，则需使用支持速率变更的设备，比如网桥、交换机或路由器。

以下是以太网的主要分类及其特点：

| 以太网种类 | 电缆最大长度 | 电缆种类 | 特点 |
| --- | --- | --- | --- |
| 10BASE2 | 185m（最多30个节点） | 同轴电缆 | - |
| 10BASE5 | 500m（最多100个节点） | 同轴电缆 | - |
| 10BASE-T | 100m | 双绞线（UTP CAT3-5） | - |
| 10BASE-F | 1000m | 多模光纤（MMF） | - |
| 100BASE-TX | 100m | 双绞线（UTP CAT5/STP） | - |
| 100BASE-FX | 412m | 多模光纤（MMF） | - |
| 100BASE-T4 | 100m | 双绞线（UTP CAT3-5） | - |
| 1000BASE-CX | 25m | 屏蔽铜线 | - |
| 1000BASE-SX | 220m/550m | 多模光纤（MMF） | - |
| 1000BASE-LX | 550m/5000m | 多模/单模光纤（MMF/SMF） | - |
| 1000BASE-T | 100m | 双绞线（UTP CAT5/5e） | - |
| 10GBASE-SR | 26m~300m | 多模光纤（MMF） | - |
| 10GBASE-LR | 1000m~2500m | 单模光纤（SMF） | - |
| 10GBASE-ER | 3000m/4000m | 单模光纤（SMF） | - |
| 10GBASE-T | 100m | 双绞线（UTP CAT6a） | - |

#### 传输速度与计算机内部表现值
在计算机内部，数据通常以二进制形式表示，因此常用如下换算：
- 1K = 1024
- 1M = 1024K
- 1G = 1024M

然而，在以太网中，传输速率是基于时钟频率定义的，因此：
- 1K = 1000
- 1M = 1000K
- 1G = 1000M

请勿将两者混淆。

### 73.3.3 以太网的历史
最早被标准化的以太网采用了同轴电缆构建的10BASE5网络。随后出现了许多其他版本，包括细同轴电缆的10BASE2（Thin Ethernet）、双绞线的10BASE-T（Twisted Pair Ethernet）、高速100BASE-TX（Fast Ethernet）、1000BASE-T（Gigabit Ethernet）以及10G以太网等。

初期，以太网访问控制主要依靠CSMA/CD（载波侦听多路访问/冲突检测）机制，这是一种半双工通信方式。随着技术的发展，尤其是ATM交换技术和CAT5 UTP电缆的普及，以太网结构逐渐演变为直接与交换机相连的方式，减少了冲突的发生，并提高了网络速度。

从此以后，以太网凭借其简单且低成本的优势，迅速取代了FDDI等其他局域网技术。如今，以太网已成为主流有线局域网技术，从100Mbps到10Gbps的速度不断提升，使得研究其他有线局域网技术变得不再必要。

所有这些以太网标准都由IEEE 802.3工作组（Ethernet Working Group）进行标准化。以下是IEEE 802委员会的部分工作小组及其职责：

- **IEEE 802.1**：高层局域网协议工作组
- **IEEE 802.2**：逻辑链路控制工作组
- **IEEE 802.3**：以太网工作组（CSMA/CD）
- **IEEE 802.4**：令牌总线工作组
- **IEEE 802.5**：令牌环工作组
- **IEEE 802.6**：城域网工作组
- **IEEE 802.7**：宽带TAG
- **IEEE 802.8**：同步局域网工作组
- **IEEE 802.9**：安全工作组
- **IEEE 802.11**：无线局域网工作组
- **IEEE 802.12**：需求优先工作组（100VGAnyLAN）

### 73.3.4 以太网帧格式
以太网帧的前端包含一个称为前导码（Preamble）的部分，它由交替的0和1组成，用于标记帧的开始并确保接收方能够同步。前导码后是一个名为SFD（Start Frame Delimiter）的字段，其值为“11”。前导码和SFD总共占用8个字节。

以太网帧本体的前端是帧头，共14个字节，包括6个字节的目标MAC地址、6个字节的源MAC地址和2个字节的上层协议类型。

| 字段 | 长度（字节） | 描述 |
| --- | --- | --- |
| 目标MAC地址 | 6 | 目标工作站的物理地址 |
| 源MAC地址 | 6 | 发送工作站的物理地址 |
| 类型 | 2 | 标识上一层协议类型的编号 |

数据部分可以容纳46至1500字节的数据。帧尾是一个4字节的FCS（Frame Check Sequence），用于检测帧是否损坏。通过计算整个帧除以生成多项式的余数并与发送端的FCS进行比较，可以判断帧是否有误。

在IEEE 802.3以太网中，帧头的“类型”字段实际上表示的是数据长度。此外，数据部分的前端还包含了LLC和SNAP字段，用于标识上一层协议类型。

在VLAN（虚拟局域网）中，帧格式会有所变化，添加了VLAN标签字段，具体格式如下：

| 字段 | 长度（比特） | 描述 |
| --- | --- | --- |
| VLAN ID | 12 | 虚拟局域网标识符 |
| 优先级 | 3 | 帧的优先级 |
| CFI | 1 | 规范格式指示符 |
| 类型 | 16 | 上一层协议类型 |

更多关于协议类型的详细信息，请参考以下网站：
- [IEEE Ethertype](http://standards.ieee.org/regauth/ethertype/eth.txt)
- [IANA Ethernet Numbers](http://www.iana.org/assignments/ethernet-numbers)
