Hello folks, I have a really irritating bug in Visual Studio 2015 Update 2
using TypeScript. The auto complete dialog in combination with generics works
a little bit fuzzy. It treats it like a greater than comparison. Look at the
code below.
**TypeScript Version:**
1.8.30.0
**Code**
    // Function declaration
    function invoke(methodName: string, ...args: any[]) {
    }
    // Call the function, Type first letter of the generic T.
    invoke<M
    // Auto complete window pops up, which is expected.
    // I choose Model from the auto complete window using "enter".
    invoke < Model
    // Notice the inserted spaces, the compiler thinks its a greater than comparison. While its actually a generic.
**Expected behavior:**
    // I expect Model to be inserted without spaces.
    this.invoke<Model
**Actual behavior:**
    // I actually inserts it with spaces.
    this.invoke < Model
