### 2.5 编址规则扫描模块
通过学习编址规则或利用已知的编址规则，本功能模块可以推测出一批可能存活的 IPv6 地址集合。这些地址依据统计学观点具有较高的存活概率。编址规则可以基于 IP 地址信息熵的规律，也可以基于各企业机构内部的规范定义。具体实现方法如下：

1. **地址推测**：若能获取大量存活的 IPv6 地址样本，可通过多种算法分析样本地址的规律，并推测出其他可能存活的 IPv6 地址。或者，采用人工智能的方式，根据大规模样本训练出 IPv6 地址存活模型，进而由该模型对未知的 IPv6 地址段进行存活地址的推测。

2. **存活性扫描**：对步骤 1 中推测出的 IPv6 地址集进行存活性扫描探测，得到存活的地址集合。

3. **去重处理**：对步骤 2 中的 IPv6 地址进行去重处理，生成该模块的 IPv6 存活地址集合 Address-Pool-4，并将其汇入总的 IPv6 存活地址集中。

### 2.6 随机地址探测扫描模块
通过随机方式，可以在一定概率下发现网络中存活的 IPv6 地址。本功能模块基于已知的存活 IPv6 地址段，随机生成部分 IPv6 地址并判断其存活性。随机生成可以通过已知的算法或人工智能工具实现。具体实现方法如下：

1. **地址扩充**：通过对输入的 IPv6 地址信息集中的每一个 IPv6 地址进行前后一定范围的地址范围扩充，生成对应的 IPv6 地址段信息。

2. **随机地址生成与验证**：结合随机抽样算法等方式，对步骤 1 生成的每一个 IPv6 网段进行一定数量的随机地址生成，并进行存活性验证，仅保留经验证存活的地址。

3. **去重处理**：对步骤 2 中的 IPv6 地址进行去重处理，生成该模块的 IPv6 存活地址集合 Address-Pool-5，并将其汇入总的 IPv6 存活地址集中。

### 3 结果分析
通过设计 IPv6 存活地址快速发现模型，开发相应的原型，并在被授权的网络上进行应用，获得了丰富的数据。进一步分析这些数据，可以发现一些 IPv6 存活地址的共性。

在某些网络中，特别是云资源池中，管理员倾向于将同一类型的资产设备部署在一个或相邻的若干个 IPv6 网段中。例如，在某网段中，IP 地址均被分配给 Web 服务的 HAproxy 代理。由于这种地址使用方式将大量同类型设备集中在相邻的地址中间，因此当设备出现漏洞时，必须对全网段的设备进行加固，以避免恶意攻击者利用漏洞一次攻击大量设备。

对于 Web 网站服务器，往往容易在其所在网段中暴露数据库服务和远程连接服务等高危服务。在 IPv4 网络中，Web 服务一般是通过地址转换方式对外提供服务，相应的数据库和远程连接等服务被较好地控制在内网范围内，风险较小。而在 IPv6 网络中，缺乏地址转换带来的天然安全隔离，容易导致更大的互联网暴露，带来更大风险。

### 4 结束语
IPv6 的广泛使用带来了更多的 IPv6 暴露面，开展常态化的 IPv6 资产与风险管理至关重要。在日常的安全工作中，要做好 IPv6 资产的管理，需要有效结合技术手段与管理手段，二者互补；有效控制 IPv6 暴露面的数量，加强网络安全策略的梳理，做到非必要不增加暴露面；不断探索更高效的 IPv6 资产自动化发现方法和 IPv6 快速扫描方法。
