=== 基础搜索语法
日志易支持基础搜索语法和高级搜索语法。在故障排查等简单场景中，仅需掌握基础语法即可。
==== 全文检索
检索一个或多个查询词时，系统不区分大小写。如果查询多个关键字，则不区分这些词的先后顺序，且查询结果将同时包含这些词。例如
 hello world
相当于
 hello AND world
即日志结果中必须包含 hello 与 world。
==== 短语查询
如果需要查询短语（即短语内的多个词都必须都出现，而且保持同样的顺序及相邻状况），需要用双引号把短语括起来，查询结果不区分大小写。例如查询“hello world”，匹配的日志也必须包含“hello world”。
==== 字段值
在不了解字段是什么，以及如何使用它们的时候，您无法利用日志易的很多高级功能。
字段以多种形式存在于计算机数据中。通常，字段是一个值（行上有固定的、带分隔符的位置）或键值对，其中每个字段名称有一个单一值。字段可以为多值，即它会在一个事件中多次出现，并且每次出现都有一个不同的值。（日志易暂时不对多值做特殊处理，您可以参阅《日志易V1.8数据接入手册》中KeyValue规则章节选择多值的保留方式。）
一些字段示例如下：apache.clientip 为访问 Web 服务器的 IP 地址，timestamp 为事件的时间戳，以及 hostname 为服务器的域名。在日志易中，字段是区分不同事件的可搜索的键值对，因为不是所有事件都具有相同的字段和字段值。利用字段可编写更多定制的搜索以检索您所需的特定事件。
数据接入日志易后，将提取每个处理事件的默认和其他索引字段。默认字段包括hostname、appname、tag以及logtype。其他索引字段，取决于数据对应的字段提取规则，详细介绍请参阅《日志易数据接入手册》。
.格式
====
field:value
====
field是字段名，应当为字符串，搜索时区分大小写字母，不得使用? *等字符；value可以是字符串或数值，字符串区分大小写字母，可以使用? *等字符。此用法也可在界面左栏的字段值处点击建立字段过滤。例如
 tag:test
匹配 tag 字段值为 "test" 的日志；
 apache.status:200
匹配 apache.status 字段值为200的日志；
 apache.status:*
匹配带 apache.status 字段的日志；
 apache.clientip:192.168.1.1*
匹配 apache.clientip 为 192.168.1.1，192.168.1.103，192.168.1.114 等值的日志。
==== 逻辑运算符
AND，OR，NOT，( )，AND、OR、NOT 必须大写。
    term1 AND term2
匹配的日志里 term1 和 term2 都必须都出现，AND 默认可缺省；
    term1 OR term2
匹配的日志里 term1 或 term2 出现1个即可；
    NOT term
匹配的日志里不含有 term；
可用（）改变布尔运算规则，例如：
 term1 AND (falseterm2 OR term3)
表示匹配的日志里必须包含 term1，以及falseterm2、term3中的任一个。
==== 数值范围
.格式
====
field:[value1 TO value2]
field:{value1 TO value2}
field:{value1 TO value2]
field:[value1 TO value2}
field:>value
field:>=value
field:400 
匹配 apache.status 值大于400的日志。
==== 通配符
'*' 匹配0个或多个字符，单独搜索 * 会显示所有日志；？匹配1个字符。例如：
 syslog.appName:abc* 
 syslog.appName:ab? 
都会匹配 syslog.appName 为 abc 的日志
 tag:mytag* 
匹配 tag 为 mytag 或 mytag1的日志
 tag:mytag? 
匹配 tag 为 mytag1 的日志，但 无法匹配 tag 为 mytag 的日志。
==== 基本正则表达式
.格式
====
/regexp/
====
支持[ ] (范围), {n} (出现n次), {n,m} (出现n次到m次), {n,} (至少出现n次) 等基本正则表达式。 例如
 /[wb]all/ 
匹配包含 wall 或 ball 的日志
 /0{4,}/
匹配包含0000的日志
 /Chrome[0-9]{2}/
匹配Chrome 后面带有2位数字的日志。
==== 特殊字符的检索处理
只有字母、数字和字符检索时会匹配在事件日志。特殊字符+ – | | !（）{ } [ ] ^ " ~ *？：\ 等不应包括在查询语句的一部分来匹配，如果你在搜索时有上述特殊字符请务必使用引号括起来。例如，这两个搜索将显示相同的结果：
例1：
    "http://yourwebsite.com:9000"
例2：
    "http yourwebsite com 9000" 
下面这个搜索将导致一个无效的查询，由于 : 没有被""括起来引用:
    http://yourwebsite.com:9000
==== 区分大小写
日志易在搜索时除以下情况情况区分大小写之外，其他情况不区分大小写：
* appname/tag等内置字段
* 字段名称
* 字段值在指定字段（非raw_message）的情况下
* 数值范围TO
* 逻辑运算符AND/OR/NOT
全文检索不区分大小写，指定字段为raw_message的情况；不指定字段，默认为raw_message的情况。如搜索语句是"SUCCESS"，则小写的success也会搜出来。
=== 搜索技巧
为了保证您能搜索到想要的结果，在搜索时请注意以下事项：
1. 明确搜索权限
+
普通用户必须得系统管理员分配搜索权限才可以进行搜索，普通用户登录后应该先去搜索权限页面查看当前拥有的搜索范围，如果没有被分配搜索权限或分配的搜索权限不对，应立即联系系统管理。
日志易系统只有拥有\__admin__角色的管理员可以新建/编辑/删除搜索权限，所以管理员需合理规划，按需分配给普通用户搜索权限。
2. 请避免直接使用特殊字符，如果需要请使用引号括起来
+
特殊字符: `+ – & || ! ( ) { } [ ] ^ ” ~ * ? : \`
+
这些字符是日志易的查询语言保留字，请避免直接出现在查询中。
3. 明确时间范围
+
请检查您选择的时间范围，并根据搜索界面上的时间轴柱状图趋势，按需缩放搜索的时间范围。较小的时间范围有助于更快的获取真正有用的关键信息。
4. 检查你的拼写
+
小心拼写错误，您的日志事件和查询语句都有可能出错。
5. 不要使用部分字符串
+
日志易底层采用搜索引擎技术。检索时请尽量使用完整单词。下面是一条日志示例：
    Timer-2 S3Stats.processStatEvents INFO: Processed stats for Stats-6297-138
这个日志接入以后，实际索引的是以下单词：
* "timer"
* "2"
* "s3stats.processstatevents"
* "info"
* "processed"
* "stats"
* "for"
* "stats"
* "6297"
* "138"
因此，以下关键词在检索时将成功命中结果返回：
    timer
    processed
    stats*
这些关键词在检索时则无法命中结果：
    event
    time
