**摘要：Red Hat 已经掌握了如何跟随开源社区的步伐，并成功将其商业化。你的公司也可以做到这一点。**

在开源领域，最强大但也最具挑战性的一个方面就是构建和维护一个活跃的社区。红帽（Red Hat）首席执行官 Jim Whitehurst 在与 Slashdot 的一次采访中提到：“只要有强大的社区支持，开源项目总是能够取得胜利”。然而，建立这样一个社区是非常困难的。虽然预测出哪些因素是构成繁荣开源社区的关键并不难，但要准确判断这些因素何时何地能促成像 Linux 或 Kubernetes 这样的成功案例则极其复杂。

因此，明智的做法通常是追随现有的、已经显示出增长潜力的社区，而不是试图从零开始创建新的社区。

### 开源寄生虫策略
Whitehurst 的言论给我留下了深刻的印象。尽管他讨论了许多话题，包括 Linux 桌面到 systemd 等等，但他关于社区的观点尤其引人注目：
> “建立一个新的社区非常艰难。我们红帽也尝试过一些，但我们大部分时间都在寻找那些已经拥有坚实基础的现有项目。当有强大的社区存在时，开源总是会赢。”

这种做法——即强化已有的且正在成长中的社区，看似是一种逃避现实的方式，但实际上是最为理智的选择。早期阶段，由于每个项目都在努力争取足够多的支持者，开源领域常常显得支离破碎。随着时间推移，人们往往会聚集在少数几个领先的项目周围（如 KDE vs. GNOME 或 Kubernetes vs. Docker Swarm 和 Apache Mesos）。最终，维持多个不同标准变得越来越难，大多数参与者都会倾向于选择其中一个赢家（例如 Kubernetes）。

这并不是放弃，而是更好地利用资源并推动标准化的一种方式。以 Linux 为例，它之所以成为如此强大的力量之一，是因为它鼓励在一个统一平台上进行操作系统创新，即使存在不同的分支社区和个人贡献者。红帽之所以能够迅速发展，部分原因在于其早期就做出了明智的选择，选择了赢家（比如 Kubernetes），并通过帮助它们变得更加成功来从中受益。

反过来，这也为其商业模式提供了动力。

### 从社区混乱中获利
对于红帽来说，社区越强大，项目就越成功。不过，“成功”的定义不仅仅是企业愿意采用该项目，更重要的是它们实际会这样做。红帽很早就意识到，那些传统而保守的企业希望既能享受到开源带来的活力，又不需要承担过多的风险。Whitehurst 对此进行了精辟总结：
> “红帽之所以成功，在于我们致力于通过增加每个产品代码的价值来吸引客户。我们的目标是让企业用户更容易地消费开源创新成果。”

仅举一例：对于所有产品，我们都关注生命周期管理。虽然“尽早发布、频繁发布”是开源开发模式的一大优势，但这使得生产环境下的实施变得相当棘手。我们在 Linux 中提供的一个重要价值是，在长达十年以上的时间内为受支持的内核提供错误修复和安全更新，同时确保不会破坏 API 兼容性。这对于需要长期运行应用程序的企业而言极具吸引力。

对于许多渴望参与开源世界的公司来说，最大的挑战在于他们可能认识到了社区的重要性，但却不知道如何不再依赖传统的软件销售模式。毕竟，出售专有软件在过去几十年里一直是一个非常盈利的业务模式，造就了一些世界上最富有的企业。

然而，正如 Whitehurst 所指出的那样，开源需要采取一种全新的方法。他在采访中强调：“你不能仅仅依靠售卖功能，因为这些都是免费提供的。”相反，必须找到其他形式的价值点，比如提供长期稳定的支持服务。虽然这听起来可能有些乏味，但对于红帽来说，每年却可以带来数十亿美元的收入。

这一切都始于社区。更活跃的社区不仅能够促进开源产品的市场化，还能为企业创造更多商业机会。为什么呢？因为更多的开发者参与意味着更高的价值产出，从而使自由发展的项目对企业来说更加可靠。这是一个成功的公式，既保留了开源的所有优点，又避免了受限于过时的商业模式。
