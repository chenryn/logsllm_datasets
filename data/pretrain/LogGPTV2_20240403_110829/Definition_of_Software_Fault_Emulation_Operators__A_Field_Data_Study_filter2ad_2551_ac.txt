### 3.3.4 Algorithm Faults

The occurrences of algorithm faults are summarized in Table 9. The following explanations provide details on the specific types of faults:

- **Missing "if (cond) statement(s)" (MIFS)**: This fault type represents the omission of a block of code that includes an `if` construct and its associated statements.
- **Missing "if (cond)" around statement(s)**: This fault type involves having a number of statements that are always executed instead of being conditionally executed.
- **Missing "if (cond) statement(s) else" before statement(s) (MIEB)**: One or more statements should exist inside the `else` part of a missing `if-else` construct.
- **Missing "if (cond) else statement(s)" around statement(s) (MIEA)**: This fault type represents the situation where one or more statements should be placed inside the `if` part of a missing `if-else` construct.
- **Wrong branch/flow construct**: This fault occurs when the wrong execution flow is used, such as using an `if` construct instead of a `while` loop.
- **Wrong conditional compilation**: These faults occur at the preprocessor level, causing incorrect portions of the source code to be compiled.
- **Extraneous function call (EFC)**: This type of fault involves unnecessary function calls that are not part of a larger expression.
