> 开源社区和行业趋势的每周总览。

![一个人站在一个巨大的计算机屏幕前，屏幕上显示着数字和数据](/data/attachment/album/201910/13/111738qz4h2tqt44tim43h.png "一个人站在一个巨大的计算机屏幕前，屏幕上显示着数字和数据")

作为一家采用开源开发模式的企业软件公司的高级产品营销经理，我定期为产品营销人员、经理及其他影响者发布关于开源社区、市场和行业趋势的更新。以下是本周我和他们最喜欢的五篇文章。

### 《Java 还有用吗？》
* [文章地址](https://sdtimes.com/java/is-java-still-relevant/)

Eclipse 基金会执行董事 Mike Milinkovich 认为，Java 将不断发展以支持新的技术。“我认为 Java 本身将从 JVM 到整个生态系统发生变化，”Milinkovich 表示，“JVM 中任何有助于与 Docker 容器集成的新特性，都将是一个巨大的帮助。因此，我们将期待 Java SE 朝着这个方向发展。”

**影响**: Jakarta EE 是 Java Enterprise Edition 的完全开源版本，奠定了 Java 未来发展的基础。Java 在开发中的成本效益以及开发者多年的经验使其仍然具有价值。结合生态系统中的创新（例如 Quarkus 或 GraalVM），答案显然是“是”。

### 《GraalVM：多语言 JVM 的圣杯？》
* [文章地址](https://www.transposit.com/blog/2019.01.02-graalvm-holy/?c=hn)

虽然大多数关于 GraalVM 的宣传都集中在将 JVM 项目编译成原生程序上，但其 Polyglot API 也极具价值。尽管迁移路径存在一些困难，主要是因为缺乏文档，但 GraalVM 已经是一个引人注目的替代 Nashorn 的选择。

**影响**: 对于开源项目来说，最好的事情之一就是用户开始对一些新奇的应用程序赞不绝口，即使这些应用程序不是主要用例。例如，Polyglot API 的使用就是一个很好的例子。

### 《你可以说我疯了，但 Windows 11 或可以在 Linux 上运行》
* [文章链接](https://www.computerworld.com/article/3438856/call-me-crazy-but-windows-11-could-run-on-linux.html#tk.rss_operatingsystems)

微软已经做了很多工作，使得 Windows 和 Linux 之间的互操作性成为可能。Windows 的 Linux 子系统 (WSL) 的开发人员一直在努力将 Linux API 调用映射到 Windows 中，反之亦然。Carmen Crincoli，微软与存储和独立硬件供应商的合作伙伴经理，在推特上表示：“2017 年归根结底还是 Linux 桌面年。只不过这个桌面是 Windows。”

**影响**: 微软的选择及其与更广泛的开源社区的互动可能会导致科幻般的未来。敬请关注！

### 《Python 正在吞噬世界：一个开发人员的业余项目如何成为地球上最热门的编程语言》
* [文章链接](https://www.techrepublic.com/article/python-is-eating-the-world-how-one-developers-side-project-became-the-hottest-programming-language-on-the-planet/)

Python 核心开发人员和 Python 指导委员会的组成是否能更好地反映 2019 年 Python 用户群的多样性，这是一个问题。Wijaya 称：“我希望看到在所有不同指标上都有更好的代表，不仅在性别平衡方面，而且在种族和其他方面。”

**影响**: 在这个动荡的时代，谁不想听到一位仁慈独裁者（指 Python 创始人）把项目的统治权移交给最经常使用它的人呢？

希望你喜欢这张上周让我印象深刻的列表，并在下周一回来了解更多的开源社区、市场和行业趋势。
