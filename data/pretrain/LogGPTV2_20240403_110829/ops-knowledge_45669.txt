User: How to use Crontab to configure regular backup of MySQL database?
Assistant: The steps are as follows: 1. Log in to the MySQL server to be backed up 2. Enter the following statement in the command line to create a backup script sudo nano /etc/mysql/mysql-backup.sh 3. Add the following backup statement to the script#!/bin/ bash DATE=$(date +%d-%m-%Y) TIME=$(date +%H:%M:%S) MYSQL_USER='root' MYSQL_PASSWORD='password' MYSQL_HOST='localhost' MYSQL_PORT='3306 ' DATABASE_NAME='database_name' BACKUP_DIR='/var/backups/mysql' BACKUP_NAME=$DATABASE_NAME-$DATE-$TIME.sql.gz mysqldump --user=${MYSQL_USER} --password=${MYSQL_PASSWORD} --host =${MYSQL_HOST} --port=${MYSQL_PORT} ${DATABASE_NAME} | gzip > $BACKUP_DIR/$BACKUP_NAME 4. Save and close the backup script 5. Set the backup script to be executable sudo chmod +x /etc/mysql/mysql -backup.sh 6. Test whether the backup script works sudo /etc/mysql/mysql-backup.sh 7. Use crontab to configure regular backup crontab -e Add the following statement in the text editor, the MySQL database will be backed up once a day 0 0 * * * /etc/mysql/mysql-backup.sh 8. Save and close the text editor The backup script will now execute once a day, saving the backup file in the /var/backups/mysql/ directory.
