### 问题描述
我正在尝试在Windows Server 2012 R2上配置工作文件夹（Work Folders），但遇到了一些问题。我已经按照所有步骤完成了服务器配置，并且服务器日志中没有错误。

### 端口443的问题
端口443被“Windows Sync Share”服务占用。如果我停止该服务，端口将不再被使用。但是，我希望保持该服务运行。

### 自动发现的问题
由于其中一个域控制器仍然是Windows Server 2008 R2，我无法使用自动发现功能。因此，我通过直接URL来配置客户端，但出现了以下错误：

### 服务器上的角色和服务
- IIS（带有一个FTP站点）
- WSUS（HTTP端口81，HTTPS端口8350）
- Hyper-V（未使用，仅作为备份）
- WDS
- 文件服务器（带有DFSR）

### 解决方案
经过一番排查，我发现需要创建一个证书，即使所有计算机都在同一个受信任的域内也是如此。以下是解决步骤：

1. **创建并注册证书**
   - 创建一个官方授权或自签名的证书。
   - 使用以下命令在服务器上注册证书：
     ```sh
     netsh http add sslcert ipport=<IP地址>:443 certhash=<证书指纹> appid={CE66697B-3AA0-49D1-BDBD-A25C8359FD5D} certstorename=MY
     ```

2. **安装证书到客户端**
   - 将证书安装到客户端的根证书存储中。

3. **安装更新**
   - 我还必须在服务器上安装General Availability (GA) Update 2。

通过以上步骤，我成功解决了工作文件夹的问题。希望这对您有所帮助！
