User: 请列举出常见的网络攻击类型，以及相应的防御措施。
Assistant: 1. DDoS 攻击：攻击者通过控制大量傀儡计算机（或智能设备），向目标服务器发送大量的请求，使其系统资源消耗殆尽而无法响应合法用户的请求。防御措施：搭建防火墙、CDN、IDS/IPS、DDoS 清洗设备等，以便过滤非法流量。
2. SQL 注入：攻击者在 Web 应用程序中输入恶意代码，从而破坏或提供对后台数据库的访问。防御措施：使用防护软件或构建防火墙，限制数据库特权，使用参数化查询来避免破坏。
3. 垃圾邮件（Spam）：攻击者向大量电子邮箱发送垃圾广告或欺诈信息。防御措施：使用反垃圾邮件软件或服务，阻止恶意发送者的电子邮件地址和域名。
4. 恶意软件（Malware）：包括病毒、间谍软件、木马、蠕虫等。攻击者可以通过恶意软件窃取敏感信息或对系统进行破坏。防御措施：及时更新防病毒软件和操作系统，不要下载未知来源的应用程序，不要点击可疑的链接或附件，使用防火墙和 IDS 等安全设备。
5. CSRF 攻击（Cross-site request forgery）：攻击者利用受害者的浏览器的漏洞，强制对一个网站进行请求，从而向该网站共享用户登录的口令。防御措施：使用随机令牌、检查引荐头等安全措施。 
6. XSS 攻击：攻击者在 Web 页面上注入一个恶意脚本，使用户浏览该页面时执行该脚本造成安全问题。防御措施：使用安全编码技术，过滤并转义用户输入，使用 HTTP 标头等措施。 
7. 钓鱼（Phishing）：攻击者冒充受害者信任的实体（如银行、电商等），通过邮件、短信、社交媒体等手段诱骗信息被盗。防御措施：教育和训练用户，加强监控，使用反钓鱼技术和软件，采取防范措施。
8. Brute Force 攻击：攻击者通过不断尝试用户名和密码的组合，来尝试破解密码。防御措施：使用强密码及两步验证，对请求进行限制封锁等。
9. DNS 欺骗（DNS Spoofing）：攻击者通过篡改域名系统（DNS）的记录，将受害者引向恶意站点。防御措施：该防御策略可以使用DNSSEC技术进行控制。
