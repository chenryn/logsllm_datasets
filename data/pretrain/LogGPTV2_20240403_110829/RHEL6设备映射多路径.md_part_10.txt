  -------------------------------------------------------------------------------
  选项                   描述
  ---------------------- --------------------------------------------------------
  `-l`{.option}          显示来自 `sysfs`{.filename}

  `-ll`{.option}         显示来自 `sysfs`{.filename}

  `-F`{.option}          删除所有不使用的多路经设备。

# [⁠]{#MPIO_admin-troubleshoot.html#dmsetup_queries}5.10. 使用 dmsetup 命令确定设备映射器条目 {.title}
:::
[]{#MPIO_admin-troubleshoot.html#idm139642171785872 .indexterm}
::: para
您可以使用 `dmsetup`{.command}
找出哪个设备映射器条目与多路径的设备映射。
:::
::: para
以下命令显示所有设备映射器设备及其主、副号码。副号码确定 dm
设备的名称。例如：副号码 3 与多路径的设备 `/dev/dm-3`{.filename} 对映。
:::
``` screen
# dmsetup ls
mpathd  (253:4)
mpathep1        (253:12)
mpathfp1        (253:11)
mpathb  (253:3)
mpathgp1        (253:14)
mpathhp1        (253:13)
mpatha  (253:2)
mpathh  (253:9)
mpathg  (253:8)
VolGroup00-LogVol01     (253:1)
mpathf  (253:7)
VolGroup00-LogVol00     (253:0)
mpathe  (253:6)
mpathbp1        (253:10)
mpathd  (253:5)
```

# [⁠]{#MPIO_admin-troubleshoot.html#multipath_config_confirm}5.11. 使用 multipathd 互动控制台进行故障排除 {.title}
:::
[]{#MPIO_admin-troubleshoot.html#idm139642169229200
.indexterm}[]{#MPIO_admin-troubleshoot.html#idm139642169227600
.indexterm}
::: para
`multipathd -k`{.command} 命令是一个 `multipathd`{.command}
守护进程的互动界面。执行此命令将进入互动的多路径控制台。执行此命令后，您可以输入
`help`{.literal}
来获得可用的命令列表。您可以输入互动命令，或者您还可以输入
`CTRL-D`{.literal} 退出。
:::
::: para
`multipathd`{.command}
互动控制台可用来在您的系统有问题时进行故障排除。例如：以下命令会在退出控制台前显示多路径配置，其中包括默认配置。
:::
``` screen
# multipathd -k
> > show config
> > CTRL-D
```
::: para
以下命令确定多路径已经识别了所有对 `multipath.conf`{.filename} 的修改。
:::
``` screen
# multipathd -k
> > reconfigure
> > CTRL-D
```
::: para
使用以下命令组确定路径检查器工作正常。
``` screen
# multipathd -k
> > show paths
> > CTRL-D
```
