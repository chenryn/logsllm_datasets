毫无疑问，Linux 是由那些拥有深厚计算机知识背景且才华横溢的程序员所创造。让我们来看看这些著名的 Linux 开发者们推荐给当今开发者的书籍和技术资料吧，也许你已经读过其中几本了呢？

Linux 无疑是 21 世纪的操作系统之一。尽管 Linus Torvalds 在构建开源社区方面做出了巨大贡献，并在许多社区决策中发挥了重要作用，但网络专家和开发者之所以接受 Linux，主要还是因为它卓越的代码质量和高可用性。Torvalds 是一位编程天才，但他也得到了其他同样才华横溢的开发者的无私帮助。

我咨询了 Torvalds 和其他一些顶级 Linux 开发者，了解哪些书籍曾帮助他们成为顶尖开发者。下面请听我一一道来。

### C 语言的辉煌

Linux 是在上世纪 90 年代开发出来的，同时还有许多其他基础性的开源软件。那个时代的开发者使用的工具和语言反映了当时的特征，尤其是 C 语言。虽然 [C 语言可能不再流行](https://www.codingdojo.com/blog/7-most-in-demand-programming-languages-of-2018/)，但对于许多成功的开发者来说，C 语言是他们在实际开发中使用的第一门语言，这一点也在他们推荐的对他们影响深远的书单中有所体现。

Torvalds 表示：“你不应该再选用我那个时代使用的语言或开发方式。”他的编程之路始于 BASIC，然后转向机器码（“甚至不是汇编语言，而是真正的‘二进制’机器码”，他解释道），接着转向汇编语言和 C 语言。

“任何人都不应该再从这些语言开始进入编程领域了，”他补充说。“其中一些语言今天已经没有意义（如 BASIC 和机器语言）。尽管 C 仍然是一门主流语言，我也不建议从它开始。”

这并不是因为他不喜欢 C 语言。毕竟，Linux 是用 [GNU C 语言](https://www.gnu.org/software/gnu-c-manual/) 编写的。“我一直认为 C 是一门伟大的语言，它有着非常简单的语法，适用于多种开发方向，但我怀疑你会遇到重重挫折，从你的第一个‘Hello World’程序到真正能开发出实用的东西之间有很长一段路要走。”他认为，按照现代标准，如果作为入门语言的话，C 语言的成本太高。

在他那个时代，Torvalds 唯一的选择就是 Brian W. Kernighan 和 Dennis M. Ritchie 合著的《C 编程语言，第二版》（通常称为 K&R）。“这本书简洁精炼，但你需要有一定的编程背景才能欣赏它。”Torvalds 说道。

Torvalds 并不是唯一一个推荐 K&R 的开源开发者。以下几位也同样引用了这本他们认为值得推荐的书籍：
- Linux 和 Oracle 虚拟化开发副总裁 Wim Coekaerts
- Linux 开发者 Alan Cox
- Google 云 CTO Brian Stevens
- Canonical 技术运营部副总裁 Pete Graner

这些推荐不仅展示了 C 语言的经典地位，也体现了这些顶级开发者们对基础知识的重视。
