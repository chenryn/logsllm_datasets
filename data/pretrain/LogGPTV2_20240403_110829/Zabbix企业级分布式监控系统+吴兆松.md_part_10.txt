### 4.4 添加主机

#### 目的
添加主机是为了对具体设备进行监控。

#### 步骤
1. 依次点击 **Configuration → Hosts → Create host**，如图4-15所示。

**图4-15**

2. 在图4-15中单击“Create host”，将出现图4-16所示的界面。

**图4-16**

#### 参数说明
表4-1列出了图4-16中各参数的详细说明。

| 参数 | 描述 |
| --- | --- |
| **Hostname** | 输入一个唯一的主机名。允许使用大小写字母、数字、标点符号和下划线。注意：客户端配置文件中的主机名必须与此处输入的值相同，用于主机存活检查。 |
| **Visible name** | 如果设置此名称，它将出现在主机列表和地图等地方。支持UTF-8编码。 |
| **Groups in groups** | 选择主机所属的群组。每个主机必须至少属于一个主机组。 |
| **New group** | 创建一个新的群组并自动链接到该主机。如果为空，则忽略此选项。 |
| **Agent interfaces** | 主机支持的接口协议类型包括：Agent、SNMP、JMX和IPMI。要增加新接口，请单击“Add”按钮并输入IP/DNS、连接方式和端口等信息。 |
| **IP address** | 要监控主机的IP地址（可选，默认为IP）。 |
| **DNS name** | 要监控主机能够解析的DNS名称（可选）。 |
| **Connect to** | 选择与Agent通信的方式（IP或DNS）。 |
| **Port** | TCP协议的端口，默认值为10050。 |
| **Monitored by proxy** | 选择是否通过代理服务器进行监控。 |
| **Status** | 设置主机的状态（Monitored或Not monitored）。 |

**图4-17**

### 4.5 配置图形

#### 目的
图形（Graphs）以视觉化形式展示数据。它们可以存在于主机或模板中。

#### 步骤
1. 在主机或模板中选择“Graphs”，如图4-22所示。

**图4-22**

2. 单击“Create graph”，如图4-23所示。

**图4-23**

3. 出现新建图形的界面，如图4-24所示。输入名称并选择所需的监控项（Items）。

**图4-24**

4. 单击“Save”保存，在对应的主机或模板中可以看到新增的图形，如图4-25所示。

**图4-25**

5. 单击“Graphs”，可以看到新添加的图形，如图4-26所示。

**图4-26**

6. 单击刚才创建的图形，选择“Preview”查看图形，如图4-27所示。

**图4-27**

### 图形参数说明

表4-3列出了图形中各项参数的详细说明。

| 参数 | 描述 |
| --- | --- |
| **Sort order (0-100)** | 确定绘图顺序，可以在开始的线条箭头处拖放项目以设定分类顺序。 |
| **Name** | 显示Item的名称。 |
| **Type** | 仅对饼图和分解饼图适用。<br>Simple: 单一<br>Graph sum: 图表总数 |
| **Function** | 当一个Item存在多个值时，决定显示哪个数据。<br>All: 全部（最小值、平均值和最大值）<br>Min: 最小值<br>Avg: 平均值<br>Max: 最大值 |
| **Draw style** | 仅对常规图表适用。<br>Line: 线条<br>Filled region: 填充区域<br>Bold line: 粗线<br>Dot: 圆点<br>Dashed line: 虚线 |
| **Y axis side** | Y轴的一侧分给哪一个元素。 |
| **Colour** | 使用十六进制RGB颜色代码。 |
