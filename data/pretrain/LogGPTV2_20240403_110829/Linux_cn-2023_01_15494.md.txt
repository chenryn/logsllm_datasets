### 甲骨文新的授权模式导致 Java 使用成本激增

甲骨文公司最近对其 Oracle Java SE 订阅模式进行了调整。在新模式下，Java 将按每个员工进行授权，而非旧模式中的按用户或处理器授权。尽管甲骨文承诺允许用户根据现有订阅模式续约，但随着时间推移，这种转变可能会迫使用户采用新方案。这将对那些长期努力减少 Java 使用量的公司造成不利影响，因为无论实际使用情况如何，他们都将按员工数量支付费用。据分析，某些公司的 Java 成本因此上升了 10 倍，平均增长 2 至 4 倍。

**消息来源：[The Register](https://www.theregister.com/2023/01/27/oracle_java_licensing_change/)**

**老王点评：是时候考虑放弃甲骨文的 Java 了。**

### GitHub 用户数突破 1 亿

成立于 2008 年的 GitHub 宣布其活跃开发者人数已达到 1 亿。与十年前的 300 万用户、五年前微软以 75 亿美元收购时声称的 2800 万用户，以及三个月前透露的 9000 多万用户相比，这一数字显著增加。在 2019 年的一次主题演讲中，GitHub 的前 CEO 表示，公司目标是在 2025 年实现 1 亿用户的里程碑。实际上，这一目标提前两年达成。

**消息来源：[Tech Crunch](https://techcrunch.com/2023/01/26/github-says-it-now-has-100m-active-users/)**

**老王点评：若非部分地区访问 GitHub 经常不稳定，这一目标或许会更快实现。**

### ChatGPT 在寻找和解决程序错误方面表现优异

研究人员测试了四个不同的代码修复系统（ChatGPT、Codex、CoCoNut 和标准 APR）对 40 段有错误的代码的处理能力。在第一轮测试中，ChatGPT 解决了 19 个问题，Codex 解决了 21 个，CoCoNut 解决了 19 个，而标准 APR 方法则解决了 7 个。通过进一步与 ChatGPT 对话并提供更多关于问题的信息后，最终 ChatGPT 解决了 31 个问题。

**消息来源：[《PC 杂志》](https://www.pcmag.com/news/watch-out-software-engineers-chatgpt-is-now-finding-fixing-bugs-in-code)**

**老王点评：ChatGPT 不仅能编写代码，还能查找和修复错误。程序员们可能真的需要重新评估自己的职业前景了。**
