### LEEF 2.0 格式

**格式定义：**
```
LEEF:2.0|供应商|产品|版本|事件 ID|[分隔符字符（可选，Tab 分隔）]|扩展
```

**日志条目示例：**
```
LEEF:2.0|AsiaInfo Security|Deep Security Manager|9.6.2007|4000010|cat=Anti-Malware name=SPYWARE_KEYL_ACTIVE desc=SPYWARE_KEYL_ACTIVE sev=6 cn1=45 cn1Label=Host ID dvchost=laptop_mneil TrendMicroDsTenant=Primary TrendMicroDsTenantId=0 cs3=C:\\Windows\\System32\\certreq.exe cs3Label=Infected Resource cs4=10 cs4Label=Resource Type cs5=50 cs5Label=Risk Level act=Clean msg=Realtime
```

**扩展字段说明：**

- **主机标识符 (cn1, cn1Label)**
  - `cn1`: 客户端计算机的内部标识符。
  - `cn1Label`: 主机 ID 的友好名称标签。

- **文件大小 (cn2, cn2Label)**
  - `cn2`: 隔离文件的大小。
  - `cn2Label`: 文件大小的友好名称标签。

- **文件路径 (filePath)**
  - `filePath`: 目标文件的位置。

- **操作 (act)**
  - `act`: 防恶意软件引擎执行的操作。可能的值包括：拒绝访问、隔离、删除、不处理、清除和未指定。

- **消息 (msg)**
  - `msg`: 扫描类型。可能的值包括：实时 (Realtime)、计划 (Scheduled) 和手动 (Manual)。

- **设备地址 (dvc, dvchost)**
  - `dvc`: 如果地址是 IPv4 地址，则使用 `dvc`；否则使用 `dvchost`。

- **事件标记 (TrendMicroDsTags)**
  - `TrendMicroDsTags`: 事件标记。

- **租户名称 (TrendMicroDsTenant, TrendMicroDsTenantId)**
  - `TrendMicroDsTenant`: 租户名称。
  - `TrendMicroDsTenantId`: 租户 ID。

- **严重性 (sev)**
  - `sev`: 事件的严重性。1 代表最低的严重性，10 代表最高的严重性。

- **类别 (cat)**
  - `cat`: 事件类别，例如“防恶意软件”。

- **名称 (name)**
  - `name`: 事件名称。

- **描述 (desc)**
  - `desc`: 事件描述。对于防恶意软件事件，通常没有单独的描述，而是使用事件名称。

### CEF 格式

**基本格式：**
```
CEF:版本|设备供应商|设备产品|设备版本|签名 ID|名称|严重性|扩展
```

**日志条目示例：**
```
CEF:0|AsiaInfo Security|Deep Security Agent||5000000|WebReputation|5|cn1=1 cn1Label=Host ID dvchost=hostname request=site.com msg=Blocked By Admin
```

**扩展字段说明：**

- **主机标识符 (cn1, cn1Label)**
  - `cn1`: 客户端计算机的内部标识符。
  - `cn1Label`: 主机 ID 的友好名称标签。

- **请求 (request)**
  - `request`: 请求的 URL。

- **消息 (msg)**
  - `msg`: 操作类型。可能的值包括：实时 (Realtime)、计划 (Scheduled) 和手动 (Manual)。

- **设备地址 (dvc, dvchost)**
  - `dvc`: 如果地址是 IPv4 地址，则使用 `dvc`；否则使用 `dvchost`。

- **事件标记 (TrendMicroDsTags)**
  - `TrendMicroDsTags`: 事件标记。

- **租户名称 (TrendMicroDsTenant, TrendMicroDsTenantId)**
  - `TrendMicroDsTenant`: 租户名称。
  - `TrendMicroDsTenantId`: 租户 ID。

- **严重性 (sev)**
  - `sev`: 事件的严重性。1 代表最低的严重性，10 代表最高的严重性。

- **类别 (cat)**
  - `cat`: 事件类别，例如“Web 信誉”。

- **名称 (name)**
  - `name`: 事件名称。

- **描述 (desc)**
  - `desc`: 事件描述。对于 Web 信誉事件，通常没有单独的描述，而是使用事件名称。

### 中继组

中继是亚信安全服务器深度安全防护系统客户端内的一个模块，负责安全更新和软件更新的下载和发布。管理中心指示中继获取最新的更新；当新更新可用时，会自动指示客户端和设备从中继提取其更新。中继模块仅可用于 64 位 Windows 和 Linux 客户端，默认情况下是关闭的。

#### 启用中继模块

要启用客户端中的中继模块，请打开运行激活的 64 位 Windows 或 Linux 客户端的计算机编辑器窗口，转至概述 >> 操作 >> 软件，然后单击启用中继。

#### 中继组配置

中继组织成中继组。新启用的中继被分配给默认中继组。除非另外配置，否则客户端/设备从默认中继组检索更新。亚信安全建议特定地理区域或办公室内计算机上的客户端配置为从相同区域中的中继组下载更新。

- **中继组成员**
  - 中继组可以包含一个或多个成员中继，以提高性能和冗余。
  - 组中的中继没有优先级，客户端/设备将随机选择连接到的成员中继。
  - 如果初始中继无法响应，客户端/设备将从组中随机选择其他成员中继进行更新。

- **中继组层次结构**
  - 中继组可以在层次结构中排列，以优化带宽并提供更多冗余。
  - 环境中必须始终至少有一个中继组可以从亚信安全更新服务器下载安全更新。
  - 如果无法与已分配的中继组联系，客户端/设备将切换到父中继组。

- **中继更新源**
  - 中继始终从在中继组层次结构中高一级的组或者从亚信安全更新服务器中检索更新。
  - 可以指定如果已启用中继的客户端不可访问，客户端/设备是否可以从主安全更新源下载特征码更新。

#### 创建中继组

1. 安装并激活中继之后，进入管理 >> 更新 >> 中继组。
2. 单击新建，并使用中继组向导创建并命名您的中继组，选择属于该组成员的中继。
3. 对于主中继组，在从以下位置下载更新部分，选择主安全更新源。此设置将从管理 >> 系统设置 >> 更新选项卡上的中继部分中配置的更新源 URL 中下载更新。
4. 重复执行步骤 2 以创建更多中继组。要创建层次结构，请在从以下位置下载更新中选择要使新中继组成为现有中继组的源。

#### 将客户端/设备分配给中继组

1. 在计算机页面中，右键单击选定计算机，然后选择处理措施 >> 分配中继组。从下拉列表中选择要使用的中继组，或从计算机详细信息窗口中使用从以下位置下载更新：以选择中继组。
2. 要分配多台计算机，请在计算机页面中按住 Shift 或 Ctrl 单击列表中的选定计算机。选择处理措施 >> 分配中继组。从下拉列表中选择希望所有选定计算机使用的中继组。
3. 要查看所有中继组分配，请在管理 >> 系统设置 >> 更新中单击查看中继组...按钮。对于列表中的每个中继组，单击鼠标右键并选择属性。转到已分配给选项卡以查看已分配给该中继组的客户端/设备列表。（为了快速更改客户端/设备的分配，单击已分配给列表上的链接将打开该客户端/设备的计算机详细信息页面，您可以从中选择其他中继组分配）。

#### 修改中继组

修改中继组后，已向其（包括子中继组）分配的计算机上将自动更新配置。还可以创建一个基于事件的任务，该任务会在将计算机添加到管理中心的计算机页面后自动向其分配中继组。有关更多信息，请参阅基于事件的任务。

#### 仅更新防恶意软件特征码

在某些情况下，您可能希望仅应用防恶意软件特征码更新，而将防恶意软件引擎更新排除在外。为此，请：

1. 转至管理 >> 更新 >> 中继组。
2. 双击某个中继组以打开其属性窗口。
3. 在中继组属性选项卡的更新区域中，选择仅更新特征码。单击确定。

#### 启动安全更新

- **全系统更新**：转至管理 >> 更新 >> 安全，然后单击检查更新并下载...按钮。
- **特定客户端/设备更新**：从计算机页面的计算机列表中选择客户端/设备，然后右键单击并选择操作 >> 下载安全更新。
- **预设常规检查安全更新任务**：转至管理 >> 预设任务，然后创建检查安全更新类型的新预设任务。

#### 更新源

中继组层次结构顶部的中继组内的中继连接至主安全更新源。您可以在亚信安全服务器深度安全防护系统管理中心的管理 >> 系统设置 >> 更新选项卡中配置主更新源。

#### 代理服务器

每个中继组（默认中继组除外）可以配置为使用单独的代理服务器连接至亚信安全以检索安全更新。默认中继组与亚信安全服务器深度安全防护系统使用相同的代理服务器连接至 Internet。

**配置代理服务器：**

1. 在亚信安全服务器深度安全防护系统管理中心，转至管理 >> 更新 >> 中继组，然后双击一个中继组以显示其属性窗口。
2. 在代理服务器选项卡上，从主安全更新代理服务器下拉列表中选择代理服务器。
3. 单击确定。

注意：在管理 >> 系统设置 >> 代理服务器选项卡上维护可用代理服务器的列表。

### 安全更新

亚信安全服务器深度安全防护系统需要定期更新，以便包含最新的安全更新和软件更新。以安全更新形式从亚信安全检索更新包。已启用中继的客户端以中继组的形式进行组织（也由亚信安全服务器深度安全防护系统管理中心管理和配置），用于从亚信安全检索安全更新并将它们分发到客户端和设备。

#### 配置安全更新

在配置安全更新前，您必须已安装并激活客户端和设备（以及将用于分发更新的已启用中继的客户端）。《亚信安全服务器深度安全防护系统安装指南》中提供了所有亚信安全服务器深度安全防护系统软件的安装说明。

要配置安全更新，您将需要：

1. **配置安全更新源**
   - 转至管理 >> 系统设置 >> 更新：
     - 设置您的更新源。默认情况下，更新源是通过 Internet 访问的亚信安全更新服务器。除非您的支持提供商已告知您进行更改，否则请保留原设置。
     - 如果特征码更新已经从亚信安全下载并可使用一小时以上但电脑仍未更新时，会引发警报。
     - 在安全更新区域中，可以选择在中继不可用时允许客户端直接从主安全更新源下载特征码更新，或者在亚信安全服务器深度安全防护系统管理中心无法访问时允许客户端下载特征码更新。

2. **将已启用中继的客户端组织成中继组**
   - 参见“创建中继组”部分。

3. **将中继组分配给客户端/设备**
   - 参见“将客户端/设备分配给中继组”部分。

4. **特例：在气隙环境中已启用中继的客户端上配置更新**
   - 在气隙环境中，可以通过手动方式将更新文件传输到中继客户端，并从中继客户端分发到其他客户端和设备。

#### 自动将规则更新应用到策略

亚信安全有时会发布对现有亚信安全服务器深度安全防护系统规则的更新。此设置决定是否在安全更新期间将已更新的规则发送到计算机。更新现有规则可以改进规则的效率，或者修复缺陷。尽管最好在将新规则部署到生产环境之前先对其进行测试（仅检测模式下或在测试环境中），但自动将更新应用到现有规则是一个安全选择。
