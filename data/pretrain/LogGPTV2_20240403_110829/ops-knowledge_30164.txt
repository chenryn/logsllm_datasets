User: How to configure Nginx reverse proxy and add certificates for HTTPS access?
Assistant: The steps are as follows: 1. Install Nginx You can use the system package manager to install, such as apt, yum, etc., or you can download the source code from the official website to compile and install. 2. Configure Nginx reverse proxy Open the /etc/nginx/nginx.conf file, find the server section in the http section, and add the following configuration: ``` server { listen 80; server_name example.com; location / { proxy_pass http:/ /backend_server:8080/; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; } } ``` Among them, listen specifies the listening port; server_name specifies the service name; location / specifies the address of the reverse proxy, followed by real The address and port of the backend server. proxy_set_header is used to set the request header, where Host and X-Real-IP are set. 3. Add SSL certificate Use Let's Encrypt free certificate, execute the following command: ``` sudo apt update sudo apt install certbot python3-certbot-nginx sudo certbot --nginx -d example.com ``` Among them, example.com is required Add the domain name of the certificate. After executing the above command, certbot automatically configures the SSL certificate for nginx and automatically configures the SSL settings of nginx. 4. Restart Nginx Execute the following command to restart Nginx: ``` sudo systemctl restart nginx ``` After completing the above steps, you can access the Nginx reverse proxy server through HTTPS.
