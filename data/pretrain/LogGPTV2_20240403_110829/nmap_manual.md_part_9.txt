### 文件无效，无法正常解析，需要手工修改。

#### \--resume (继续中断的扫描)
一些扩展的Nmap运行可能需要很长时间，甚至几天。这类扫描经常会被中断。为了避免在工作时间运行Nmap导致网络中断、主机重启或因其他原因取消运行（如按下`Ctrl-C`），可以使用`--resume`选项恢复中断的扫描。如果标准扫描（-oN）或Grep扫描（-oG）的日志被保留，用户可以通过提供这些日志文件并使用`--resume`选项来恢复扫描。例如：
```
nmap --resume <logfile>
```
此方法不支持XML输出格式，因为将两次运行结果合并到一个XML文件中较为困难。

#### \--stylesheet (设置XSL样式表，转换XML输出)
Nmap提供了XSL样式表`nmap.xsl`，用于查看或转换XML输出为HTML。XML输出文件包含了一个指向`nmap.xsl`文件的`xml-stylesheet`指令。默认情况下，Nmap会在安装目录或当前工作目录中查找该文件。若希望使用不同的样式表，可将其路径或URL作为`--stylesheet`参数传递给Nmap。例如：
```
nmap --stylesheet http://www.insecure.org/nmap/data/nmap.xsl
```
这告诉浏览器从Insecure.Org加载最新的样式表，从而方便在未安装Nmap的机器上查看结果。

#### \--no-stylesheet (忽略XML声明的XSL样式表)
使用该选项可以禁止Nmap的XML输出关联任何XSL样式表，使`xml-stylesheet`指令被忽略。

## 其它选项

本节描述了一些重要的和其他不太重要的选项，这些选项不适合归类于其他类别。

- **-6 (启用IPv6扫描)**
  自2002年起，Nmap开始支持IPv6的一些主要功能，包括ping扫描（仅TCP）、连接扫描和版本检测。只需添加`-6`选项即可启用IPv6扫描，并使用IPv6地址替换主机名。IPv6尚未在全球广泛采用，但在某些国家（如亚洲）及一些高级操作系统中已得到支持。进行IPv6扫描时，源和目标均需配置IPv6。若ISP不分配IPv6地址，可使用免费的隧道代理服务，如BT Exact或Hurricane Electric。

- **-A (激烈扫描模式选项)**
  此选项启用额外的高级和高强度扫描功能，目前包括操作系统检测（-O）和版本扫描（-sV）。目的是提供一个全面的扫描选项集合，而无需用户记住大量参数。该选项不会影响与时间相关的选项（如-T4）或细节选项（-v）。

- **\--datadir (指定Nmap数据文件位置)**
  Nmap在运行时会从特定文件中获取数据，如`nmap-service-probes`等。通过`--datadir`选项，用户可以指定这些文件的位置。Nmap将依次在指定目录、环境变量`BMAPDIR`、用户主目录下的`.nmap`文件夹、Nmap可执行文件所在目录以及编译时的默认目录中查找所需文件。

- **\--send-eth (使用原始以太网帧发送)**
  要求Nmap在以太网层而不是IP层发送报文。默认情况下，Nmap会选择最适合其运行平台的方式。在UNIX系统中，如果没有其它选择（如无以太网连接），无论是否指定了该选项，Nmap都会使用原始IP包。

- **\--send-ip (在原始IP层发送)**
  与`--send-eth`相反，要求Nmap通过原始IP套接字发送报文。

- **\--privileged (假定用户具有全部权限)**
  告诉Nmap假定用户拥有足够的权限进行原始套接字包发送、报文捕获等操作。在Linux等允许非特权用户进行原始报文扫描的系统中，此选项特别有用。

- **\--interactive (启动交互模式)**
  在交互模式下启动Nmap，提供一个交互式的提示符，便于同时进行多个扫描。这对于多用户系统中的安全测试非常有用。输入`h`可获得帮助信息。由于需要对正确的shell程序和整个功能非常熟悉，这个选项较少使用。

- **-V; \--version (打印版本信息)**
  打印Nmap的版本号并退出。

- **-h; \--help (打印帮助摘要)**
  打印一个简短的帮助屏幕，列出常用命令选项。这与不带参数运行Nmap的效果相同。

## 实例

以下是一些从简单到复杂的Nmap扫描示例。为了具体起见，某些例子使用了实际的IP地址和域名。请确保在进行扫描前获得必要的许可。

- **基本扫描**
  ```
  nmap -v scanme.nmap.org
  ```
  扫描`scanme.nmap.org`的所有保留TCP端口，并启用详细模式。

- **SYN扫描和操作系统检测**
  ```
  nmap -sS -O scanme.nmap.org/24
  ```
  对`scanme.nmap.org`所在的C类网段进行秘密SYN扫描，并尝试确定每台主机的操作系统类型。

- **指定端口扫描**
  ```
  nmap -sV -p 22,53,110,143,4564 198.116.0-255.1-127
  ```
  对B类网段`198.116`中的255个子网进行主机枚举和TCP扫描，检查特定端口上的服务版本。

- **随机主机扫描**
  ```
  nmap -v -iR 100000 -P0 -p 80
  ```
  随机选择100,000台主机，扫描其Web服务器（80端口），禁用主机发现。

- **保存结果**
  ```
  nmap -P0 -p80 -oX logs/pb-port80scan.xml -oG logs/pb-port80scan.gnmap 216.163.128.20/20
  ```
  扫描4096个IP地址，查找Web服务器（不ping），并将结果以Grep和XML格式保存。

- **DNS区域传输**
  ```
  host -l company.com | cut -d ' ' -f 4 | nmap -v -iL -
  ```
  进行DNS区域传输，以发现`company.com`中的主机，并将IP地址提供给Nmap进行扫描。适用于GNU/Linux系统。
