### 问题描述
根据 RFC 2616 的规定：
- 当发送方未明确指定字符集时，"text" 类型的媒体子类型默认使用 "ISO-8859-1" 字符集。

然而，RFC 7231 已经移除了 "text" 类型媒体子类型的默认字符集 "ISO-8859-1"。

### 影响
如果 `requests` 库继续使用 "ISO-8859-1" 作为默认字符集，将会导致以下问题：
- 响应的 `text` 属性将无法充分利用 `chardet` 检测到的字符集。
- 可能会出现乱码问题。

### 建议
建议删除或修改相关代码，以避免上述问题。具体来说，可以考虑删除以下代码行：

```python
if 'text' in content_type:
```

这样可以确保 `requests` 库能够正确处理字符集检测，从而避免乱码问题。
