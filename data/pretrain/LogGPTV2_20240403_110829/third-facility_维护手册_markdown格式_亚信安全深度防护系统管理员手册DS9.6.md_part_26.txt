### 高级日志记录策略模式
为了减少记录的事件数量，可以将亚信安全服务器深度安全防护系统管理中心配置为在以下几种高级日志记录策略模式之一下运行。这些模式可以在设置 >> 网络引擎 >> 高级网络引擎设置区域中的策略和计算机编辑器中进行配置。

#### 模式与忽略事件类型
以下是四种较复杂的“高级日志记录策略”模式及其忽略的事件类型：

1. **状态和规范化抑制**
   - 连接断开
   - 标志无效
   - 序列无效
   - ACK 无效
   - 未经请求的 UDP
   - 未经请求的 ICMP
   - 不允许的策略
   - 丢弃的重新传送量

2. **状态、规范化和片段抑制**
   - 上述所有事件
   - CE 标志
   - IP 无效
   - IP 数据报长度无效
   - 片段化
   - 片段偏移量无效
   - 第一个片段太小
   - 片段超出界限
   - 片段偏移量太小
   - IPv6 数据包
   - 最大传入连接数
   - 最大传出连接数
   - 已发出的最大 SYN 数
   - 使用授权已过期
   - IP 版本未知
   - 数据包信息无效
   - 最大 ACK 重新传送量
   - 已关闭的连接上的数据包

3. **状态、片段和验证程序抑制**
   - 上述所有事件
   - 数据偏移量无效
   - 无 IP 标头
   - 不可读的以太网标头
   - 未定义
   - 源 IP 和目标 IP 相同
   - TCP 标头长度无效
   - 不可读的协议标头
   - 不可读的 IPv4 标头
   - 未知 IP 版本

4. **分接模式**
   - 上述部分事件
   - ACK 无效
   - 最大 ACK 重新传送量
   - 已关闭的连接上的数据包
   - 丢弃的重新传送量

### 命令行工具
#### 亚信安全服务器深度安全防护系统客户端 `dsa_control`
`dsa_control` 是一个命令行工具，用于管理亚信安全服务器深度安全防护系统的客户端。以下是该工具的一些常用选项及其说明：

- `-a <URL>`：激活客户端。URL 必须是 "dsm://hostOrIp:port/" 格式，其中 port 是管理中心的波动信号端口（默认端口为 4120）。
- `-b`：创建更新包。
- `-c <cert_file>`：标识证书文件。
- `-d`：生成客户端诊断数据包。
- `-g <agent_url>`：客户端 URL，默认为 "https://localhost:4118/"。
- `-m`：要求客户端立即联系管理中心。
- `-p <password>`：认证密码。使用 "*" 提示输入密码。
- `-r`：重置客户端配置。
- `-R <file_path>`：恢复隔离文件。
- `-s <self_protect>`：启用或禁用客户端自我防护（仅限 Windows）。(1: 启用，0: 禁用)。
- `-t <retries>`：指定 dsa_control 在无法与客户端服务联系时的重试次数。各重试之间暂停一秒。
- `--buildBaseline`：生成完整性监控基线。
- `--scanForChanges`：扫描以发现完整性监控的更改。

#### 客户端启动的激活
- **手动激活**：通过右键单击“计算机”窗口中的计算机并选择操作 >> 激活/重新激活，可以从亚信安全服务器深度安全防护系统管理中心手动激活客户端。
- **命令行激活**：使用 `dsa_control -a` 命令启动激活过程。例如：
  ```sh
  dsa_control -a dsm://[managerurl]:[port]/
  ```
  其中：
  - `--a` 是激活客户端的命令。
  - `dsm://[managerurl]:[port]/` 是指向管理中心的参数。（"managerurl" 是管理中心的 URL，"port" 是客户端与管理中心之间的通信端口，默认为 4120。)

#### 通过代理服务器激活
- **设置代理服务器**：
  - 在亚信安全服务器深度安全防护系统管理中心，转至管理 > 系统设置 > 客户端页面。
  - 选择允许客户端启动的激活。
  - 选择允许客户端指定主机名。
  - 在如果同名计算机已存在列表中，选择“激活具有相同名称的新计算机”。
  - 单击保存。

- **命令行选项**：
  - `dsa_control -x "dsm_proxy://proxy_ip:port/"`：设置代理服务器地址。
  - `dsa_control -u "username:password"`：设置代理服务器用户名和密码。
  - 示例：
    ```sh
    dsa_control -x "dsm_proxy://172.21.3.184:808/"
    dsa_control -u "root:Passw0rd!"
    dsa_control -a "dsm://seg-dsm-1:4120/"
    ```

#### 客户端启动的波动信号命令
- **命令**：`dsa_control -m` 将指示客户端立即对亚信安全服务器深度安全防护系统管理中心执行波动信号操作。此命令可以传递更多参数给管理中心。
