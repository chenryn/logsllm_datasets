![The impact GitHub is having on your software career](/data/attachment/album/201707/08/125709cd7n5xt4jxpuf7uk.jpg "The impact GitHub is having on your software career")

在未来的 12 到 24 个月内（即 2018 年或 2019 年），雇佣软件开发者的方式将发生根本性变化。

2004 至 2014 年间，我曾在红帽公司工作，这是全球最大的开源软件公司之一。记得 2004 年 7 月的一天，我的老板 Marty Messer 对我说：“你在红帽的所有工作都会被开源。未来，你不再需要简历，因为所有人都可以通过 Google 找到你的信息。”

在红帽工作的一个独特优势是，我们有机会建立个人品牌并树立声誉。通过邮件列表、Bug 跟踪器与其他工程师交流，并且提交到 Mercurial、Subversion 和 CVS 仓库的代码都是开源的，可以被搜索到。

如今，我们生活在一个充满开源的世界中。以下两点可以帮助你更好地理解这个时代：

1. 微软曾长期坚持闭源甚至排斥开源，但现在已开始积极拥抱开源。他们成立了 .NET 基金会（红帽也是成员之一），加入了 Linux 基金会，并将 .NET 项目作为开源项目进行开发。
2. GitHub 已成为一个独特的社交网络，结合了问题追踪和分布式版本控制系统。

对于那些从闭源环境走过来的开发者来说，他们可能还不了解开源的意义。对他们而言，开源意味着“无偿公开业余时间的工作成果”。

对于我们这些在过去十年里创建了一家价值数十亿美元的开源软件公司的人来说，参与开源意味着没有空闲时间。但开源带来的好处也很明显：所获得的声誉属于个人，而不只是公司的一部分。GitHub 是一个社交网络，在这里你可以创建提交、在自己擅长的领域为全球组织做出贡献，而这些工作并不依附于任何特定公司。

聪明的人会利用这种工作环境。他们会为平时工作中使用的语言和框架（如 TypeScript、.NET 和 Redux）贡献补丁、工单和评论。他们也积极拥抱开源，尽可能多地开源自己的创新成果，甚至向私有仓库提交贡献。

GitHub 推动了平等。例如，你可能很难在澳大利亚找到一份来自印度的工作，但在 GitHub 上，地理限制不再是障碍，你可以与世界各地的同事合作。

过去十年里，想加入红帽的方法很简单：只需在某个小方面与红帽的工程师在开源项目上合作，当他们认可你的贡献并认为你是良好的合作伙伴时，就有可能获得工作机会。

现在，开源为所有人提供了平等的机会。随着开源在全球范围内的普及，这种情况将在更多地方出现。

在最近的一次访谈中，Linux 和 Git 的创始人 Linus Torvalds（在 GitHub 上有 49K 粉丝，关注 0 人）表示：

> “当你提交了许多小补丁后，项目的维护者开始信任你。那一刻，你不仅是一个贡献者，而是成为了一个受信任的团队成员。”

实际上，你的名声存在于这个信任网络中。我们知道，当你离开一家公司时，人脉和声誉可能会减弱甚至消失。就像在一个小村庄里生活多年后，全村人都认识你；但当你搬到新地方时，可能没有人听说过你，更糟糕的是，没有人认识任何知道你的人。

你失去了直接和间接的人际联系。除非通过会议或其他大型活动来建立个人品牌，否则你在公司内部建立的信任最终会消逝。但在 GitHub 上完成的工作仍然可见，信任网络依然存在。

首先受益的将是弱势群体，如学生、新毕业生和移民，他们可以利用这个平台“前往”澳大利亚等地。

这将改变现状。以前一些开发者可能经历过人际网络突然中断的情况，而开源的原则是精英制——最好的创意、最多的提交、最多的测试和最快的执行等。

虽然它并不完美，也没有什么事物是完美的。无法与他人合作也会大大影响人们对你的看法。红帽公司会解雇那些不能与团队和谐相处的人，而在 GitHub 上工作的员工主要与其他贡献者互动。

GitHub 不仅仅是一个代码仓库或原始提交者的列表。它是一个社交网络。我认为：

> GitHub 上有多少代码并不重要，重要的是有多少关于你代码的讨论。

在未来 12 到 24 个月内，使用 GitHub 的开发者与不使用的开发者之间的差距将会变得非常明显。就像有电子邮件和没有电子邮件的区别（现在每个人都有电子邮件），或者有手机和没有手机的区别（现在每个人都有手机）。最终，绝大多数人都会参与开源，这将成为竞争中的差异化优势。

然而，目前开发者的职业生涯已经被 GitHub 打乱了。
