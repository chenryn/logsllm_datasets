# []{#kernel_parameters_changes.html#kernel_parameters_changes}第 4 章 对外部内核参数的重要更改 {.title}
:::
本章为系统管理员提供了与 Red Hat Enterprise Linux 7.9
一起发布的内核有显著变化的总结。这些更改包括添加或更新的
`proc`{.literal} 条目、`sys`{.literal} ctl 和 `sysfs`{.literal}
默认值、引导参数、内核配置选项或任何可见的行为更改。
::: section
::: titlepage
# []{#kernel_parameters_changes.html#new_kernel_parameters}新内核参数 {.title}
:::
::: variablelist
[bert_disable \[ACPI\]]{.term}
:   这个参数在有问题的 BIOS 上禁用 Boot Error Record Table（BERT）。
    BERT 是 ACPI
    平台错误接口表之一，用于获取上一次启动中发生的硬件错误日志，固件不会在运行时通知内核错误，例如通过不可屏蔽中断(NMI)或机器检查异常(MCE)通知内核。
[bert_enable \[ACPI\]]{.term}
:   仅限 RHEL7。这个参数启用 Boot Error Record
    Table（BERT）。默认状态被禁用。
[page_owner = \[KNL\]]{.term}
:   在默认情况下，存储每个页由谁分配的信息被禁用。这个参数允许使用以下选项存储这些信息：
    ::: itemizedlist
    -   `on`{.literal} - 启用该功能
    :::
[srbds = \[X86,INTEL\]]{.term}
:   这个参数控制特殊寄存器缓冲区数据采样(SRBDS)缓解。
    某些 CPU 可能会受到类似 MDS（Microarchitectural Data
    Sampling）的漏洞影响。这类漏洞可能会从随机数生成器中泄漏信息。
    默认情况下，使用 microcode 可以缓解此问题。但是，microcode
    修复可能会导致 `RDRAND`{.literal} （读取随机）和 `RDSEED`{.literal}
    指令变得非常慢。除了其他影响外，这将减少 `/dev/urandom`{.literal}
