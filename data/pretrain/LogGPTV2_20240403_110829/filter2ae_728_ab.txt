### 优化后的文本

#### SetImeInfoEx 漏洞
SetImeInfoEx 函数未对目标窗口站 `tagWINDOWSTATION` 的指针成员域 `spklList` 进行有效性校验，直接进行读取访问。此漏洞已在2018年5月被修复。

#### Windows 10 0day 漏洞 (CVE-2018-8414)
2018年6月，Windows 10 被曝出一个远程代码执行漏洞（CVE-2018-8414）。该漏洞源于 Windows Shell 在某些情况下未能正确验证文件路径，通过精心构造的恶意脚本可以触发该漏洞，从而实现任意读写操作。该漏洞仅影响 Windows 10 的新文件类型 `.SettingContent-ms`，并于2018年8月14日分配 CVE 编号并发布补丁修复。

#### Internet Explorer “双杀”二代漏洞 (CVE-2018-8242)
2018年7月，Internet Explorer 再次曝光一个名为“双杀”二代的 0day 漏洞（CVE-2018-8242）。该漏洞是由于4月发布的“双杀”一代（CVE-2018-8174）补丁未能完全解决问题，导致 VBScript 脚本引擎中仍存在类似缺陷。此漏洞由360Vulcan团队发现并提交，并于2018年7月10日修复。

#### Exchange Server 内存损坏漏洞 (CVE-2018-8302)
2018年8月，Exchange Server 被公开了一个内存损坏漏洞（CVE-2018-8302）。攻击者可以通过钓鱼邮件触发该漏洞，进而对企业用户的计算机发起攻击，并最终接管 Exchange Server 服务器。该漏洞存在于语音邮件的接收存储过程中，具体是在反序列化 `TopNWords.Data` 时发生的。

#### Internet Explorer “双杀”三代漏洞 (CVE-2018-8373)
同样在8月，Trendmicro 报告了另一个名为“双杀”三代的 0day 漏洞（CVE-2018-8373），它与“双杀”一代基于相似原理，通过 VBScript.dll 中的缺陷获取任意读取权限。这两个漏洞均于2018年8月14日修复。

#### ALPC 提权漏洞 (CVE-2018-8440)
2018年9月，Windows 被曝出一个 ALPC 提权漏洞（CVE-2018-8440）。该漏洞利用高级本地过程调用 (ALPC) 函数 `SchRpcSetSecurity` 无法正确检查用户权限的缺陷，实现本地权限提升 (LPE) 并执行恶意代码。

#### Microsoft Jet Database Engine RCE 漏洞 (CVE-2018-8423)
同月，Microsoft Jet Database Engine 被公开了一个远程代码执行漏洞（CVE-2018-8423）。该漏洞是一种越界 (OOB) 写入漏洞，攻击者可通过诱导用户打开特制的 JET 数据库文件来触发漏洞。上述两个漏洞分别于2018年9月11日和10月9日修复。

#### Microsoft Edge RCE 漏洞 (CVE-2018-8495)
2018年10月，Microsoft Edge 被公开了一个关于 Windows Shell 的高危 RCE 漏洞（CVE-2018-8495）。攻击者可以构造包含特殊 URI 的网页，诱导用户打开后即可在远程计算机上运行恶意代码。该漏洞是由于 Windows Shell 处理 URI 时未能过滤特定的 URI 所致。

#### Win32k 提权漏洞 (CVE-2018-8453)
同月，Windows 被曝出一个复杂的 Win32k 提权漏洞（CVE-2018-8453）。该漏洞利用 `win32k.sys` 组件中的 `win32kfull!xxxDestroyWindow` 函数中的 UAF 漏洞获取本地提权。这两例漏洞均于2018年10月9日修复。

#### Win32k 提权漏洞 (CVE-2018-8589)
2018年11月，Windows 再次被曝出一个 Win32k 提权漏洞（CVE-2018-8589）。该漏洞源于 `win32k!xxxMoveWindow` 函数中存在的不恰当竞争条件，可能导致线程间的信息传递不当锁定。该漏洞已于2018年11月13日修复。

#### Microsoft DNS Server 堆溢出漏洞 (CVE-2018-8626)
2018年12月，Microsoft DNS Server 被曝光存在一个高危堆溢出漏洞（CVE-2018-8626）。所有设置为 DNS 服务器的 Windows 服务器都会受到影响。攻击者可通过发送精心构造的请求触发堆溢出并执行远程代码。该漏洞于2018年12月11日发布补丁修复。

#### Kernel 内核事务管理器驱动程序提权漏洞 (CVE-2018-8611)
同月，Windows 被曝出一个高危内核事务管理器驱动程序提权漏洞（CVE-2018-8611）。该漏洞源于 kernel 模式下对文件操作的不当处理，导致内核事务管理器产生竞争条件。此漏洞绕过了主流 Web 浏览器的进程缓解策略，使黑客能够构建完整的远程代码执行攻击链。该漏洞于2018年10月29日被发现，并于2018年12月11日分配 CVE 编号并公布修复补丁。

### 2018典型漏洞安全事件

#### 新一代幽灵 —— 英特尔 CPU 漏洞持续升级
继年初发现的 Meltdown 和 Spectre 漏洞后，2018年5月初 Google Project Zero 安全研究团队又曝出英特尔处理器存在8个新的“幽灵式”硬件漏洞，被称为“新一代幽灵”——Spectre-NG。这些漏洞可绕过云主机系统与虚拟机的隔离，实现虚拟机逃逸并窃取机密信息。此外，还可以攻击同一服务器上的其他虚拟机。

下半年，再次发现了英特尔 CPU 存在多个超线程漏洞，如 TLBleed、Foreshadow 和 PortSmash。其中，PortSmash 漏洞（CVE-2018-5407）影响所有支持超线程技术的 Intel 处理器。利用该漏洞，攻击者所在的进程可以窃取在同一物理内核上运行的另一进程的隐私数据。研究人员已成功从 OpenSSL 进程中窃取私钥。

一系列 CPU 漏洞的修复过程一波三折，仓促发布的补丁带来了新的风险并导致 CPU 性能下降。最终，英特尔不得不多次更新补丁，并加快了新一代处理器的发布进程，甚至考虑彻底放弃超线程技术。

#### Office 公式编辑器再曝新漏洞，商贸信钓鱼攻击屡试不爽
Office 公式编辑器再次曝出新漏洞，成为商贸信钓鱼攻击的常用手段。这类攻击通常通过诱骗用户打开含有恶意公式的文档来实施。
