### 配置软件更新服务器

亚信安全服务器深度安全防护系统的软件更新通常由已启用中继的客户端托管和分发。要在计算机上部署亚信安全服务器深度安全防护系统客户端，必须先将平台的软件包导入到管理中心。实际客户端安装包最初只在计算机上安装核心客户端功能。安全模块所需的插件（如入侵防御、防火墙等）仅在需要时才安装到客户端上。开启防护模块后，系统将通过中继将所需的插件部署到计算机上。这样做是为了降低客户端在受保护计算机上所占的内存。

如果您已部署了 Web 服务器，可能更希望让这些服务器执行软件更新分发任务，而不是部署中继来执行此任务。为此，您需要在 Web 服务器上反映亚信安全服务器深度安全防护系统中继的软件存储库。

**注意：** 尽管可以指示客户端从新更新的 Web 服务器获取其软件更新，您仍然需要至少一个中继来分发安全更新。

以下信息描述如何在本地 Web 服务器上设置自己的软件存储库。

#### Web 服务器要求
- **磁盘空间：** 8GB
- **端口：**
  - **44112222：** 用于客户端/设备与中继之间的通信 (TCP)
  - **44112233：** 用于内部中继与本地主机之间的通信 (TCP)

#### 文件夹结构

您必须在 Web 服务器上创建文件夹，以反映亚信安全服务器深度安全防护系统中继的软件存储库文件夹结构。

**注意：** 反映文件夹的步骤取决于您的 IT 环境，不在本文档的讨论范围之内。

- **Windows 中继上的默认位置：**
  ```
  C:\ProgramData\AsiaInfo Security\Deep Security Agent\relay\www\dsa\
  ```

- **Linux 中继上的默认位置：**
  ```
  /opt/ds_agent/relay/www/dsa/
  ```
