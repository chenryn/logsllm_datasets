尽管 Canonical 已不再维护 Unity 桌面，但 Ubuntu Unity 的开发者们接手了这项任务，并发布了一项主要更新（现已可用于测试）。

![Unity 7.6](/data/attachment/album/202205/03/173216whhgjgj3dgglush3.jpg)

需要注意的是，Canonical 并没有重新开始维护 Unity 桌面。得益于 Ubuntu Unity 发行版的开发者 Rudra Saraswat 的不懈努力，Unity 桌面环境在时隔六年后终于迎来了更新。

补充说明一下，[Ubuntu Unity](https://ubuntuunity.org/) 是一个采用 Unity 桌面（而非 GNOME）的社区项目。如果你希望在 Ubuntu 22.04 LTS 上使用 Unity 桌面，那么 [Ubuntu Unity](https://ubuntuunity.org/) 绝对是你的最佳选择。

最初，它仅提供经过微调的 Unity 体验，但现在 **Unity 7.6** 带来了许多界面优化和视觉变更。以下是关于此次更新的一些关键信息：

### Unity 7.6 更新内容

**提示**：Unity 7.6 是为公共测试而发布的版本，不建议作为替代其他桌面环境的正式使用环境。

此次更新不仅包括面向用户的改进，还涉及开发方面的优化，以方便贡献者参与 Unity7 的开发工作。具体改进如下：

#### 用户界面变更

![Unity 桌面](/data/attachment/album/202205/03/173216hnnqaul1nda5n15f.jpg)

Dash 启动器（应用启动器）与 HUD 现已重新设计，采用了更现代、简洁的视觉风格。整体设计更加扁平化，但仍保留了良好的系统全局模糊效果。此外，还进行了一些细微的视觉改进，如将停靠区上的“清空回收站”按钮改为使用 Nemo（而不是 Nautilus），并修复了 Dash 预览中的应用详情与评分显示问题。

#### 性能改进

![Unity 桌面](/data/attachment/album/202205/03/173217vqeqsuwjlk9wxa1w.png)

最新的更新中，Unity7 的内存使用量显著降低，Ubuntu Unity 22.04 的内存使用量也减少至约 700-800 MB。此外，低端显卡模式现在运行得更好，Dash 的响应速度也有所提升。

#### 其他变化

Unity7 Shell 的源代码已完全迁移至 [GitLab](https://gitlab.com/ubuntu-unity)。用于独立测试的 Unity7 启动器已被修复，部分有问题的测试项也被禁用，从而大幅缩短了构建时间。这些改进将有助于 Unity7 的贡献者更好地参与开发。

### 测试 Unity 7.6

你可以按照 [官方测试公告](https://unity.ubuntuunity.org/blog/unity-7.6/) 中提到的方法来编译并亲自尝试 Unity 7.6。你也可以访问其官方网站获取更多信息。如果你不想添加测试 PPA 源，可以等待 Ubuntu Unity 22.04 的正式更新。
