### 慕尼黑市项目负责人称“将整个城市迁移到 Linux 需要政策支持”

**摘要：**
慕尼黑市政府已将近 15,000 台 PC 从 Windows NT 操作系统迁移到其自有的基于 Ubuntu 的 Linux 发行版 LiMux。项目负责人 Peter Hofmann 强调了持续政策支持的重要性，并分享了迁移过程中的经验和挑战。

**正文：**

> 慕尼黑市政府已将近 15,000 台 PC 从 Windows NT 操作系统迁移到其自有的基于 Ubuntu 的 Linux 发行版 LiMux。

根据 IDG 新闻报道，慕尼黑市十年前开始向 Linux 迁移，今年终于见证了这项工程的竣工。“我们学到的最深一课是，你无法在没有持续政策支持的情况下完成这样的项目。”LiMux 项目的负责人 Peter Hofmann 在总结经验时说。

![](/data/attachment/album/201404/16/112504ih23zh45it88ufi3.png)

慕尼黑市政府将其网络上的 15,000 台 PC 中大约 14,800 台迁移到了其自有的基于 Ubuntu 的 Linux 发行版 LiMux 上。这超出了他们原定迁移 12,000 台桌面计算机的计划。

慕尼黑市是在 Microsoft 宣布将不再为 Windows NT 4.0 提供支持的时候决定迁移其 IT 系统的。当时，该市不得不在迁移到新版 Windows 和寻找一个替代平台之间做出抉择，因为新的软件以及现有软件的新版本都将无法在 Windows NT 上运行。市议会决定选择 Linux，以保持与软件供应商的独立关系。

Hofmann 表示，政策支持是迁移成功的关键。“我们从始至终都拥有政策的支持。我们定期向政府官员汇报项目进展，就像对待股东一样。”他说。

通过这种方式，这些政客们从未丧失对项目的兴趣，也始终能了解项目执行人的工作。“我目睹过很多开源项目因缺乏或失去政策支持而失败。”他说。

市政府原定于 2009 年完成迁移计划，但实际上从最初作出决定到 LiMux 项目完工耗费了约十年时间。项目全程经历了几次延期。

迁移工作在原计划一年之后才真正开始，这是第一次延期，Hofmann 说。第二次延期缘于 2007 年市议会的一项决定：慕尼黑市的 IT 部门也要负责 Linux 客户端基础设施所必需的标准化工作。然而，慕尼黑市既没有正确的程序也没有正确的组织来实现这种标准化。

项目第三次延期是在 2010 年，当时市议会决定扩大项目规模。风险管理、测试管理和需求工程作为三个附加任务目标被加进了项目里。

尽管困难重重，Hofmann 仍然表示愿意再次进行这样的项目。

慕尼黑市 IT 组织的异质性基础设施曾是该项目最大的问题之一。项目启动之初，22 个组织都有自己独特的配置、软硬件、进程、Windows 客户端知识以及相应的基础设施。“我们想建设一个标准化的、集中交付和开发的 Linux 客户端。”他说。

Hofmann 预计，四分五裂的基础设施会造成问题。事实证明，不论是技术因素还是组织因素，标准化客户端的工作都比他预想的还要艰难。

幸运的是，他有足够的自由度重建整个城市的 IT 基础设施。“任何人在打算迁移之前都要重新审视他们的整个 IT 组织。迁移到 Linux 远非节省开支和使用免费软件那么简单。”他补充道。

但慕尼黑市的迁移确实节省了大量成本。2012 年 11 月，一名市议员答复道，慕尼黑市计算出迁移到 LiMux 会比更新已有 Microsoft 软件[节省超过 1100 万欧元](http://www.itworld.com/operating-systems/321474/switching-linux-saves-munich-over-11-million)！

这项计算横向对比了 LiMux、Windows 7 + Microsoft Office 和 Windows 7 + OpenOffice（慕尼黑市为 LiMux 选择的生产力套件）三种方案，包含必要的硬件升级、培训、外部迁移支持和优化事务以及其他指标。两个 Windows 方案都明显比 LiMux 方案昂贵许多，主要原因是微软对软件使用许可收取的费用。

Hofmann 表示，LiMux 不存在支持合同。“你要支持合同干什么？实际上你并没有得到支持，你得到的是新版本。你需要它的唯一理由来自于你的律师，他让你这么做以便日后出现问题有责任人可以追究。我们不追究任何人，我们自己会搞定那些问题。”他说。

如果慕尼黑市的 IT 人员不能自己修复 bug，他们可以寻求专家解决特定的 bug。“你不再依赖从软件供应商那里买来的服务，你依靠你自己和你掌握的信息。”他说。

尽管如此，仍有一些抱怨。从外部组织收到的 Word 和 Excel 文档有时需要做一些更改再发回去，这会给互操作性带来麻烦。慕尼黑市正在说服与其通信的人员使用 OpenOffice 的 ODF 格式文档，或者对不需要修改的文档使用 PDF 格式。慕尼黑市已经借助这一途径协助了互通工具在金融业的推广。

作为 OpenOffice 迁移计划的一部分，慕尼黑市还开发了 WollMux。这是一个用于模板和表单的 Office 扩展，于 2008 年以自由软件的形式发布，如今已经被少数其他组织采用。

清除 Microsoft Office 的工作中还存在一些障碍——例如，该市内部应用程序对数千个 Microsoft Office 和 Visual Basic 宏的依赖。如今仍然有约 100 个这样的宏在少数 Windows PC 上使用。

“我们的目标从来都不是彻底清除 Windows，”他说，尽管该市已经超出了迁移 80% PC 的最初目标并且一直运转良好。

以财政部门为例，三台 Windows PC 仍然在运行特殊的银行软件。如果要迁移到 LiMux，市政府得付钱让软件供应商为这三台机器上运行的应用程序开发 LiMux 版本。

该市与德国政府印刷护照的“联邦印刷”打交道时也面临类似的问题。联邦印刷要求使用一个 Windows 应用程序发送护照所需的个人数据。

Hofmann 对这个城市的未来充满信心，但也认识到并不是所有人都适合迁移到 Linux。即使不想迁移到 Linux 的人也一样可以从慕尼黑市的经验中受益：“曾有人对我说，‘从你启动项目那时起，我就可以和 Microsoft 谈判了。’”
