User: 在Docker中如何实现容器间网络通信？
Assistant: 在Docker中，可以使用网络模式来实现容器间的网络通信。以下是几种常见的网络模式：

1. Bridge模式：默认模式，使用Docker自带的虚拟网桥，将所有容器连接到同一网络中。
2. Host模式：容器使用主机实际的网络接口，与主机共享IP地址和端口号，可以提高网络性能，但可能会导致安全问题。
3. Overlay模式：用于跨主机的容器通信，可在多个Docker主机上创建的多主机网络中进行容器通信，通过VXLAN覆盖网络连接各个节点。
4. None模式：容器没有网络接口，只能通过添加额外的网络模式或手动配置容器网络接口，才能进行网络通信。

无论哪种模式，都可以使用docker network命令进行创建和管理网络。例如，使用以下命令创建一个自定义的Bridge方式的网络：

```
docker network create my_net
```

然后，在运行容器时，可以通过以下命令将容器连接到网络中：

```
docker run --rm -it --network my_net ubuntu /bin/bash
```

其中，--network参数指定要连接的网络，Ubuntu表示使用什么镜像启动容器，/bin/bash则是运行的命令。成功运行后，可以在容器中ping其他容器的IP地址，进行容器间的网络通信。
