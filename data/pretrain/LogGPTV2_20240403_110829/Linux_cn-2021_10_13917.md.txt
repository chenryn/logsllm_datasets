### 在开源社区的强烈抗议下，微软放弃了有争议的 .NET 变化

![](/data/attachment/album/202110/24/125054fqk4q5669kq6z77w.jpg)

本周早些时候，微软因商业利益原因，在即将发布的 .NET 6 中 [删除](https://github.com/dotnet/sdk/pull/22217) 了“热重载”功能的部分特性，此举 [激怒](https://github.com/dotnet/sdk/pull/22262) 了 .NET 开源社区。该功能允许开发者在应用运行时修改源代码并立即看到结果，是许多人期待已久的功能，特别是在 VS Code 和多个平台上使用。然而，微软在最后一刻做出了一个有争议的 [决定](https://devblogs.microsoft.com/dotnet/update-on-net-hot-reload-progress-and-visual-studio-2022-highlights/)，将其限制在 Visual Studio 2022 中，这是一个仅限于 Windows 平台的付费产品。这一决定是由微软开发者部门负责人 Julia Liuson 做出的，显然是出于商业利益考虑。在开源社区的巨大抗议后，微软最终推翻了这一改变，并轻描淡写地 [表示](https://devblogs.microsoft.com/dotnet/net-hot-reload-support-via-cli/)：“无意中最终删除了源代码”，但承认“有时我们不能正确处理这个问题，……我们能做的就是从错误中学习。”

**老王点评**：虽然商业企业有时会支持开源，但其根本目标与开源理念并不一致。因此，企业在某些情况下可能会“慷慨”支持开源，而在其他情况下则可能因短期利益而将开源视为可弃之物。

### 阿里巴巴/蚂蚁的 OceanBase 因集赞活动引发社区批评

![](/data/attachment/album/202110/24/125117z9lkd8ku38heeek9.jpg)

昨天，蚂蚁旗下的再次开源的 OceanBase 数据库发起了一项 [活动](https://www.zhihu.com/question/494108102/answer/2184918236)，邀请程序员为其 GitHub 仓库点赞，并根据收集到的点赞数提供不同奖品，从贴纸到手办不等。这一活动引发了开源社区的大量 [批评](https://www.zhihu.com/question/494108102/answer/2184934080)。阿里巴巴集团副总裁、阿里云计算平台事业部负责人贾扬清在知乎上回应称，“除了搞烂一个社区活跃度指标之外，完全没有意义”，并表示“我们在负责设计开源评判的制度，没设计好自然执行会出问题”，承诺“这种瞎搞绝对是要认真处理的”。他还表达了希望留待日后评判的意思。此外，一些开源界的意见人士也评论说，“上有所好，下必盛焉”。阿里在其宣传中确实将点赞数列为重要成绩。

**老王点评**：以任何硬性指标来评判开源项目，只要有绩效考核驱动，就难免会有主动或被动迎合指标的行为。然而，如果没有指标和模型来指导，开源项目的发展也将无从谈起。我认为，开源项目需要一个更复杂均衡的模型，一个能够适应变化的弹性模型。

### 76 岁的 EFF 共同创始人被踢出董事会

![](/data/attachment/album/202110/24/125133czc472kl0t2o9w7g.jpg)

电子前沿基金会（EFF）周五 [发布声明](https://www.eff.org/deeplinks/2021/10/john-gilmore-leaves-eff-board-becomes-board-member-emeritus) 将 [John Gilmore](https://en.wikipedia.org/wiki/John_Gilmore_(activist)) 除名，并授予他名誉董事的角色。除名原因是双方在沟通和合作方式上存在分歧，无法就 Gilmore 负责的方向达成一致。Gilmore 于 1990 年共同创立了 EFF，并在此后的 31 年里一直在其中担任职务。他曾帮助发起了伯恩斯坦诉司法部案，该案的结果是法院裁定软件源代码受美国第一修正案保护，美国政府阻止其发布的法规违宪。这一裁决使得 1999 年 Web 浏览器、网站以及 PGP 和 Signal 等软件可以合法使用自己选择的加密方式。他也是一系列 GNU 软件和 RFC 的作者或共同作者。

**老王点评**：究竟是老一代自由软件领袖跟不上新时代的步伐，还是他们所建立的事业偏离了他们的理想？
