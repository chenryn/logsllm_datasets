> **简介**: 英特尔计划在 Linux 内核 5.17 中引入一项新功能，使得在不重启系统的情况下更新系统固件成为可能。

![](/data/attachment/album/202201/06/140244iepkefghpeeezgzg.png)

去年发现的多个安全漏洞使系统管理员难以在不停机的情况下迅速修补系统。如果有一种方法可以在不重启系统的情况下更新关键组件以提高安全性和性能，会怎么样呢？

英特尔希望通过其新的 PFRUT（平台固件运行时更新和遥测）驱动程序实现这一目标。

### 英特尔计划在 Linux 内核 5.17 中提供 PFRUT

尽管 Linux 内核 5.16 将于本周晚些时候发布，但英特尔的目标是将这项新功能合并到即将发布的 Linux 内核 5.17 稳定版中。

那么，PFRUT 到底是什么？

借助 PFRUT 驱动程序，特定组件（或系统固件）可以在系统运行时进行更新，而无需重新启动。最初，英特尔将其称为“无缝更新”解决方案。然而，最近添加到 Linux 电源管理的 linux-next 分支中的 Linux 内核提交表明，他们可能会继续使用一个厂商中立的名字，即 `pfrut_driver`。

顺便提一下，linux-next 分支代表这些更改将进入下一个 Linux 内核稳定版（5.17）。

技术上，[这个提交](https://git.kernel.org/pub/scm/linux/kernel/git/rafael/linux-pm.git/commit/?h=linux-next&id=0db89fa243e5edc5de38c88b369e4c3755c5fb74) 对该变化的解释如下：

> 用户应提供 EFI 封包，并通过将封包写入设备的特殊文件传递给驱动程序。驱动程序通过特定的 ACPI 平台固件运行时更新设备（INTC1080）的 ACPI \_DSM 方法的帮助，将封包传输到平台固件，实际的固件更新由平台固件中的低级管理模式代码完成。

正如人们通常期望的那样，这应该可以消除停机时间，从而对解决任何安全和性能改进的固件进行基本更新。而且，系统固件更新可以直接通过操作系统（这里是 Linux）进行。

正如其中一个 [详细说明其工作原理的 PDF 文件](https://uefi.org/sites/default/files/resources/Intel_MM_OS_Interface_Spec_Rev100.pdf) 中强调的那样，驱动程序的遥测部分是为了“从 MM 中检索日志信息，以便进行监控并找出问题的根本原因”。

请注意，这只有在 Linux 系统和英特尔芯片上才能实现。

考虑到当需要修补系统固件以抵御安全问题时，等待系统上的任务完成并不是理想情况，这种能力的增加应该是非常方便的。

### 这是为 Linux 桌面还是服务器准备的？

该改进主要是为服务器专用硬件量身定制的。

英特尔的官方文件指出，这是为具有高服务水平协议（SLA）的系统准备的，这种 SLA 需要尽量减少重启。然而，这对拥有企业级系统的特定桌面用户群也应该是有用的。

虽然这可能不是桌面 Linux 发行版必不可少的功能，但它可能是改善用户体验的一个令人兴奋的开始。特别是对于热衷于保持系统固件更新而不严重干扰其工作的用户。

这也应该可以引入更多类型的更新可能性，当涉及到 BIOS 或 UEFI 时，可以由操作系统而不是主板处理。

这不仅限于为你的桌面配置服务器级别硬件的 Linux 桌面用户。虽然目前仅限于 Linux 系统，但这也应该很快可用于 Windows 和其他操作系统。
