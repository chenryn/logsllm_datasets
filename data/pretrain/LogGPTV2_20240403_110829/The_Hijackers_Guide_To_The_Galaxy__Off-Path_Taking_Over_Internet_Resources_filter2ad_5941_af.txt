### 优化后的文本

#### 攻击概述
攻击者在接管客户账户后可以执行以下操作，这些操作可能导致不同程度的损害。表5总结了这些可能的操作及其后果。

| 操作 | 结果 |
| --- | --- |
| 禁用电子邮件警报 | 长期控制 |
| 更改账户详细信息 | 长期控制 |
| 关闭账户 | 长期控制 |
| 保持隐蔽 | 不触发警报 |
| 转移资源 | 长期控制 |
| 删除资源 | 无法恢复 |
| 出售资源 | 经济损失 |
| 修改Whois数据库 | 促进劫持 |
| 创建新的ROA/证书 | 促进劫持 |
| 创建无效的ROA | DoS攻击 |
| 撤销证书 | DoS攻击 |

#### 实验方法
为了评估这些攻击的影响，我们首先通过第4节中的方法接管受害者的账户，然后执行上述操作。我们的目的是了解在这些攻击过程中，真实账户所有者会收到哪些通知，攻击者可以执行哪些操作，以及哪些操作被阻止。

#### 区域互联网注册机构 (RIRs)
攻击者可以通过控制一个LIR（本地互联网注册机构）的SSO账户来操纵与该LIR相关的所有互联网资源，例如IPv4和IPv6地址、ASN等。不同规模的LIR管理的资源数量差异巨大。小型LIR可能只管理自己的AS、一个IPv6前缀和少量IPv4前缀；大型LIR则可能管理大量PA地址池（可聚合提供商地址）。

**示例：**
- **RPKI管理**：攻击者可以创建、删除或修改托管RPKI系统中的路由源授权 (ROA)。这有两个目的：(1) 扰乱由受害者LIR管理的资源的合法BGP更新传播；(2) 通过授权攻击者的ASN来发起任何子集的IP前缀，从而促进BGP劫持。
- **RIPE DB修改**：攻击者可以操纵RIPE DB中的记录，包括互联网地址资源注册表和互联网路由注册表 (IRR)。修改这些记录可以让攻击者冒充受害者LIR的代表，将资源转移到不知情的接收方。
- **IPv4地址资源转移**：攻击者可以出售由受害者LIR管理的资源。具体步骤包括：
  1. 修改RIR数据库和IRR中的相关联系信息。
  2. 找到买家并冒充受害者代表完成IPv4资源转移合同。
  3. 从BGP路由表中释放IPv4前缀。
  4. 向RIPE NCC发送扫描的IPv4资源转移合同。

#### 域名注册商
域名注册商处理客户的域名所有权。我们使用GoDaddy作为示例，展示了攻击者在接管账户后可以执行的操作：
- 更改名称服务器的IP地址，以劫持客户的域名。
- 授予自己账户访问权限或执行跨注册商的域名转移。
- 更改电子邮件转发设置，以劫持转发给账户所有者的电子邮件。
- 删除与被攻破账户关联的域名，甚至关闭账户。

#### 云服务提供商
攻击者可以操纵与账户关联的虚拟资源，如虚拟机、网络接口和磁盘。此外，攻击者还可以利用这些资源对互联网上的其他目标或同一云平台上的目标进行攻击，例如通过侧信道攻击。

#### 证书颁发机构 (CAs)
当控制一个CA客户的账户时，攻击者可以撤销和重新颁发证书，从而更改与证书关联的密钥对。某些CA在重新颁发证书时不强制验证，因此攻击者可以通过劫持账户来更改目标域名的密钥。

#### 易受攻击的数字资源
我们分析了不同提供商的账户被劫持的风险，并计算了这些账户关联的数字资源（域名、IP地址和ASN）的数量。结果显示，攻击者可以通过第4节中的任一方法接管68%的IPv4地址空间。

#### 总结
通过以上实验和分析，我们发现许多账户和相关资源存在严重的安全风险。加强账户安全措施，如启用双因素认证，是防止此类攻击的关键。
