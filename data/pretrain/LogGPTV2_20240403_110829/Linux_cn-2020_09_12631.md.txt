## GNOME 下一个版本将从 3.38 跳到 40.0

原计划在 2021 年 3 月发布的下一个 GNOME 版本，即 GNOME 3.40，将被重命名为 GNOME 40。在首个稳定版 GNOME 40.0 发布后，后续的 bug 修复版本将以小数点后的数字递增，例如 40.1、40.2 和 40.3 等。再下一个版本则为 GNOME 41。

此次版本号调整的主要原因是，在可预见的未来，开发者不会像从 GNOME 2 到 GNOME 3 那样进行核心技术的重大变更。因此，大的版本号也不会随之增加。为了避免可能带来的误解，开发者决定将 3.40 直接更名为 GNOME 40，而非 GNOME 4.0。

来源：[solidot](https://www.solidot.org/story?sid=65578)

**拍一拍：** 又一轮版本号竞赛开始——反正特性不够版本号凑。开源技术界中像 0.01 这样的微小版本号传统正在逐渐消失。

## GitHub CLI 1.0 发布

GitHub 的命令行工具（CLI）发布了 1.0 正式版本。GitHub CLI 允许开发者直接在终端内操作 GitHub 的各项功能，如拉取请求和议题管理。此外，它还支持用户为任意命令创建别名，并提供直接访问 GitHub API 的能力。

来源：[solidot](https://www.solidot.org/story?sid=65576)

**拍一拍：** 没有命令行的 GitHub 显得不够极客——当然，命令行确实有助于实现更多自动化功能。

## 用了 21 年，Thunderbird 78.2 正式加入对 OpenPGP 的支持

早在 21 年前，用户就向 Mozilla 提出了希望加入 PGP/OpenPGP 支持的功能请求，当时电子邮件客户端 Thunderbird 尚未诞生。在此之前，用户通常需要使用 Enigmail 扩展来加密邮件。随着 Thunderbird v78.2.1 的发布，该软件正式默认启用了对 OpenPGP 的支持。

来源：[solidot](https://www.solidot.org/story?sid=65575)

**拍一拍：** 尽管 PGP/OpenPGP 在保护电子邮件安全方面表现优异，但其普及程度一直不尽人意。
