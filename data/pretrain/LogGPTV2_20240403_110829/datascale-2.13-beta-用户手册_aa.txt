### DataScale 2.13.0-beta 版本介绍

DataScale 是一款轻量级、高性能的数据采集工具，旨在帮助用户实时采集、处理来自多种数据源的数据（如日志、指标、调用链数据等），并将这些数据导入到炎凰数据平台（商业版或社区免费版）。炎凰数据平台是一款由[公司名称]开发的异构大数据即时分析平台。

#### 主要特点
- **直观的Web UI**：用于管理数据管道，快速连接各种数据源，并配置数据处理逻辑。
- **集成开源数据采集器**：支持丰富的开箱即用数据源类型和完善的数据处理方法。
- **调试工具**：提供便捷的工具来调试数据采集配置和数据处理逻辑。
- **定制化支持**：允许用户开发定制的数据采集器。
- **批量部署与集群管理**：提供易于使用的批量部署和集群管理功能。
- **可观测性**：提供更完善的监控和日志记录功能。

#### 使用方式
DataScale 支持两种部署模式：
- **独立部署**：每个 DataScale 实例独立负责其自身的数据采集任务的管理和执行。
- **集群部署**：分为 worker 和 manager 两种角色。worker 负责执行数据采集任务，而 manager 则集中管理和分配这些任务给 worker。

#### 运行环境
DataScale 目前支持的操作系统包括 Linux 各版本、Windows 以及 macOS。您可以通过下载安装包在本地运行 DataScale 服务，也可以使用 Docker 镜像在容器环境中运行。

#### 基础功能
默认情况下，DataScale 以独立模式运行，具备以下核心功能：
- **Web UI 和 RESTful API**：用于管理 DataScale 服务。
- **服务与 dataflow 配置管理**：支持服务配置及数据流配置的管理。
- **数据采集器管理**：能够管理和调度数据采集器的运行。
- **与炎凰数据平台集成**：确保数据可以顺利传输至目标平台。

#### Web 用户界面
通过浏览器访问 DataScale 的 Web UI，您可以查看服务状态、管理系统设置、管理群组及 dataflow 配置等。

#### RESTful API
DataScale 的大部分功能可通过调用 RESTful API 来实现，同时 API 也提供了部分不在 Web UI 中展示的功能。更多关于 RESTful API 的信息，请参阅相关文档。

#### 配置文件
DataScale 的行为受 `datascale.toml` 文件控制，该文件位于安装目录下的 `config/` 文件夹内。您可以在此文件中修改服务监听端口、运行模式（worker 或 manager）、配置存储方式（文件或数据库）等参数。更改后需重启服务使新配置生效。

#### Dataflow
DataScale 的核心在于定义并执行一系列数据采集任务。每个任务对应一个数据流转通道（dataflow），包含三个步骤：从源获取数据 -> 数据预处理 -> 将数据存入炎凰数据平台。这三步分别由 Source、Pipeline 和 Sink 组件实现。

#### 群组 (Group)
Dataflow 可以被组织成不同的组，便于管理和分配任务给特定的 worker。每个 worker 可以隶属于一个或多个组，根据其配置选择执行相应的 dataflow。

#### 第三方数据采集器整合
除了内置的数据采集器外，DataScale 也支持用户扩展更多的数据采集器，以满足特定需求。

#### 与炎凰数据平台的集成
DataScale 不仅能将收集到的数据发送至炎凰数据平台，还会自动上传自身运行时的一些关键指标，以便于监控和服务优化。
