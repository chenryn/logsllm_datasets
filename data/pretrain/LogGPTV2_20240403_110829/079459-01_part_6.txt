### 1.1.3 许可程序

微软提供了多种不同的许可程序，每种程序都有其独特的优势、限制和成本。通过这些程序，可以获取Windows Server 2016的许可及其客户端访问许可证（CAL）。由于这些许可程序会随时间发生变化，因此建议与相关专家讨论以确定最佳购买方式。以下是一些主要许可类型的概述：

- **原始设备制造商 (OEM) 许可**：当购买新的物理服务器时，可以选择这种类型的许可。尽管它通常是成本最低的选择，但该许可不能转移到其他硬件上。
- **批量许可**：这种许可比OEM许可更为灵活，因为它不受特定物理服务器的限制。不过，在服务器之间转移此类许可的频率是有限制的。对于需要在虚拟化主机间移动虚拟机以实现高可用性的场景来说，这是一个重要的考量因素。
- **软件保证**：此类型许可作为批量许可的一个附加选项提供，包括了软件升级的权利。此外，软件保证还允许用户随时在不同物理服务器间迁移许可。
- **企业协议**：基于用户的许可形式，而非基于服务器。如果组织为每位用户支付固定费用，则可以根据需求运行任意数量的服务器实例。这类许可通常也涵盖了CAL，并可能包含其他产品如SQL Server和Exchange Server。

### 1.1.4 Windows Server 2016 的其他版本

- **Windows Server 2016 Essentials**：专为小型企业设计的简化版操作系统。相较于标准版或数据中心版，Essentials不需要CAL支持，但是对用户数（不超过25个）及连接设备数（不超过50台）有所限制。此外，不支持多实例虚拟化，内存上限为64GB，并且最多只能使用两个物理CPU。为了便于部署，部分服务器角色和功能会被自动安装和配置。
- **Windows Storage Server 2016**：仅能通过存储设备制造商获得。该版本旨在作为一个通用的操作系统使用，因此支持的服务器角色相对较少，例如无法将其配置为域控制器。

更多关于Windows Server 2016许可的信息，请访问[Microsoft官网](https://www.microsoft.com/en-us/cloud-platform/windows-server-pricing)上的“Windows Server 2016 Licensing & Pricing”页面。

### 1.2 安装 Windows Server 2016

#### 物理服务器安装
在开始安装之前，请确保已准备好所有必要的驱动程序。某些制造商可能会提供一个专门的过程来在安装过程中注入这些驱动程序。现代服务器通常采用统一可扩展固件接口(UEFI)，而不是传统的基本输入输出系统(BIOS)。虽然可以通过设置让UEFI模拟BIOS模式工作，但这并不是必须的。实际上，利用UEFI能够带来诸如支持更大容量磁盘以及更安全启动等好处。

#### 磁盘分区规划
在进行安装前，还需要规划好磁盘分区方案。特别需要注意的是C:盘的空间分配，不仅要满足初始安装的需求，还需预留足够的空间供将来更新之用。同时，建议将应用程序和数据存储于独立于操作系统的分区中，这样有助于防止操作系统盘空间不足的问题，并简化备份与恢复流程。

#### 虚拟机环境下的安装
很多情况下，服务器会被部署为虚拟机。这不仅提高了部署和管理的灵活性，同时也要求Windows Server 2016具备针对相应虚拟环境的正确驱动程序。对于Hyper-V平台而言，安装文件已经包含了所有必需的组件；而对于其他虚拟化技术（如VMware），则可能需要额外安装特定工具（如VMware Tools）。

### 1.2.1 安装步骤
1. 将服务器设置为从DVD引导。
2. 放入安装光盘并启动服务器，按照屏幕提示选择语言、时间和键盘布局。
3. 点击“Install Now”按钮继续。
4. 在激活窗口中输入产品密钥或选择稍后输入。
5. 选取所需的操作系统版本后点击下一步。
6. 阅读并接受许可条款。
7. 根据实际需求选择“Server Core”或“Desktop Experience”安装模式。
8. 指定操作系统应被安装的目标磁盘。
9. 等待文件复制完成，整个过程可能耗时较长。
10. 重启后设置本地管理员账户密码。

### 1.2.2 安装后的配置
安装完成后，可能需要立即调整以下几个方面：
- 计算机名称：默认生成的名称不符合大多数企业的命名规范。
- 工作组/域成员身份：通常希望加入到现有域中而非保持默认的工作组状态。
- IP地址配置：默认启用DHCP获取IP地址，但对于多数企业来说，静态IP地址更加适用。
- 时区设定：根据服务器地理位置调整正确的时区。

若选择了带有图形界面的“Desktop Experience”，可通过“Server Manager”工具轻松完成上述更改。而如果是“Server Core”模式，则需借助命令行工具或PowerShell脚本来实现相应的配置。

### 1.2.3 激活
所有版本的Windows Server 2016都需要经过激活才能正常使用。未激活的产品将在180天后进入通知模式，在此期间将无法享受全部功能。对于小规模部署，可以直接联系微软在线或电话激活OEM许可；而对于大型企业级应用，则推荐使用更为灵活的批量许可机制，特别是通过MAK方式进行多次激活。
