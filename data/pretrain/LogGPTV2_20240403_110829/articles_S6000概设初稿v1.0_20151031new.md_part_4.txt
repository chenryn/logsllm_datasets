表3 业务活动清单
+------------+---------------------------------------------------------+
| **业务     | **业务活动内容/步骤**                                   |
| 活动名称** |                                                         |
+------------+---------------------------------------------------------+
| 数据获取   | 系统从网省公司其他关联平台获取流量和日志信息。          |
+------------+---------------------------------------------------------+
| 原         | 系统将原始数据流量以及日志存在网省公司本地。            |
| 始数据取证 |                                                         |
+------------+---------------------------------------------------------+
| 数据预处理 | 系统对原始数据进行处理，使其变为格式化的数据            |
|            | ，将输出结果输入进分析系统进行场景分析，预处理步骤为：  |
|            |                                                         |
|            | 1.  数据去重；                                          |
|            |                                                         |
|            | 2.  数据去噪；                                          |
|            |                                                         |
|            | 3.  数据范式化；                                        |
|            |                                                         |
|            | 4.  数据加强。                                          |
+------------+---------------------------------------------------------+
| 情         | 系统通过其他网站的信息，自动收集情报。                  |
| 报自动获取 |                                                         |
+------------+---------------------------------------------------------+
| 情         | 1.  系统对收集的情报进行验证；                          |
| 报信息验证 |                                                         |
|            | 2.  系统剔除重复的情报以及不需要或者不合规的情报；      |
|            |                                                         |
|            | 3.  提交后，系统做情报关联分析。                        |
+------------+---------------------------------------------------------+
| 情         | 1.  分析员将网站发布的情报录入系统中；                  |
| 报人工录入 |                                                         |
|            | 2.  录入后，系统做情报关联分析。                        |
+------------+---------------------------------------------------------+
| 建模创建   | 分析员将现有的场景规则化，建立场景模型。                |
+------------+---------------------------------------------------------+
| 语义审核   | 1.  审核员对场景的规则进行审核；                        |
|            |                                                         |
|            | 2.  如果认为场景规则无问题，则提交；                    |
|            |                                                         |
|            | 3.  如果认为场景规则有问题，则驳回。                    |
+------------+---------------------------------------------------------+
| 模型试运行 | 新建好的场景规则由监控员在总部进行试运行。              |
+------------+---------------------------------------------------------+
| 模型发布   | 监控员对条件成熟的场景模型进行发布。                    |
+------------+---------------------------------------------------------+
| 模型部署   | 监控员将模型部署到网省公司的系统中去。                  |
+------------+---------------------------------------------------------+
| 日志关联   | 系统对采集的数据做实时分析、定时分析、交互式分析。      |
+------------+---------------------------------------------------------+
| 生成告警   | 系统在日志关联结束后生成告警信息。                      |
+------------+---------------------------------------------------------+
| 情报关联   | 系统对录入的情报与网省公司的设备资产进行关联。          |
+------------+---------------------------------------------------------+
| 生成预警   | 系统在情报关联结束后生成预警信息。                      |
+------------+---------------------------------------------------------+
| 审核研判   | 1.                                                      |
|            |  总部的分析员对各网省系统生成的预警和告警信息进行查看； |
|            |                                                         |
|            | 2.  其中低危威胁判定为通告；                            |
|            |                                                         |
|            | 3.  高危威胁判定为事件；                                |
|            |                                                         |
|            | 4.  误判的威胁则关闭。                                  |
+------------+---------------------------------------------------------+
| 发送通告   | 系统将提交的通告信息通告到指定的省公司或者全网。        |
+------------+---------------------------------------------------------+
| 事件查看   | 总部的审核员定期查看需要处置的事件。                    |
+------------+---------------------------------------------------------+
| 下发审核   | 总部的审核员将需要处置的事件下发给需要处置的网省公司。  |
+------------+---------------------------------------------------------+
| 事件处置   | 1.  各网省的处置员定期查看需要处置的事件；              |
|            |                                                         |
|            | 2.  处置员对事件进行处置。                              |
+------------+---------------------------------------------------------+
| 处置反馈   | 网省公司的处置员在处置好事件后，将结果反馈给总部。      |
+------------+---------------------------------------------------------+
| 审核确认   | 1.  总部的审核员定期查看已经处置好的事件；              |
|            |                                                         |
|            | 2.  处置员调查事件是否确实处置完毕。                    |
+------------+---------------------------------------------------------+
| 关闭事件   | 对确实处置完毕的事件，进行关闭。                        |
+------------+---------------------------------------------------------+
