> Canonical 是否计划用这个基于 Flutter 的应用程序取代 Ubuntu 软件中心？也许是，也许不是。

![](/data/attachment/album/202209/22/113219bq5q80aisz5a89si.jpg)

Ubuntu 通常被认为是适合初学者的最佳发行版之一，但其默认的软件中心（GNOME Software Center）仍需改进以提供更佳的用户体验。原因在于它运行缓慢且占用大量资源，导致许多用户不愿使用或体验不佳。尽管多年来有所改善，但仍远未达到理想的用户体验。

近期，一些贡献者联手开发了一个轻量级、基于 Flutter 的替代方案——“Ubuntu 软件”。需要注意的是，这并非官方替代品，但人们不禁猜测它是否会很快取代现有的 Ubuntu 软件中心？

### 使用 Flutter 制作的 “Ubuntu 软件”

![ubuntu software built with flutter](/data/attachment/album/202209/22/113220y4b6x3u5wwwuvwfs.png)

这款基于 Flutter 实现的 “Ubuntu 软件”旨在实现轻量化、适应性强和快速响应，从而为用户提供优于传统软件中心的体验。首要考虑因素是可用性，因为用户最终目的是快速安装软件，而无需面对繁琐的过程。

在我个人的系统上试用时，我发现“我的应用程序”下有一个单独的 “Snaps” 和 “Deb 包”部分：

![separate section for deb and snap packages in ubuntu software](/data/attachment/album/202209/22/113221ekttt4pt7bykzpto.png)

这种设计使得在一个界面中管理不同类型的应用程序变得更加便捷。此外，应用信息和安装页面经过重新设计，加入了微妙的动画效果，提升了整体体验。

![app info section in unofficial ubuntu software](/data/attachment/album/202209/22/113222ked8eiwcoraihhd8.png)

自适应布局则进一步增强了这一新软件中心的实用性，类似于 elementary OS 7 中的做法：

**[elementary OS 7 准备添加响应式应用程序，并移植到 GTK 4](https://news.itsfoss.com/elementary-os-7-dev-updates/)**

![adaptive layout in unofficial ubuntu software center](/data/attachment/album/202209/22/113222cpvip9xpxemicgwv.png)

### 它会取代 Ubuntu 软件吗？

尽管该项目被标记为非官方替代品，但从参与者的角度来看，似乎更像是一个实验性的尝试。特别是考虑到 Canonical 公司负责 Flutter 的高级工程师 [JP Nurmi](https://twitter.com/jpnurmi) 也参与其中，未来的发展值得期待。目前我们只能拭目以待，不过从当前的趋势来看，它有潜力成为正式的替代品。

### 想尝试这个基于 Flutter 的商店吗？

你可以在其 [GitHub 页面](https://github.com/ubuntu-flutter-community/software) 上找到构建指南并开始使用。请注意，该项目正处于积极开发阶段。

**[下载 Ubuntu 软件](https://github.com/ubuntu-flutter-community/software)**

对于一个正在开发中的项目来说，它的表现已经相当不错了。*我个人希望现有的软件中心能够被取代。你怎么看？请在评论区分享你的看法！*
