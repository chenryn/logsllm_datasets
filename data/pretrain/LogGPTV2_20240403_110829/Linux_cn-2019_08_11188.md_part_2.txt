### 网易云：专注于解决问题的技术选择
网易云团队并不刻意追求新兴技术，而是更注重能够实际解决问题的方案。因此，他们选择了对微服务架构支持最佳的 Kubernetes 作为其核心平台。

### 企业云：致力于解决客户问题
与许多云计算公司不同，网易云并未将所有精力都投入到公有云上，而是专注于为企业提供业务云化的解决方案。尽管网易云很早就采用了 Docker 和 Kubernetes 等容器技术，但它们更多地被视为工具而非卖点。网易云通过结合容器网络和存储方案等云原生技术积累，旨在更好地服务于客户。
  
网易云选择深耕企业云领域的原因在于，他们认为自己更适合在云原生领域进行深度开发。与其在竞争激烈的公有云市场中挣扎，不如专注于提升基于 Kubernetes 标准化 PaaS 服务的技术水平和竞争力。这种策略不仅有助于避开红海竞争，还使得其产品具有更强的生命力和灵活性，能够在任何 Kubernetes 服务商的平台上运行。

陈谔指出，作为一个面向企业的解决方案提供商，网易云希望贴近企业的 IT 技术认知，避免给客户带来不必要的认知负担或业务侵入性。只有当企业真正需要实施微服务框架时，才会建议进行容器化迁移。对于那些不打算采用容器化的企业，网易云也提供了诸如服务网格（Service Mesh）的支持，以适应不同的环境需求。

### DevOps 实践与观点
谈及内部 DevOps 实践，陈谔表示，自2014年起，网易云就开始推行服务化的组织结构和协作模式。每个接口背后都有一个对应的服务及研发团队，从而实现接口先行的工作方式。此外，除了基础设施层面由专门的 SRE 团队负责外，其他服务运维均由各自团队自行处理，这促使团队必须具备解决运维需求的能力。同时，为了促进更好的协作，所有 API 都被统一管理在一个网关中，用户可以通过这些 API 完成所需操作，而无需依赖 Web 界面。

关于容器化与 DevOps 的关系，陈谔认为两者是相辅相成的。随着企业业务向微服务架构转型，传统集中式运维成为瓶颈，促使企业寻找新的运维方法。在这种背景下，容器技术应运而生，为开发者提供了便捷的运维工具。即使没有 Docker 公司推出容器化，市场上也会出现类似的产品来满足这一需求。

### 轻舟微服务：助力企业微服务转型
轻舟微服务是网易云基于对行业趋势和技术栈标准化的研究后推出的解决方案。该项目旨在帮助企业克服异构系统整合、多供应商合作等方面的挑战，并通过集成服务网格、分布式事务框架 GTXS 以及全新 API 网关等功能，完成全栈在线中台技术体系升级，支持业务快速创新。

轻舟服务网格基于 Istio 和 Envoy 等主流开源技术构建，可以兼容多种编程语言并支持跨平台部署。它不仅实现了与现有微服务框架 NSF 的无缝对接，还达到了业界领先的异构系统支持水平。

对于那些希望进行微服务改造但缺乏相关经验和技术储备的企业来说，轻舟微服务提供了一整套包含咨询服务和技术指导在内的完整解决方案，帮助企业高效地完成转型过程。例如，在测试阶段，利用流量染色功能可以在单一环境中同时测试多个版本的服务，显著降低了测试成本和复杂度。

### 结语
通过这次采访，我对网易云及其产品的了解更加深入。显然，网易云已经找到了企业数字化转型过程中的关键痛点，并且能够根据客户需求提供定制化解决方案。这种以客户为中心的发展理念不仅支撑了像网易云音乐、网易考拉这样的明星项目，也在推动整个行业的进步方面发挥了重要作用。
