### 目标如何实现
- **场景**：我要重启应用怎么操作？
  1. 在PSP系统中提交工单。
  2. 提交后，将链接发给相关人员审核（DevOps审核权限）。
  3. 审核通过后，执行重启并验证结果。

## Page 13
### 交接范围
| 分组 | 工单类型 | 提单权限 | 审批权限 |
|------|----------|----------|----------|
| 应用交付 | 新应用上线 | 不限制 | DevOps、PE |
| 应用扩容 | PE、Dev | 无需审批（预算内） | - |
| 应用维护 | 账号申请 | 不限制 | AppOps、DevOps、PE |
| 应用重启 | Owner、PE —> Dev | - | - |
| 机器重启 | Owner、PE —> Dev | - | - |
| JAVA应用Dump | Owner、PE —> Dev | 无需审批 | - |
| 日志清理 | Owner、PE —> Dev | PE —> 无需审批 | - |
| 应用Docker化灰度验证 | Owner、PE —> Dev | 无需审批 | - |
| 配置变更 | PE —> Dev | 无需审批 | - |
| 应用回收 | 机器下线 | PE —> DevOps | - |
| 应用下线 | Owner、PE —> Dev | - | - |
| 网络变更 | 域名变更 | Owner、PE —> Dev | PE、安全 —> DevOps、安全 |
| VIP变更 | Owner、PE —> Dev | PE、安全 —> DevOps、安全 |
