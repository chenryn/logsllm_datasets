### 访问此计算机上的 Web 服务器

目前，我们已经设置了一个基本的防火墙策略，该策略允许 TCP、UDP 和 ICMP 回复请求，并允许外部访问此计算机上的 Web 服务器。除此之外，所有其他传入流量均被拒绝。

### 如何进一步修正防火墙策略

为了更好地控制来自网络中其他计算机的流量，我们可以使用拒绝和强制允许规则。例如，我们可能希望允许内部用户访问此计算机上的 Web 服务器，但拒绝 DMZ 中的任何计算机进行访问。为此，可以通过添加一条拒绝规则来禁止位于 DMZ IP 范围内的服务器访问。

1. **针对 DMZ 的拒绝规则**：接下来，我们将为源 IP 为 10.0.0.0/24 的传入 TCP 流量添加一条拒绝规则。这将阻止 DMZ 中的计算机与本计算机之间的任何流量。
   
2. **针对特定服务的强制允许规则**：然而，我们可能还需要进一步调整策略，以允许从 DMZ 中的邮件服务器传入的流量。为此，可以对来自源 IP 10.0.0.100 的传入 TCP 流量使用强制允许规则。这条规则将覆盖之前创建的拒绝规则，从而允许来自 DMZ 中特定计算机的流量。

### 重要注意事项
- 所有流量在经过状态检查引擎分析前，都必须先对照防火墙规则进行检查。如果流量通过了防火墙规则，则会继续由状态检查引擎进行处理（前提是已启用防火墙状态检查）。
- 允许规则是必需的。未在允许规则中指定的任何流量都将自动丢弃。因此，您需要确保包含必要的流量类型规则，如 ARP 流量。
- 如果启用了 UDP 或 ICMP 状态检查，必须使用强制允许规则以允许未经请求的 UDP 或 ICMP 流量。
- 强制允许规则仅在相同的优先级上下文中才是最佳实践。
- 如果没有配置 DNS 或 WINS 服务器，在测试环境中应为 NetBios 配置强制允许传入 UDP 端口 137 规则。

### 故障排除建议
当对新的防火墙策略进行故障排除时，首先应检查客户端或设备上的防火墙规则日志。这些日志提供了关于哪些流量被拒绝的信息，有助于进一步修正策略。

### 放行规则
放行规则是一种特殊的防火墙规则，主要用于处理媒体密集型协议。这类规则的特点包括：
- 符合放行的数据包不会由入侵防御规则处理。
- 在防火墙状态配置开启时，放行不会自动允许对 TCP 连接进行响应。
- 有些放行规则已经过优化，以提高流量效率。

### 入侵防御
入侵防御模块保护计算机免受已知和零日漏洞攻击、SQL 注入攻击等。其功能包括虚拟修补、协议清理和应用程序控制。在启用入侵防御功能时，请注意以下配置选项：
- 桥接模式：活动数据包流直接通过网络引擎，所有规则在继续协议堆栈前应用到网络流量。
- 分接模式：系统复制活动数据包流并从主数据流中转移，不会修改活动数据流。

### NSX 安全标记
亚信安全服务器深度安全防护系统可以在检测到恶意威胁时将 NSX 安全标记应用于受保护的 VM。NSX 安全标记可与 NSX Service Composer 结合使用，以自动执行某些任务，如隔离受感染的 VM。

### 完整性监控
完整性监控允许您监控计算机上特定区域的更改，如已安装的软件、运行的服务等。启用完整性监控的基本步骤包括打开完整性监控、运行漏洞扫描、应用建议的完整性监控规则以及定期扫描更改。

### 日志审查
OSSEC 日志审查引擎集成于亚信安全服务器深度安全防护系统中，提供审查操作系统和应用程序生成的日志的功能。日志审查规则可以直接分配给计算机或作为安全配置文件的一部分。

### 检查日志审查规则
日志审查规则提供了一个框架，用于解析、分析、排序和关联事件。这些规则位于管理中心中的“策略 > 通用对象 > 规则 > 日志审查规则”下。

通过以上措施，您可以更有效地管理和保护您的网络安全。
