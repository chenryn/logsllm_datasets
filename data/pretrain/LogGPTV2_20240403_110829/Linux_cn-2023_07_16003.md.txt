### 互联网档案馆要求 GitHub 下架 DRM 去除工具

互联网档案馆已向 GitHub 提交移除请求，要求删除一个名为 DeGouRou 的工具。该工具专门用于规避行业标准的图书馆 TPM（可信平台模块），使读者能够保存他们借阅图书的无 DRM 副本。互联网档案馆及其他图书馆使用这些 TPM 来允许读者借阅、阅读和归还加密图书。值得注意的是，互联网档案馆并未获得版权所有者的授权即提交了此移除通知。GitHub 已满足其要求，但随后该软件代码在其他地方重新出现。

> **[消息来源：Torrent Freak](https://torrentfreak.com/internet-archive-targets-book-drm-removal-tool-with-dmca-takedown-230714/)**
>
> **老王点评**: 被出版商们紧追不舍的互联网档案馆，抽空打击了一下盗版工具。

![](/data/attachment/album/202307/15/173942hv6z5iwnew5nwjnc.jpg)

### Meta 公司将发布 LLaMA 的商业版本

LLaMA 是 Meta 开源的大型语言模型（LLM），于今年 2 月公开发布，并提供给研究人员和学者使用。它有 70 亿、130 亿、330 亿和 650 亿等不同参数规模。开源的 LLaMA 并未包含所使用的权重数据，但这些数据很快被“意外”泄露，并迅速引发了一系列基于它的开源创新。尽管开源人工智能模型已经存在，但 Meta 的 LLaMA 商业化仍是一个重要步骤，因为它比市场上现有的许多开源 LLM 更大，且来自全球最大的科技公司之一。

> **[消息来源：ZDNet](https://www.zdnet.com/article/meta-to-release-open-source-commercial-ai-model-to-compete-with-openai-and-google/)**
>
> **老王点评**: 先开源培养生态，再推出商业版本，这是一步好棋。

![](/data/attachment/album/202307/15/174007t5vc8l84cyycuy8c.jpg)

### GTK 对 macOS 的支持将降至“尽力而为”的状态

GNOME 的开发者正在考虑退役 macOS 上的 GLib 和 GTK 持续集成（CI）构建工具。除了 CI 硬件老化外，CI 维护者对 macOS 也没有太多经验，而 GTK/GLib 开发者对 macOS 开发也不特别感兴趣或熟悉。如果没有志愿者来处理 macOS 特定的回归问题，macOS CI 生成器可能会逐步淘汰，从而导致 GTK/GLib 在 macOS 上的支持降至“尽力而为”的状态，依赖下游打包者和消费者进行测试。

> **[消息来源：Phoronix](https://www.phoronix.com/news/GTK-macOS-Back-Seat)**
>
> **老王点评**: 苹果公司不感兴趣，用户也寥寥无几，开发者自然没有足够的动力和支持来进行。不过，还是有几个不错的 GNOME 软件可以在 macOS 上运行。
