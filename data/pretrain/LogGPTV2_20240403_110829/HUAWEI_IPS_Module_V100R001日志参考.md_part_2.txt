### 日志信息的格式

日志信息具有固定的输出格式，通过了解其格式，可以更好地理解日志信息。Syslog日志使用UDP协议进行传输，并通过端口514输出到日志主机中。

#### 日志信息输出格式
```
TimeStamp Hostname %% dd ModuleName/Severity/Brief (l): Description
```

示例：
```
2014-07-22 11:19:31 sysname %%01SHELL/4/LOGIN(l): access type:console user:admin login from con0
```

该日志产生时间为2014年07月22日11点19分31秒，主机名为`sysname`，由`SHELL`模块发出，级别为4，描述的内容是：`admin`用户从Console口登录。

#### 字段说明
| 编号 | 字段       | 字段含义         | 说明                                                                                           |
|------|------------|------------------|------------------------------------------------------------------------------------------------|
| 1    | TimeStamp  | 时间戳           | 日志产生的时间，缺省格式为“yyyy-mm-dd hh:mm:ss”。时间戳与主机名之间用一个空格隔开。          |
| 2    | Hostname   | 主机名称         | 本机的系统名。主机名与厂商标识之间用一个空格隔开。                                             |
| 3    | %%         | 厂商标识         | “%%”用来标识该日志是由华为的产品输出。                                                         |
| 4    | dd         | 版本号           | 用来标识该日志格式的版本，两位十进制数字，从01开始编号。                                       |
| 5    | ModuleName | 模块名称         | 标识产生日志的模块。模块名与级别之间用一个斜杠“/”隔开。                                      |
| 6    | Severity   | 日志级别         | 日志的级别，共分为8级，从0～7，数字越小，严重等级越高。级别与信息摘要之间用一个斜杠“/”隔开。 |
| 7    | Brief      | 描述符           | 该日志信息的内容概要。                                                                         |
| 8    | (l)        | 日志类型         | “(l)”用来标识该信息为日志信息。如果括号中为其他字母，代表该条信息不是日志信息。               |
| 9    | Description| 内容描述         | 各个模块实际向信息中心输出的字符串信息，详细描述该日志的具体内容。                             |

#### 日志信息级别说明
| 级别 | 程度       | 说明                                                                                                 |
|------|------------|------------------------------------------------------------------------------------------------------|
| 0    | Emergency  | 设备致命的异常，系统已经无法恢复正常，必须重启设备。如程序异常导致设备重启，内存在使用过程中被检测出错误等。 |
| 1    | Alert      | 设备重大的异常，需要立即采取措施。如设备内存占用率达到极限等。                                     |
| 2    | Critical   | 设备较大的异常，需要采取措施进行处理或原因分析。如设备内存占用率超过低界限，温度超过低温告警线，检测出错误的消息（消息是由本设备内部生成）等。 |
| 3    | Error      | 错误的操作或设备的异常流程，不会影响后续业务，但是需要关注和原因分析。如用户输入错误命令，用户密码错误，检测出错误协议报文（报文是由其他设备获得）等。 |
| 4    | Warning    | 设备的异常运转，可能引起业务故障的流程，需要引起注意。如用户关闭路由进程，检测出错误协议报文等。     |
| 5    | Notification| 用于设备正常运转的关键操作信息。                                                                   |
| 6    | Informational| 用于设备正常运转的一般性操作信息。                                                                  |
| 7    | Debugging  | 设备正常运转的一般性信息，用户无需关注。                                                             |

### 2. AAA

#### 2.1 AAA/4/CURRENT_USER_MODIFY_PASSWORD
- **日志信息**
  ```
  AAA/4/CURRENT_USER_MODIFY_PASSWORD(l): The current user failed to change the password. (UserName=[user-name])
  ```
- **日志含义**
  当前用户修改密码失败。
- **日志级别**
  - 级别名称: 4
  - 级别含义: 可能存在某种差错
- **日志参数**
  - 参数名称: `user-name`
  - 参数含义: AAA用户登录名。
- **可能原因**
  当前用户使用`current-user password-modify`命令修改自身密码。
- **处理步骤**
  - 步骤1: 请检查输入的初始密码是否正确，或者输入的新密码是否一致。
  - 结束

#### 2.2 AAA/3/LOCK
- **日志信息**
  ```
  AAA/3/LOCK(l): The user was locked out. (User Name=[user-name], Lock Time=[lock-time], Lock Reason=[lock-reason])
  ```
- **日志含义**
  管理员用户被锁定。锁定期间，该管理员用户不能再登录。
- **日志级别**
  - 级别名称: 3
  - 级别含义: 出现了错误
- **日志参数**
  - 参数名称: `user-name`, `lock-time`, `lock-reason`
  - 参数含义: 管理员用户登录名、锁定时长、锁定原因（参数值为：Password incorrect for 3 times（3次密码错误））
- **可能原因**
  同一管理员用户连续登录失败次数超过3次。导致登录失败的原因可能是密码输入错误。
- **处理步骤**
  - 步骤1: 管理员用户需等待锁定时间到期之后，重新登录。
  - 结束

#### 2.3 AAA/4/UNLOCK
- **日志信息**
  ```
  AAA/4/UNLOCK(l): The user was unlocked. (User Name=[user-name])
  ```
- **日志含义**
  解除管理员用户锁定状态。
- **日志级别**
  - 级别名称: 4
  - 级别含义: 可能存在某种差错
- **日志参数**
  - 参数名称: `user-name`
  - 参数含义: 管理员用户的登录名。
- **可能原因**
  管理员用户已被解除锁定。
- **处理步骤**
  - 步骤1: 正常运行信息，无需处理。
  - 结束

#### 2.4 AAA/6/CURRENT_USER_MODIFY_PASSWORD
- **日志信息**
  ```
  AAA/6/CURRENT_USER_MODIFY_PASSWORD(l): The current user successfully changed the password. (UserName=[user-name])
  ```

### 3. ADDR

#### 3.1 ADDR/3/AUTOCFGADDR_FAIL
- **日志信息**
  ```
  ADDR/3/AUTOCFGADDR_FAIL(l): Failed to auto-configure link local address while adding IPv6 address [ipv6-address] on interface [interface-name].
  ```
- **日志含义**
  自动配置地址失败。
- **日志参数**
  - 参数名称: `ipv6-address`, `interface-name`
  - 参数含义: 新增的IPv6地址、添加IPv6地址的接口名称
- **可能原因**
  为接口添加全球单播地址时，自动配置link local地址失败。
- **处理步骤**
  - 步骤1: 查看指定的接口是否存在。
  - 结束
