User: How to configure Nginx reverse proxy to implement domain name forwarding function?
Assistant: To configure Nginx reverse proxy and implement domain name forwarding function, you need to follow the steps below: 1. Install Nginx You need to install Nginx package, you can use the following command to install: ``` sudo apt-get update sudo apt-get install nginx ``` 2. Configure Nginx Create a configuration file named example.com under the /etc/nginx/sites-available directory, and add the following content to it: ``` server { listen 80; server_name example. com; location / { proxy_pass http://localhost:8000; proxy_redirect off; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; } } ``` Note: You need to replace `example.com` with what you want to forward domain name, replace `http://localhost:8000` with the IP address and port number of the page you want to forward. 3. Enable Nginx configuration To enable Nginx configuration, create a symbolic link between the sites-available and sites-enabled directories with the following command: ``` sudo ln -s /etc/nginx/sites-available/example.com /etc/ nginx/sites-enabled/ ``` 4. Restart Nginx Use the following command to restart the Nginx service so that your configuration changes will take effect: ``` sudo service nginx restart ``` 5. Test the configuration After completing the above steps, you can try Visit `example.com` in the browser, if the page you want to forward appears, the configuration is successful. Note: If your forwarded page uses HTTPS protocol, you need to add the configuration of SSL certificate and key in the Nginx configuration file.
