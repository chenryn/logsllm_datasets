### 黑客操纵一物联网僵尸网络 8 年，只为下载动漫视频

据报道，一个名为“Cereals”的僵尸网络自 2012 年首次被发现以来，一直活跃至 2015 年达到顶峰，当时其控制的设备数量超过 1 万个。尽管规模庞大，但大多数网络安全公司并未察觉其存在。然而，随着旧式 D-Link 设备的老化和用户更换产品，“Cereals”逐渐消失。此外，一种名为“Cr1ptT0r”的勒索病毒在 2019 年冬季破坏了许多 D-Link 系统中的“Cereals”恶意软件，进一步加速了该僵尸网络的衰落。现有证据表明，这个僵尸网络的创建者是一名德国人，名叫 Stefan，他制作“Cereals”似乎只是为了下载动画短片，并无犯罪意图。

来源：[凤凰网科技](https://www.cnbeta.com/articles/tech/976031.htm)

**硬核老王点评：这种僵尸网络的用途真是别开生面啊。**

### 成功在 iPhone 上运行的 postmarketOS 已经适配了 200 台移动设备

postmarketOS 目前已成功在大约 [200 台](https://tuxphones.com/static/counter/) 移动设备（包括手机和平板）上运行，这是 Linux 领域的一个重要成就。其中最引人注目的是 iPhone 7。通过开发者的逆向工程，iPhone 7 已成功引导了 postmarketOS，成为首款运行 Linux 的苹果智能手机。

来源：[cnBeta.COM](https://www.cnbeta.com/articles/tech/975969.htm)

**硬核老王点评：这种黑客精神才是真正有趣的黑客精神。**

### 有黑客从微软私有 GitHub 库中窃取了 63.2GB 的源代码

根据文件目录列表截图显示，泄露的数据涵盖了 Azure、Office 和部分 Windows runtimes。这一消息得到了数据泄露监控和防范服务机构 Under the Breach 推特账户的证实。需要注意的是，尽管此次泄露的源代码规模较大，但并不意味着黑客能够获取到微软的核心商业机密。微软在 GitHub 仓库中存储的内容通常是公开的，即使是在私人仓库中也是如此。

来源：[cnBeta.COM](https://www.cnbeta.com/articles/tech/976263.htm)

**硬核老王点评：究竟发生了什么，私有仓库也不安全了吗？**

### IBM 开源用于简化 AI 模型开发的 Elyra 工具包

IBM 在官方博客文章中宣布推出 Elyra，这是一组以开源 AI 为中心的扩展工具，专门针对 Jupyter Notebook 和新的 JupyterLab 用户界面。IBM 表示：“Jupyter Notebooks 现已成为数据科学和人工智能模型开发的开放标准。为了兑现我们对开源和 Jupyter 社区的承诺，IBM 推出了 Elyra。”

来源：[开源中国](https://www.oschina.net/news/115466/ibm-open-sources-elyra-ai-toolkit)

### 全国首起暗网平台案告破，注册会员逾 6 万

无锡市公安局召开新闻通气会，公布了该市警方破获的全国首起“暗网”平台案件详情。该案中，男子王某在“暗网”上搭建了一个传播淫秽物品牟利的网站，被查获时已有超过 6 万名注册会员。检法部门认为，专案组获取的证据确凿、指向清晰、证据链完备，足以认定犯罪事实，可以在“零口供”前提下对王某按涉嫌传播淫秽物品牟利罪予以提捕。

来源：[环球时报](https://www.cnbeta.com/articles/tech/976175.htm)

### 欧盟正试图修正其糟糕的 Cookie 同意政策

欧盟的 Cookie 同意政策自 2018 年作为《一般数据保护条例》（GDPR）的一部分出台以来，一直是许多互联网用户的困扰。它要求用户在首次访问网站时同意被跟踪，但措辞往往具有误导性或无法拒绝。为了解决这些问题，欧盟本周发布了最新的指导方针，禁止对这些政策进行最糟糕的解释，使 cookie 同意更符合用户的意愿。

来源：[cnBeta.COM](https://www.cnbeta.com/articles/tech/976085.htm)

### 伦敦帝国学院的新冠模型代码存在严重 bug

在新冠疫情爆发初期，伦敦帝国学院的 MRC 全球传染病分析中心多次做出广泛报道的预测，这些预测被认为对政府的公共政策决策产生了影响。然而，科学家编写的代码可能存在严重问题，导致基于这些代码的研究也存在问题。MRC 全球传染病分析中心发布在 GitHub 上的 COVID-19 CovidSim 模型就被发现存在严重缺陷，有软件开发者呼吁撤回所有基于这些代码的论文。科学家对此感到被冒犯。

来源：[solidot](https://www.solidot.org/story?sid=64292)

**硬核老王点评：来自程序员的鄙视。**

### Facebook iOS SDK 导致众多应用崩溃

开发者在 Facebook iOS SDK 的 GitHub 问题页面报告称，多个版本的 SDK 导致了大量应用崩溃。Facebook SDK 被广泛用于身份验证和广告展示。受影响的流行应用包括 Spotify、TikTok、Pinterest、Winno 等。据说受此影响，连统计应用崩溃的 Firebase Crashlytics 服务都崩溃了十几个小时。

来源：[solidot](https://www.solidot.org/story?sid=64285)

**硬核老王点评：除非绝对必要，不要轻易使用第三方 SDK。**
