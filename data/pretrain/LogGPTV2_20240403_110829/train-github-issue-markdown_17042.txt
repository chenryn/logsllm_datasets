Bug.

**What is the current behavior?**
When editing the default value `2.34` to another value, such as `3.45`, the cursor starts to jump around in the input field as soon as a dot (.) is inserted. You can observe this issue in the provided [JSFiddle](<JSFiddle link here>).

**Which versions of React, and which browser/OS are affected by this issue? Did this work in previous versions of React?**
- **React Versions:** This issue occurs in React `v15.3.2` and `v15.3.1`.
- **Browser/OS:** The problem is observed in Google Chrome `v53` on both Windows and macOS.
- **Previous Versions:** I tested this on an older JSBin with React `v0.13.2`, and it worked correctly.
- **Other Browsers:** The issue does not occur in Mozilla Firefox, where the input behaves as expected.
