### A2 Co-Owned Websites

Websites that list the same contact information in their imprint are grouped as co-owned and notified in a single message. This grouping reduces the number of messages sent and models the assumption that these websites are related and may be remediated simultaneously. However, this is a first approximation of the real operator structure, as different owners may use the same web design agency, which might check and repair other websites in their portfolio upon notification. We have some indicators that this occurs, but lack a method to quantify its impact on the measurement.

Intuitively, if an owner is notified about non-compliance for multiple websites, they would likely ensure all websites are compliant (or remain non-compliant if they choose to ignore the notification). Indeed, 77 out of 88 owners with more than one website (87.5%) were either completely compliant or non-compliant by the end of the study timeframe (40 compliant, 37 non-compliant, excluding the control group). Of the 40 compliant owners, 34 (85%) made all websites compliant within a maximum of 2 days between the first and last remediation, while the remaining six groups took between 4 and 35 days to remediate the other websites.

### A3 Google Analytics Misconfigurations

When activating IP Anonymization (AIP) for Google Analytics, operators can encounter several pitfalls:
1. **Explicit Activation**: AIP must be explicitly activated. Since May 2010, Google has implemented AIP, so operators who included GA earlier must be aware of this addition and update their website.
2. **Activation Method**: The method to enable AIP depends on how GA is included. For example, for inclusion via Google Tag Manager, the option must be set in Google’s web interface, while adding the Analytics library via a `<script>` tag requires additional JavaScript code.
3. **Library Versions**: There are several versions of the Analytics library (analytics.js and ga.js), each requiring different approaches to activate AIP. Loaders like gtag.js add further complexity.
4. **Case Sensitivity**: The option to enable AIP is case-sensitive and spelled “anonymizeIp” (except in gtag.js, where it is “anonymize_ip”). Misspelling does not raise an error but silently ignores the option.
5. **Timing**: The “anonymizeIp” option must be set after configuring the tracking ID but before any requests to Google are issued. There is no warning if non-anonymized requests are issued due to incorrect timing.
6. **Multiple Trackers**: If an operator defines several GA trackers on a single page, AIP must be enabled separately for each tracker.

An example of different misconfigurations is shown in Listing 1.

```javascript
(function(i, s, o, g, r, a, m) {
    i['GoogleAnalyticsObject'] = r;
    i[r] = i[r] || function() {
        (i[r].q = i[r].q || []).push(arguments)
    }, i[r].l = 1 * new Date();
    a = s.createElement(o),
    m = s.getElementsByTagName(o)[0];
    a.async = 1;
    a.src = g;
    m.parentNode.insertBefore(a, m)
})(window, document, 'script', 'https://www.google-analytics.com/analytics.js', 'ga');

ga('set', 'anonymizeIp', true);
ga('create', 'UA-XXXXX-Y', 'auto'); // Configure the tracking ID
ga('set', 'anonymizeIP', true); // Error: Must be spelled 'anonymizeIp'
```
