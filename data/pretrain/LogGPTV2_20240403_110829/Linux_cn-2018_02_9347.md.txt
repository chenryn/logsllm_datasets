Konstantin Ryabitsev 将为你解释哪些 Linux 内核版本可以被视为“稳定版”，以及如何根据你的需求选择合适的内核版本。

每当 Linus Torvalds 发布一个新的主线 Linux 内核版本时，人们往往会陷入困惑：究竟哪个版本才是最新的“稳定版”？是新发布的 X.Y 版本，还是之前的 X.Y-1.Z 版本？最新发布的内核版本是否过于新了？你应该继续使用旧版本吗？

[kernel.org](https://www.kernel.org/) 的网页信息并没有完全解答这些问题。目前，在该页面顶部显示的是最新稳定版内核为 4.15，但在表格下方，4.14.16 也被标记为“稳定版”，而 4.15 被标记为“主线版本”。这确实令人困惑，对吧？

不幸的是，这个问题并不容易回答。我们在讨论中使用“稳定”一词有两个不同的含义：一是指最初发布的 Git 树的名字，二是表示这个内核可以被视为“稳定版”，适用于生产系统。

由于 Git 的分布式特性，Linux 的开发工作在许多[不同的分支仓库](https://git.kernel.org/pub/scm/linux/kernel/git/)中进行。所有 bug 修复和新功能首先由各个子系统维护者收集和准备，然后提交给 Linus Torvalds，再由他合并到[自己的 Linux 树](https://git.kernel.org/torvalds/c/v4.15)中。他的 Git 树被认为是 Git 仓库的“主干”。我们称这个树为“主线” Linux 树。

### 候选发布版（RC）

在每个新内核版本正式发布之前，都会经过几轮“候选发布”（Release Candidate, RC），供开发者测试并打磨这些新功能。基于这几轮测试的反馈，Linus 决定最终版本是否已经准备好。通常会有 7 个每周预发布版本，但有时会达到 -rc8 甚至更多。当 Linus 确信新内核已经没有问题时，就会制作最终发行版，我们称之为“稳定版”，表示它不再是“候选发布版”。

### Bug 修复

像任何复杂的软件一样，任何新的 Linux 内核版本都包含 bug，并且需要修复。Linux 内核的 bug 修复规则非常简单：所有修复必须首先进入 Linus 的树。一旦主线仓库中的 bug 被修复后，它们会被应用到内核开发社区仍在维护的已发布内核中。在被考虑回迁到已发布的稳定版本之前，所有的 bug 修复必须满足[一套重要的标准](https://www.kernel.org/doc/html/latest/process/stable-kernel-rules.html)——其中之一是，它们“必须已经存在于 Linus 的树中”。这是一个[独立的 Git 仓库](https://git.kernel.org/stable/linux-stable/c/v4.14.16)，用于回迁 bug 修复，被称为“稳定”树——因为它跟踪以前发布的稳定内核。这个树由 Greg Kroah-Hartman 维护。

### 最新的稳定内核

因此，任何时候访问 kernel.org 网站查看最新的稳定内核时，你应该关注那个标有“最新稳定内核”的大黄色按钮。

![](/data/attachment/album/201802/14/215257g9bbamoimqqm5mmi.png)

但是，你可能会惊讶地发现——4.15 和 4.14.16 都是稳定版本，那么哪一个更“稳定”呢？有些人不愿意使用“.0”版本的内核，因为他们认为这个版本不够“稳定”，直到最新的“.1”版本发布。这种观点很难证明或反驳，两者都有其合理性和反对理由，具体选择取决于个人偏好。

一方面，任何进入稳定树的发行版都必须首先被接受进入主线内核版本，然后回迁到已发布的版本中。这意味着主线内核版本相比稳定树中的版本总是包含最新的 bug 修复。因此，如果你想使用的发行版包含最少的“已知 bug”，那么使用“.0”主线发行版是最佳选择。

另一方面，主线版本增加了许多酷炫的新功能——而这些新功能也带来了数量未知的“新 bug”，这些“新 bug”在旧的稳定版中不会存在。新的、未知的 bug 是否比旧的、已知但尚未修复的 bug 更令人担忧？这取决于你的选择。需要注意的是，许多 bug 修复只在主线版本中进行了彻底测试。当补丁回迁到旧内核时，它们可能工作得很好，但很少进行与旧内核的集成测试。通常假设，“以前的稳定版本”足够接近当前的主线版本，可以用于生产系统。实际上也是如此，这也说明了为什么选择“哪个内核版本更稳定”是一件非常困难的事情。

因此，从根本上说，我们没有定量或定性的方法明确告诉你哪个内核版本更稳定——4.15 还是 4.14.16？我们只能告诉你，它们具有“不同的稳定性”（这个答案可能没有帮到你，但至少你明白了这些版本的区别）。
