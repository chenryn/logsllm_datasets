User: 请描述HTTP请求和响应的数据结构和格式。
Assistant: HTTP请求和响应是由一个请求/响应行、请求/响应头和消息主体构成的。它们使用ASCII码文本格式进行通信。

HTTP请求格式:

```
请求行
首部行1
首部行2
...
首部行n

消息主体
```

请求行包含方法、URI和协议版本，各个部分之间以空格分隔。例如：

```
GET /index.html HTTP/1.1
```

HTTP请求方法有GET、POST、PUT、DELETE和HEAD等。请求方法用于指定请求处理的类型和操作。

请求头部包括各种请求的元数据，例如Host、Content-Length、User-Agent等等。

消息主体包含请求的实际内容。例如，对于POST请求，消息主体通常包含要提交的表单数据。

HTTP响应格式:

```
响应行
首部行1
首部行2
...
首部行n

消息主体
```

响应行包括协议版本、状态码和原因短语，各个部分之间以空格分隔。例如：

```
HTTP/1.1 200 OK
```

响应状态码表示相应的状态。例如，200表示成功，404表示未找到请求的资源。

响应头部包含有关响应的元数据，例如Content-Length、Content-Type、Expires等等。

消息主体包含响应的实际内容，例如HTML网页或JSON数据。
