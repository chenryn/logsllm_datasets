User: How to configure Nginx to reverse proxy multiple web applications
Assistant: 1. Install Nginx To install Nginx on the Ubuntu system, you can use the following command: sudo apt-get update sudo apt-get install nginx 2. Create multiple web applications Suppose we have two web applications, namely WebApp1 and WebApp2. Each web application uses a different port number: WebApp1: http://localhost:8000 WebApp2: http://localhost:9000 3. Configure Nginx In the configuration file of Nginx, we need to configure a reverse port number for each web application. To the proxy: sudo nano /etc/nginx/sites-available/default Add the following to the configuration file: server { listen 80; server_name localhost; location /webapp1/ { proxy_pass http://localhost:8000/; proxy_set_header Host $host ; proxy_set_header x-real-IP $ remote_addr; proxy_set_header X-FORWARDED-FOR $ proxy_add_X_FORWARDED_FOR;} location / webapp2 / {proxy_pas s http: // localhost: 9000/; proxy_Set_header host $ host; proxy_set_header x-real-IP $ remote_addr; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; } } The above configuration sets Nginx to listen on port 80, and configures a reverse proxy for each web application. 4. Reload Nginx Save and close the configuration file, then reload the Nginx configuration: sudo systemctl reload nginx Now, each web application can be accessed through the following URL: WebApp1: http://localhost/webapp1/ WebApp2: http://localhost /webapp2/ Note: In web applications, relative paths should be used instead of absolute paths. For example, in WebApp1, all links and resource paths should be relative to "/webapp1/", not "/".
