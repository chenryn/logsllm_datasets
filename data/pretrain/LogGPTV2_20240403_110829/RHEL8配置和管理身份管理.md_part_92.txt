根据安全性和访问控制级别，IdM 将密码库分类为以下类型：
::: variablelist
[标准密码库]{.term}
:   Vault 所有者和密码库成员可以归档和检索机密，而不必使用密码或密钥。
[对称密码库]{.term}
:   库中的机密通过对称密钥进行保护。Vault
    所有者和成员可以归档和检索机密，但必须提供 vault 密码。
[非对称密码库]{.term}
:   库中的机密通过非对称密钥进行保护。用户使用公钥存档机密，并使用私钥检索该机密。Vault
    成员只能存档机密，而 vault 所有者则可同时执行归档和检索机密。
:::

# []{#using-idm-user-vaults-storing-and-retrieving-secrets_configuring-and-managing-idm.html#storing-a-secret-in-a-user-vault_using-idm-user-vaults-storing-and-retrieving-secrets}在用户密码库中存储 secret {.title}
:::
本节介绍用户可以如何使用一个或多个私有密码库创建 vault
容器，从而安全地存储具有敏感信息的文件。在以下流程中使用的示例中，[**idm_user**]{.strong}
用户创建标准类型的密码库。标准密码库类型确保无需 [**idm_user**]{.strong}
在访问该文件时进行身份验证。[**idm_user**]{.strong} 能够从用户登录的任何
IdM 客户端检索 文件。
在此过程中：
::: itemizedlist
-   [**idm_user**]{.strong} 是想要创建密码库的用户。
-   [**my_vault**]{.strong} 是用于存储用户证书的库。
-   vault 类型是 `标准的`{.literal}，因此访问存档证书不要求用户提供
    vault 密码。
-   [**secret.txt**]{.strong} 是包含用户希望在密码库中存储的证书的文件。
:::
::: itemizedlist
**先决条件**
-   您知道 [**idm_user**]{.strong} 的密码。
-   您已登录到属于 IdM 客户端的主机。
:::
::: orderedlist
**流程**
1.  获取 `idm_user`{.literal} 的 Kerberos 票据授予 ticket(TGT)：
    ``` literallayout
    $ kinit idm_user
    ```
2.  使用 `ipa vault-add`{.literal} 命令和 `--type 标准`{.literal}
    选项来创建标准密码库：
    ``` literallayout
    $ ipa vault-add my_vault --type standard
    ----------------------
    Added vault "my_vault"
    ----------------------
      Vault name: my_vault
      Type: standard
      Owner users: idm_user
      Vault user: idm_user
    ```
    ::: {.important style="margin-left: 0.5in; margin-right: 0.5in;"}
    ### 重要 {.title}
    确保用户的第一个用户库由同一用户创建。为用户创建第一个密码库也会创建用户的
    vault 容器。创建的代理变为 vault 容器的所有者。
    例如，如果其他用户（如 `admin`{.literal} ）为 `user1`{.literal}
    创建第一个用户库，则用户的 vault 容器所有者也是
    `admin`{.literal}，并且 `user1`{.literal}
    无法访问用户密码库或创建新的用户库。
    :::
3.  使用 `ipa vault-archive`{.literal} 命令及 `--in`{.literal} 选项将
    `secret.txt`{.literal} 文件归档到密码库中：
    ``` literallayout
    $ ipa vault-archive my_vault --in secret.txt
    -----------------------------------
    Archived data into vault "my_vault"
    -----------------------------------
    ```
:::

# []{#using-idm-user-vaults-storing-and-retrieving-secrets_configuring-and-managing-idm.html#retrieving-a-secret-from-a-user-vault_using-idm-user-vaults-storing-and-retrieving-secrets}从用户密码库检索 secret {.title}

作为身份管理(IdM)，您可以从用户私有密码库中检索机密到您已登录的任何 IdM
客户端。
本节演示如何以名为 [**idm_user 的 IdM 用户从名为 my\_**]{.strong}
[**vault**]{.strong} 的用户私有 vault 中检索机密到
[**idm_client.idm.example.com**]{.strong}。
::: itemizedlist
**先决条件**
-   [**idm_user**]{.strong} 是 [**my_vault**]{.strong} 的所有者。
-   [**idm_user**]{.strong} 已在
    [密码库中存档了机密](#using-idm-user-vaults-storing-and-retrieving-secrets_configuring-and-managing-idm.html#storing-a-secret-in-a-user-vault_using-idm-user-vaults-storing-and-retrieving-secrets "在用户密码库中存储 secret"){.link}。
-   [**my_vault**]{.strong} 是一个标准密码库，这意味着
    [**idm_user**]{.strong} 不必输入任何密码才能访问密码库的内容。
:::
::: orderedlist
**流程**
1.  以 [**idm\_ [**user 用户身份通过 SSH 连接到 idm\_**]{.strong}
    client**]{.strong} ：
    ``` literallayout
    $ ssh PI:EMAIL
    ```
2.  以 `idm_user`{.literal} 身份登录：
    ``` literallayout
    $ kinit user
    ```
3.  使用 `ipa vault-retrieve --out`{.literal} 命令及 `--out`{.literal}
    选项，以检索密码库的内容并将其保存到 `secret_exported.txt`{.literal}
    文件中。
