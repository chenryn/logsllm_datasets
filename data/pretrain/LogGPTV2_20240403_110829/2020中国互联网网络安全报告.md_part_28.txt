### 会员中心、CMS、CRM、OA等系统漏洞

若依后台管理系统存在未授权访问和文件上传漏洞，攻击者可利用这些漏洞获取服务器控制权。

### Microsoft HEVC Video Extensions 远程代码执行漏洞

2020年11月22日，国家信息安全漏洞共享平台（CNVD）收录了Microsoft HEVC Video Extensions远程代码执行漏洞（CNVD-2020-65153，对应CVE-2020-17106）。目前，厂商已发布升级版本，修复了此漏洞。

**背景信息：**
Microsoft HEVC Video Extensions是美国微软公司的一个视频扩展应用程序，使计算机和设备能够读取高效视频编码或HEVC视频。

**漏洞描述：**
该应用存在高危远程代码执行漏洞，攻击者可利用该漏洞影响系统的可用性和完整性。

### Trend Micro InterScan Web Security Virtual Appliance 命令执行漏洞

2020年12月24日，CNVD收录了Trend Micro InterScan Web Security Virtual Appliance（IWSVA）命令执行漏洞（CNVD-2020-74861，对应CVE-2020-35114）。目前，厂商已发布升级版本，修复了此漏洞。

**背景信息：**
Trend Micro InterScan Web Security Virtual Appliance是美国趋势科技（Trend Micro）公司的一款针对基于Web方式的威胁，为企业网络提供动态集成安全保护的Web安全网关。

**漏洞描述：**
Trend Micro InterScan Web Security Virtual Appliance 6.5 SP2存在命令执行漏洞。攻击者可利用该漏洞以root用户身份执行代码。

### Mozilla Firefox 内存破坏漏洞

2020年12月29日，CNVD收录了Mozilla Firefox内存破坏漏洞（CNVD-2020-74861，对应CVE-2020-35114）。目前，厂商已发布升级版本，修复了此漏洞。

**背景信息：**
Mozilla Firefox是一款开源Web浏览器。

**漏洞描述：**
Firefox存在内存破坏漏洞，远程攻击者可利用该漏洞提交特殊的Web请求，诱使用户解析，从而导致应用程序崩溃或以应用程序上下文执行任意代码。

### 网络安全事件接收与处置情况

为了能够及时响应并处置互联网上发生的攻击事件，CNCERT/CC通过热线电话、传真、电子邮件、网站等多种公开渠道接收公众的网络安全事件报告。对于其中影响互联网运行安全、波及较大范围互联网用户或涉及政府部门和重要信息系统的事件，CNCERT/CC积极协调基础电信企业、域名注册服务机构以及应急服务支撑单位进行处置。

#### 事件接收情况

2020年，CNCERT/CC共接收境内外报告的网络安全事件103,109起，较2019年的107,801起下降4.4%。其中，我国境内报告的网络安全事件102,337起，较2019年的107,211起下降4.5%；境外报告的网络安全事件772起，较2019年的590起上升30.8%。

### 网络安全组织发展情况

#### CNCERT/CC 应急服务支撑单位

互联网作为重要的信息基础设施，已经融入社会生活的方方面面，深刻改变着人们的生产和生活方式。网络安全和信息化是事关国家安全和国家发展、事关广大人民群众工作生活的重大战略问题。没有网络安全就没有国家安全，就没有经济社会稳定运行，广大人民群众利益也难以得到保障。加强网络安全应急技术体系建设，培养建立强大的网络安全应急队伍，对提升网络安全事件应急处置能力、增强关键信息基础设施和国家网络安全保障能力具有重要意义。

网络安全应急服务支撑单位（以下简称“支撑单位”）自2004年启动支撑工作以来，在CNCERT/CC统一指导和协调下积极参与国家网络安全应急工作，为推动全国网络安全应急能力提升做出了积极贡献。为进一步加强网络安全技术支撑体系建设，面向国家网络安全保障需要，按照开放、合作、自愿、共享的发展理念，2019年4月，CNCERT/CC组织开展了第八届CNCERT/CC网络安全应急服务支撑单位遴选工作，得到了互联网行业和网络安全行业相关单位的大力支持和积极响应。经自主申报、提名推荐、专家评审、结果公示等环节，CNCERT/CC最终确认103家单位为第八届CNCERT/CC网络安全应急服务支撑单位，其中国家级11家，省级69家，反网络诈骗领域5家，工业控制领域18家，有效日期为2019年7月1日至2021年6月30日。

2020年8月至9月，CNCERT/CC组织开展了第八届支撑单位考核工作，对第八届支撑单位自2019年7月1日至2020年6月30日期间的工作情况进行考核，内容涉及安全漏洞信息报送、威胁数据报送、网络安全信息报送（含重大信息报送）、应急响应和专项支撑以及其他工作。根据实际工作表现及考核情况，CNCERT/CC决定取消3家单位的省级支撑单位称号。在考核期间，CNCERT/CC同时还组织各分中心开展了支撑单位增改选工作，旨在根据实际工作需求，对本地支撑力量十分薄弱的省份予以补充，进一步完善网络安全应急体系。经提名推荐、筛选和审议，CNCERT/CC决定增选5家单位为第八届省级支撑单位，增选1家单位为第八届工业控制领域支撑单位，有效日期为2020年7月1日至2021年6月30日。
