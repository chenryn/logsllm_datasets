对系统的影响
接口不能正常转发报文。
可能原因
原因1：The interface is Down.（接口为DOWN状态。）
原因2：The IPv6 address is unavailable.（接口IPv6地址不可用。）
原因3：The IPv6 address is available.（接口IPv6地址可用。）
原因4：PPP IPCP6 is Down.（PPP协议IPCP6 Down。）
原因5：IPv6 is disabled or the interface is deleted.（接口被删除。）
处理步骤
l 原因1：The interface is Down.
a. 请查看物理链接是否正常（包括网线、光模块等硬件是否松动或脱落）。
n 如果物理链路工作不正常，可以重新正确连接物理线路，然后检查告警
是否被清除。如果未被清除，则请执行步骤2。
n 如果物理链路工作正常，则请执行步骤2。
b. 请收集告警信息和配置信息，并联系技术支持人员。
l 原因2：The IPv6 address is unavailable.
a. 请在接口视图下执行display this命令，查看接口下是否配置了ipv6 enable (接
口视图)命令。
n 如果接口下配置了ipv6 enable (接口视图)命令，则请执行步骤2。
n 如果接口下没有配置ipv6 enable (接口视图)命令，则在接口视图下执行
ipv6 enable (接口视图)命令，使能接口的IPv6功能。然后检查告警是否被
清除。如果未被清除，则请执行步骤2。
b. 请收集告警信息和配置信息，并联系技术支持人员。
l 原因3：PPP IPCP6 is Down.
a. 执行命令display interface检查对端设备PPP协议是否正常。
n 如果对端设备PPP协议工作正常，则继续执行命令display interface，检
查设备的物理链路（包括中间设备）的状态，包括丢包、错包情况等。
然后检查告警是否被清除。如果未被清除，则请执行步骤2。
n 如果对端设备PPP协议工作不正常，则联系对端设备的工程师进行检查。
然后检查告警是否被清除。如果未被清除，则请执行步骤2。
b. 请收集告警信息和配置信息，并联系技术支持人员。
l 原因4：IPv6 is disabled or the interface is deleted.
通常是用户删除某逻辑接口，正常运行信息，无需处理。
