### 问题描述
我正在尝试使用以下参考文档在本地站点上创建重定向，但没有成功：

[Microsoft IIS 6.0 重定向配置指南](http://www.microsoft.com/technet/prodtechnol/WindowsServer2003/Library/IIS/6b855a7a-0884-4508-ba95-079f38c77017.mspx?mfr=true)

我希望将本地站点上的绝对链接（指向在线地址）重定向到本地站点。例如：

**原始绝对链接：**
```
http://online.com/products
```

**期望的重定向结果：**
```
http://offline/products
```

我希望能够保留域名之后的所有路径，并将其附加到本地服务器名称上，以便点击链接时能够重定向到本地站点而不是在线版本。

我已经尝试了 `[http://offline$S]`，但这并没有像我预期的那样附加 `/products` 路径。

### 问题分析
在这种情况下，IIS 重定向可能无法满足您的需求。对于这些绝对链接，您的离线 IIS 服务器实际上完全不在处理链中。如果查看离线 IIS 的日志，您会发现这些请求根本没有到达该服务器。浏览器会直接尝试连接 `online.com` 并发送请求（通过任何代理服务器）。因此，离线 IIS 服务器从未有机会处理这些请求，也就无法触发重定向。

为了实现这种重定向，您需要采用更复杂的方法，例如 DNS 或内部代理服务器。

### 解决方案建议

#### 1. 使用 DNS 重定向
您可以配置本地 DNS 服务器，将 `online.com` 解析为本地 IP 地址。这样，所有对 `http://online.com` 的请求都会被重定向到本地服务器。

**步骤：**
1. 在本地 DNS 服务器上添加一个 A 记录，将 `online.com` 指向本地服务器的 IP 地址。
2. 确保您的开发环境使用的是这个本地 DNS 服务器。

#### 2. 使用内部代理服务器
您可以设置一个内部代理服务器来拦截并重定向这些请求。

**步骤：**
1. 配置一个内部代理服务器（如 Squid 或 Nginx）。
2. 在代理服务器上设置规则，将所有对 `http://online.com` 的请求重定向到 `http://offline`。
3. 确保您的开发环境配置为使用此内部代理服务器。

### 结论
通过上述方法之一，您可以实现将绝对链接从在线地址重定向到本地地址的需求。请根据您的具体环境选择最适合的解决方案。
