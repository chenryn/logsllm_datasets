### LANE-3-LEC_LISTEN
**错误：** `[chars]` 监听失败。LANE 客户端无法注册其地址以接收来自该地址的呼叫。
**建议操作：** 重新配置客户端的 ATM 地址。可能与其他具有相同 ATM 地址的组件发生冲突。

### LANE-3-LEC_JOIN
**错误：** `[chars]` 加入失败。LES 返回 `[int]`。LANE 客户端的加入请求被拒绝。
**建议操作：** 这通常是因为用户配置错误，例如 LANE 客户端尝试加入错误的 ELAN 或 LES 中未配置 ELAN，或者 LECS 验证失败等。

### LANE-3-LEC_REGISTER
**错误：** `[chars]` 注册失败。LES 返回 `[int]`。
**描述：** 尝试向 LES 注册 MAC-ATM 地址绑定失败。

### LANE-3-LEC_RD_REGISTER
**错误：** `[chars]` 路由描述符 `[hex]` 的 RD 注册失败。发生了软件或硬件错误。
**建议操作：** 复制出现的错误消息并报告给技术支持代表。

### LANE-3-LEC_CFG
**错误：** `[chars]` CFG_REQ 失败。LECS 返回 `[int]`。
**描述：** 主 LECS 拒绝了 LEC 的配置请求。

### LANE-3-LEC_BAD_CTL_DIST_TYPE
**错误：** `[chars]` CTL DIST blli 类型错误。
**描述：** 入站电路上指定了错误的 BLLI 值。

### LANE-3-LEC_WRONG_ELAN_ID
**错误：** `[chars]: elan id 不匹配: LECS 指示 %u, LES 指示 %u`。LES 返回的 ELAN-ID 与 LECS 或通过 CLI 给出的值不匹配。
**建议操作：** 检查 LECS 数据库和 LES 中配置的 ELAN ID。这种不匹配可能是由于配置错误引起的。

### LANE-3-NAMECHG
**错误：** 已有一个使用另一个 ELAN 名称的 LANE 客户端正在运行。同时在 LANE 客户端和 LANE 服务器/BUS 的配置中指定了模拟 LAN 名称时，不能仅通过重新输入其中一个命令来更改名称。
**建议操作：** 使用 `no lane server-bus` 命令删除服务器/总线，或使用 `no lane client` 命令删除客户端。然后可以重新输入另一条命令，并使用新的模拟 LAN 名称。最后，使用新名称重新输入已删除的命令。

### LANE-4-LSV_LECID
**警告：** `[chars] elan [chars] LECIDs 耗尽`。指定的 LANE 服务器已有 65279 个客户端，不能再添加更多。
**建议操作：** 更改配置，避免在此服务器上拥有超过 65279 个 LANE 客户端。

### LANE-3-LSV_CRASH
**错误：** `[chars] elan [chars] 不可恢复的错误 [chars]`。发生了严重的 LES/BUS 内部错误。

### LANE-4-LSV_ADDR
**警告：** `[chars] elan [chars] 无法注册 LES/BUS ATM 地址`。LANE 服务器或 BUS 无法向 ILMI 注册其地址。
**建议操作：** 尝试调查原因。可能的原因包括：用户定义的覆盖地址不符合交换机前缀，或者与之注册的 ATM 交换机存在问题。

### LANE-3-BADILMI
**错误：** `意外的 ILMI 事件 [int]`。发生了内部错误。
**建议操作：** 此错误不应发生。下载系统错误日志，然后联系您的技术支持代表，并提供日志、错误消息和路由器配置。

### LANE-3-NOREGILMI
**错误：** `[chars] [chars] 无法向 ILMI 注册 [char]j`。尝试将给定的 ATM 地址注册到 ATM 交换机失败。此故障可能是由于 ATM 交换机关闭或配置错误。将重试注册。
**建议操作：** 如果多次尝试后情况仍然存在且交换机正常运行，则应检查交换机和路由器的配置。可能的原因是注册的 ATM 地址与交换机中配置的参数不兼容。请更正配置。
