概述
名词说明
| 名称 | 说明 |
|---|---|
| 流程挖掘 |对ERP 等业务系统流程日志进行模型发现、合规检查、瓶颈分析和预测的一系列统计分析操作。 |
| 流程 |一类相同目的的业务场景，称为 XX 流程。如：采购流程、发货流程、报销流程等。 |
| 案例 |具有 相同的唯一 ID的多条日志，共同代表了一笔具体的流程交易，称为案例(case)，也可以称为交易(transaction)。在 XES 国标中命名为case:concept:name。 |
| 活动 |案例中独立完成的某一个操作，称为活动(activity)，也可以称为步骤(step)。在 XES 国标中命名为 concept:name。 |
| 路径 |活动之间的流向关系，成为路径(path)。路径可能形成环路。 || 路径 |活动之间的流向关系，成为路径(path)。路径可能形成环路。 |
| 属性 |流程日志中其他可用于分析的信息，通常分为案例属性和日志属性，案例属性在单次案例中取值唯一。最常见的属性一般有订单金额、申请人、部门等。在 XES 国标中，约定有org:resource、org:role和org:group等，可用于流程并发度和组织覆盖率分析等。 |
| 变体 |当流程的多个案例的活动和路径一致时，可合并为一个变体(variant)。通常将发生次数最多的变体，默认为该流程的标准变体。 |
| 资源 |在业务流程中通常泛指完成流程活动的人力资源、或自动化机器等。 |
| 组织 |在业务流程中通常表示完成同类流程活动的资源归属单元，如部门等。 |
| 社交图谱 |在业务流程挖掘领域中，社交图谱指资源之间因为流程活动所形成的的路径关系。 || 合规性检查 |将日志中发生的活动与流程模型中设定的活动进行比较，验证日志中发生的活动是否遵循事先定义好的业务流程规范。 |
| 根因分析 |对未通过合规性检查的日志活动，通过追溯数据和事件的来源和路径，识别导致这些事件发生的根本原因。 |
| BPMN |Business Process Modeling Notation的缩写，一种业务流程的可视化建模规范。通过流、连接、泳道、人工信息等 4 类图元组合成业务流程图。 |
