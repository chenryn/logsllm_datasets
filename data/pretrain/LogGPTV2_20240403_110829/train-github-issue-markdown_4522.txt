I receive a `Neo.DatabaseError.General.UnknownError` upon trying to MERGE two
particular nodes. Nothing seems special about these nodes. I'll include as
much info as I can below, but I doubt this will be particularly helpful. Feel
free to contact me to get more info.
Also mentioned in the Neo4j-users slack.
As background--I had a problem creating an index on `:action_attribute(val)`
because of the (4096 char?) text length limit. I worked around this by
specifying the default index provider, then recreating the index.
`dbms.index.default_schema_provider=lucene+native-1.0`
But perhaps this left the database somehow confused about this index--which is
used in the query?
**Neo4j Version:** Neo4j/3.4.0-rc01  
**Operating System:** Windows Server 2012 R2
### Steps to reproduce
I receive the error when I run this command:
    match (p:part)
    with p
    match (aa:action_attribute {name:"part_number_txt", val: p.id})
    with aa, p 
    skip 212945 limit 1
    MERGE (aa)-[:IS_PART_NUMBER]->(p)
However, this command (note -1 decrement) works.
    match (p:part)
    with p
    match (aa:action_attribute {name:"part_number_txt", val: p.id})
    with aa, p 
    skip 212944 limit 1
    MERGE (aa)-[:IS_PART_NUMBER]->(p)
Nothing seems particularly special about the aa or p nodes at position 212945.
