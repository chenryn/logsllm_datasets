##### Detailed description
cap=cv2.VideoCapture (this is python 3 code!)  
cap.get(cv2.CAP_PROP_POS_MSEC) does not work, if not cap.read has been called
before.  
The current version does not seem to read the position in milliseconds, if a
cap.read() has not been issued before the query. It works in Version 4.1.1-1
##### Steps to reproduce
This python3 code shows the problem:
    import cv2
    cap=cv2.VideoCapture("set/your/file/here")
    w = cap.get(cv2.CAP_PROP_FRAME_WIDTH)
    cnt = cap.get(cv2.CAP_PROP_FRAME_COUNT)
    print("Opencv width %d frames %d"%(w,cnt))
    cap.read()
    ts= cap.get(cv2.CAP_PROP_POS_MSEC)
    print("ts @ frame 0 after read %.3f"%ts)
    #set pos
    cap.set(cv2.CAP_PROP_POS_FRAMES,5)
    ts= cap.get(cv2.CAP_PROP_POS_MSEC)
    print("ts @ frame 5 after setPos %.3f"%ts)
    ret,frame=cap.retrieve()
    ts= cap.get(cv2.CAP_PROP_POS_MSEC)
    print("ts @ frame 5 after retrieve %.3f"%ts)
    ret,frame=cap.read()
    ts= cap.get(cv2.CAP_PROP_POS_MSEC)
    print("ts @ frame 5 after read %.3f"%ts)
    cap.release()
    if __name__ == '__main__':
        pass
