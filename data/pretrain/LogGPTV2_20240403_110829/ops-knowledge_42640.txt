User: How to configure firewall rules in Linux system to protect server security?
Assistant: In the Linux system, the commonly used firewall tools are iptables and firewalld. The following are some common firewall rule configuration methods: 1. iptables iptables is one of the most commonly used firewall tools in Linux. The following are some common commands of iptables: (1) Clear iptables rules: iptables -F (2) Forbid all input connections: iptables -P INPUT DROP (3) Allow all output connections: iptables -P OUTPUT ACCEPT (4) Allow local loopback Connections: iptables -A INPUT -i lo -j ACCEPT (5) Allow established connections: iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT (6) Allow specific protocols and ports: iptables -A INPUT -p tcp --dport 22 -j ACCEPT iptables -A INPUT -p tcp --dport 80 -j ACCEPT 2. firewalld firewalld is the default firewall tool in RedHat and CentOS. The following are some common commands of firewalld: (1) Start the firewall: systemctl start firewalld (2) View the status of the firewall: systemctl status firewalld (3) Open ports: firewall-cmd --add-port=80/tcp --permanent #Permanent Effective firewall-cmd --reload #immediately effective (4) open service: firewall-cmd --add-service=http --permanent #permanently effective firewall-cmd --reload #immediately effective The above commands are only examples, specific firewalls The rule configuration should be adjusted according to actual needs. At the same time, in order to ensure the security of the server, it is recommended to test after configuring the firewall rules to ensure that the rules can take effect and the expected effect is expected to be achieved.
