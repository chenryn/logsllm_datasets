### 中国公司发布首款 RISC-V 笔记本电脑？

RISC-V 国际透露，一家名为“鉴释”的中国公司宣布了首款面向开发者的 RISC-V 笔记本电脑，并已开放预订。这款名为“ROMA”的开发平台配备四核 RISC-V 处理器，最高支持 16GB 内存和 256GB 存储空间，并且能够运行大多数 RISC-V Linux 发行版。

有趣的是，“鉴释”原本是一家专注于静态代码分析工具的公司。在该公司的中英文网站上，除了一个孤立的[预订页面](https://xcalibyte.com.cn/en/roma-preorder/)外，没有其他关于 ROMA 的信息。预订页面提供了一个表格，但未提及具体价格或预期供应量，甚至没有概念图。此外，页面还提到了 Web3 和 NFT 相关的内容。

**消息来源**: [Phoronix](https://www.phoronix.com/scan.php?page=news_item&px=RISC-V-Dev-Laptop)

**老王点评**: 这真的不是洋葱新闻吗？

### 谷歌推出“高级 API 安全”服务

谷歌宣布了“高级 API 安全”服务的预览版，这是一个将在谷歌云平台上推出的新产品，旨在检测与 API 相关的安全威胁。该服务可以识别 API 错误配置并检测恶意机器人。根据 2018 年的一份报告，三分之二的组织向公众和合作伙伴暴露了不安全的 API。

**消息来源**: [Tech Crunch](https://techcrunch.com/2022/06/30/google-launches-advanced-api-security-to-protect-apis-from-growing-threats/)

**老王点评**: 配置不当的 API 可能造成严重影响，网络安全圈对此深有体会。

![](/data/attachment/album/202207/03/170320rrverz3ozrr7v79p.jpg)

### Linux Mint 21 不准备使用 oomd

Linux Mint 21 计划在今年夏天发布，基于 Ubuntu 22.04 LTS。由于 Ubuntu 22.04 在 systemd-oomd 上受到广泛批评，Linux Mint 决定不使用该服务来强制关闭用户的应用程序。Canonical 公司的开发人员正在努力修复这个问题，最新提出的 SRU 可能会有所改善。

**消息来源**: [Phoronix](https://www.phoronix.com/scan.php?page=news_item&px=Linux-Mint-17-No-OOMD)

**老王点评**: Linux Mint 做出了明智的选择。问题不在于 oomd 的策略，而是未经用户同意就强制关闭应用。
