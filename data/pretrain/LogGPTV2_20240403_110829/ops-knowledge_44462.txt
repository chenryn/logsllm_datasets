在我们迁移到Windows 7的过程中，遇到了一个关于用户登录的问题。当按下“Ctrl+Alt+Del”组合键时，系统会显示“用户选择屏幕”。通常情况下，该屏幕上会列出所有本地用户的图标。然而，由于我们的计算机是域成员，并且已禁用了“快速用户切换”，因此屏幕上仅显示“其他用户”选项。点击“其他用户”或按回车键后，系统将跳转到常规登录界面，要求输入用户名和密码。

问题在于：我们希望能够直接跳过用户选择“其他用户”的步骤，让系统默认总是选择“其他用户”，并在用户按下“Ctrl+Alt+Del”组合键后直接进入登录界面。奇怪的是，“其他用户”选项只有在多个域用户登录后才会出现。重新安装操作系统后，登录过程会直接跳转到用户凭证输入界面。

**当前的安全策略设置如下：**

- 交互式登录：不显示上次的用户名 = 启用
- 交互式登录：不要求使用 Ctrl+Alt+Del = 禁用

请问有什么解决办法？

### 解决方案

您可以通过以下组策略配置来实现这一需求：

1. 打开 **组策略管理控制台** (Group Policy Management Console)。
2. 创建或选择 **Default Domain Policy**（默认域策略），然后编辑它。
3. 在打开的 **组策略管理编辑器** 中，进行以下修改：

   - **计算机配置** -> **策略** -> **Windows 设置** -> **安全设置** -> **本地策略** -> **安全选项**：
     - “交互式登录：不显示上次的用户名” = 启用
     - “交互式登录：不要求使用 Ctrl+Alt+Del” = 禁用

   - **计算机配置** -> **策略** -> **管理模板** -> **系统** -> **登录**：
     - “隐藏快速用户切换的入口点” = 启用
     - “始终使用经典登录” = 启用

通过这些设置，您可以确保在用户按下“Ctrl+Alt+Del”组合键后，系统直接跳转到登录界面，并且不需要选择“其他用户”选项。
