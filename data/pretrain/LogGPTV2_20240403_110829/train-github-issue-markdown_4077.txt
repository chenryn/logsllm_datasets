Challenge http://www.freecodecamp.com/challenges/waypoint-bring-your-
javascript-slot-machine-to-life has an issue.
The second condition for this challenge:
> You should have used the the selector given in the description to select
> each slot and assign it the value of slotOne, slotTwo and slotThree
> respectively
The selector in the description:
> `$($(".slot")[0]).html(slotOne);`
The condition passes when you use double quotes in the selector per the
description:
    $($(".slot")[0]).html(slotOne);
    $($(".slot")[1]).html(slotTwo);
    $($(".slot")[2]).html(slotThree);
It fails with single quotes:
    $($('.slot')[0]).html(slotOne);
    $($('.slot')[1]).html(slotTwo);
    $($('.slot')[2]).html(slotThree);
Maybe we are trying to conform to the jQuery JavaScript Style Guide? Still,
both seem to be valid JavaScript. Indeed, the challenge immediately following
this one uses single quotes for the selector in the description and passes
with either single or double quotes.
