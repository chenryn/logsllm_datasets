### 2017 年成为 Linux 专家的 4 个热门技能

> 你在新的一年里需要提升哪些技能？

![](/data/attachment/album/201701/31/233141le334md33m1i3dje.jpg)

成为一个 Linux 专家的一个挑战是“专家”这一定义在不断变化。在我刚接触 Linux 的时候，要成为一名 *Linux 专家*，你需要能够自己编译内核。甚至作为普通用户，如果你想在笔记本电脑上运行 Linux，也必须编译一个自定义内核。然而，如今编译自己的内核通常是浪费时间。这并不是说它不重要，但在开源世界中，我们建立在他人成功的基础上，而 Linux 发行版为我们提供了功能良好的内核。虽然“专家”的定义并不总是剧烈变化，但对 IT 专业人员的需求每年都在变化。

以下是 2017 年 Linux 专业人士应具备的四个关键技能：

### 1. 安全

我这里不是指安全专家或安全顾问。这些职位和服务固然重要，但随着联网设备渗透到我们生活的方方面面，我们需要在每个决策中都具备安全意识。今年，我和妻子购买了一台带有蓝牙功能的洗衣机和冰箱。黑客入侵我的漂洗系统的想法可能听起来很荒谬，但这都是潜在的攻击点。

每当激活工作、家庭或口袋中的任何系统时，我们都应该考虑它们可能引发的安全问题。由于像联网烤面包机这样的物品不太可能及时获得固件升级，我们需要设计其他系统以应对这些普通设备可能遭受的破坏。我们需要比以往任何时候都更加重视来自防火墙内部的攻击。不要让你的文件服务器被搅拌器破坏！

### 2. DevOps

[DevOps](https://opensource.com/tags/devops) 不再是一个新概念。在过去两三年里，我们一直鼓励员工学习 DevOps 以取得成功。这是一个好建议，但这并不意味着我们应该完全依赖自动化工具来完成工作。Chef、Puppet、Ansible、Salt Stack 及类似工具非常好，但我们仍需了解背后发生了什么，以便在出现不可避免的错误时能够解决它们。

即使使用 DevOps 的编程方法，我们仍然需要能够维护、修复和理解底层系统的 Linux 专家。没有这些专家，云计算将变得非常可怕，即使那个云是在你自己的数据中心里。

### 3. 开发

作为一名有 20 年经验的系统管理员，我从来没有时间学习编程。这听起来像是借口，但事实如此。我的所有开发技能仅限于基本的脚本编写，以帮助我更快地工作。不过，那些日子已经结束了。虽然我们需要在 DevOps 世界中具备系统管理技能，但我们也需要系统管理员具备编程技能。

如果你是一位像我一样经验丰富的系统管理员，你可能已经采用了 DevOps 并每天都在使用它。但如果你想超越他人，你需要学习如何通过编程解决问题，并且不要认为 Chef 或 Puppet 代码只是配置文件。每位 IT 专业人员至少需要掌握编程的基本概念，因为 DevOps 代码在某种程度上抽象了 IT 的各个方面。

### 4. 软技能

通常，在准备职业生涯时，我们最后才会考虑所谓的 *软技能* ——社交和沟通技巧——但它们可能是决定你成功的关键因素。无论你正在寻找一份新工作，还是试图适应当前职业生涯的变化，软技能都是至关重要的。

IT 各个领域的界限越来越模糊，良好的沟通能力使这些模糊的分界线变成优势而不是障碍。我们生活在一个开发人员围绕着服务器，而运维团队编写 Ruby 代码来维护服务器农场的世界。这些都是 IT 中的新思想，如果人们不能在不同部门之间有效沟通，工作场所将迅速充满敌意。此外，IT 人员还需要与其他业务领域的人员进行有效沟通。现在比以往任何时候都有更大的需求。

你计划在 2017 年里增加哪些技能？请在评论中告诉我们。
