### Description
The doc says:
    In environments that donâ€™t support this you can enable loose mode on es6.modules and instead of using Object.defineProperty an assignment will be used instead.
But it only works for the `__esModule` property.
Without loose, the output is:
    Object.defineProperty(exports, "__esModule", {
      value: true
    });
With loose, the output is:
    exports.__esModule = true;
But when I re-export a module
    export * from './mixin'
The output is (with or without the loose option):
    Object.keys(_mixin).forEach(function (key) {
      if (key === "default") return;
      Object.defineProperty(exports, key, {
        enumerable: true,
        get: function get() {
          return _mixin[key];
        }
      });
    });
Which will break on IE8
