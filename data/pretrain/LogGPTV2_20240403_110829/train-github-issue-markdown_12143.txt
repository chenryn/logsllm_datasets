When using single or double quotes within back-tick quotes, they are not
considered quoted:
![screenshot 2015-12-10 14 26
50](https://cloud.githubusercontent.com/assets/573056/11725918/1e339a48-9f4a-11e5-90ac-d02f433dcdca.png)
So if they happen to be closed within the back ticks, it's not the end of the
world, but if not, all subsequent formatting is lost.
