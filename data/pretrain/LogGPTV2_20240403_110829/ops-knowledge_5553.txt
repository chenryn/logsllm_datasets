User: 根据以下描述，设计一个基于MySQL的停车场管理系统
停车场管理系统需要具备以下功能：\n1. 维护车辆信息，包括车牌号、车型、车主信息等；\n2. 维护停车位信息，包括停车位编号、停车位类型（小型车位、大型车位等）、停车位状态（空闲、占用等）等；\n3. 实现车辆进出记录的管理，包括车辆进入时间、出发时间、停留时间等；\n4. 提供收费管理功能，包括收费标准、收费金额计算等；\n5. 提供数据统计功能，包括车位占用率、停车时长等数据的统计分析。
Assistant: 表设计：

1. 车辆信息表

| 字段名 | 字段类型 | 描述 |
| --- | --- | --- |
| id | int | 车辆编号 |
| plate_number | varchar(20) | 车牌号 |
| type | varchar(20) | 车型 |
| owner | varchar(50) | 车主信息 |

2. 停车位表

| 字段名 | 字段类型 | 描述 |
| --- | --- | --- |
| id | int | 停车位编号 |
| type | varchar(20) | 停车位类型 |
| status | varchar(20) | 停车位状态（空闲、占用等）|

3. 车辆进出记录表

| 字段名 | 字段类型 | 描述 |
| --- | --- | --- |
| id | int | 进出记录编号 |
| plate_number | varchar(20) | 车牌号 |
| entry_time | datetime | 进入时间 |
| exit_time | datetime | 出发时间 |
| duration | int | 停留时间 |

4. 收费标准表

| 字段名 | 字段类型 | 描述 |
| --- | --- | --- |
| id | int | 收费标准编号 |
| type | varchar(20) | 收费标准类型 |
| price | decimal | 收费金额 |

5. 数据统计表

| 字段名 | 字段类型 | 描述 |
| --- | --- | --- |
| id | int | 统计编号 |
| date | date | 统计日期 |
| occupancy_rate | decimal | 车位占用率 |
| average_duration | int | 平均停车时长 |

以上表中的id均为主键。在车辆进出记录表中，entry_time、exit_time字段可以通过程序自动生成。统计表中的数据可以通过程序自动计算。

需要注意，收费标准表中的收费金额可以根据不同的停车位类型、停留时间长短设置不同的收费标准。同时，停车位表和车辆进出记录表中需要增加外键关联到车辆信息表和收费标准表，方便数据管理和查询。
