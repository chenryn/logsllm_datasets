### ESXi Server
- **端口**：443
- **用途**：在将虚拟设备部署到 ESXi 时，亚信安全服务器深度安全防护系统管理中心通过端口 443 与 ESXi Server 进行通信。
- **协议**：HTTPS
- **发起方**：亚信安全服务器深度安全防护系统管理中心
- **目标**：ESXi Server
- **注意事项**：vCenter 在 ESXi 上向亚信安全服务器深度安全防护系统管理中心提供三个 URL，以上传构成虚拟设备的三个 vmdk 文件。亚信安全服务器深度安全防护系统管理中心会建立 HTTPS 连接，并将 vmdk 传递给 ESXi。

### 本地软件分发 Web 服务器
- **端口**：80 或 443
- **用途**：客户端可以被配置为从这些 Web 服务器请求软件更新，而不是通过亚信安全服务器深度安全防护系统中继请求更新。
- **协议**：HTTP 或 HTTPS
- **发起方**：亚信安全服务器深度安全防护系统客户端
- **目标**：本地软件分发服务器
- **配置需求**：无需进行额外配置；端口在 Web 服务器 URL 中定义。
- **注意事项**：有关配置软件分发 Web 服务器的信息，请参阅相关章节（第 174 页）。

### 捆绑 NIC
#### 在捆绑 NIC 环境中安装 Windows 和 Solaris 客户端
“捆绑 NIC”是指并行使用多个以太网适配器来提高数据传输速度或提供冗余。以下信息为在 Windows 和 Solaris 中配置捆绑 NIC 提供指导，确保其与亚信安全服务器深度安全防护系统客户端兼容。如遇问题，请联系技术支持。

##### Windows
- **说明**：Windows NIC 捆绑软件创建了一个新的虚拟主接口，该接口采用了第一个从接口的 MAC 地址。默认情况下，安装期间 Windows 客户端将绑定到所有虚拟和物理接口。
- **注意事项**：
  - 在捆绑 NIC 环境中，客户端将绑定到物理接口以及由捆绑软件创建的虚拟接口上。
  - 如果客户端的多个接口具有相同的 MAC 地址，则该客户端将无法正常使用。
  - 客户端必须只绑定到由捆绑软件创建的虚拟接口上。
  - 在 Windows 2003 上的捆绑 NIC 环境中使用客户端需要 SP2 或更高版本，或安装特定补丁（KB 912222）。
  - 不支持在 Windows 2000 上的捆绑 NIC 环境中使用客户端。
  - 安装后，当您向捆绑 NIC 中添加或移除网络接口时，需验证驱动程序是否仅绑定到虚拟接口，而未绑定到任何物理适配器。

##### Solaris
- **说明**：Solaris 中的 IPMP 故障转移（活动-待机）模式允许两个 NIC 具备相同的硬件 (MAC) 地址。由于亚信安全服务器深度安全防护系统客户端按适配器的 MAC 地址识别适配器，因此重复的 MAC 地址会导致客户端无法正常运行。
- **解决方案**：为每个适配器手动分配唯一的 MAC 地址。
- **示例 ifconfig 输出**：
  ```shell
  # ifconfig -a
  hme0: flags=1000843 mtu 1500 index 2
  inet 10.20.30.40 netmask 0
  ether 8:0:20:f7:c3:f
  hme1: flags=1000842 mtu 1500 index 8
  inet 0.0.0.0 netmask 0
  ether 8:0:20:f7:c3:f
  ```
- **操作步骤**：
  - 使用 `ifconfig` 命令手动设置新的唯一 MAC 地址。
  - 通过 `snoop` 命令搜索选定的 MAC 地址，确认不存在冲突。
  - 使用 `ping` 命令测试与子网广播地址的连接。
- **注意事项**：在相同子网具有多个接口的 Solaris 系统中，操作系统可通过任意接口进行数据包路由。因此，应对所有接口应用相同的防火墙状态配置选项或入侵防御规则。

### 亚信安全服务器深度安全防护系统虚拟设备界面
- **访问方式**：通过打开 VMware vSphere Client，在导航面板中选择“设备”，然后单击“控制台”选项卡。
- **系统信息**：显示设备版本、Build 号、管理此设备的亚信安全服务器深度安全防护系统管理中心的 URL 以及该设备的时区。
- **配置密码**：访问此设备控制台的当前密码。安装后应更改默认密码 ("dsva")。
- **配置管理网络**：显示设备主机名和 IP 地址。默认主机名为 "dsva"。IP 地址由本地 DHCP 服务器分配。如果没有 DHCP 服务器，必须手动输入 IP 地址、网络掩码、默认网关、主 DNS 和辅助 DNS 信息。（按 Enter 键进入编辑模式。）
- **注意事项**：如果要部署多个虚拟设备，请确保更改主机名以避免出现 DNS 问题。

### 管理 Amazon Web 服务区域
#### 添加 Amazon Web 服务区域
- **描述**：当用户尝试使用“添加云帐户”向导添加云帐户时，如果托管您的云资源的 Amazon Web 服务区域不显示，您需要使用 `dsm_c` 命令行工具手动将该区域添加到列表。
- **命令**：
  ```shell
  dsm_c -action addregion -region REGION -display DISPLAY -endpoint ENDPOINT
  ```
- **参数**：
  - `REGION`：Amazon Web 服务区域标识符。例如：`ca-east-1`
  - `DISPLAY`：添加云帐户向导中区域使用的显示字符串。例如：加拿大东部（渥太华）
  - `ENDPOINT`：为区域使用的 Amazon Elastic Compute Cloud (EC2) 端点的完全限定域名。例如：`ec2.ca-east-1.amazonaws.com`
- **注意事项**：在 Linux 环境下运行亚信安全服务器深度安全防护系统管理中心时，需要以 root 用户身份运行 `dsm_c` 命令。手动添加 Amazon Web 服务区域时，可能需要添加信任证书。请参阅相关章节（第 270 页）。

#### 查看已添加的 Amazon Web 服务区域
- **命令**：
  ```shell
  dsm_c -action listregions
  ```

#### 移除已添加的 Amazon Web 服务区域
- **描述**：之前使用该区域添加的任何云帐户将继续有效，但用户无法使用该区域创建新的云帐户。
- **命令**：
  ```shell
  dsm_c -action removeregion -region REGION
  ```
- **参数**：
  - `REGION`：Amazon Web 服务区域标识符。例如：`ca-east-1`
- **注意事项**：在 Linux 环境下运行亚信安全服务器深度安全防护系统管理中心时，需要以 root 用户身份运行 `dsm_c` 命令。

### 管理信任证书
#### 导入信任证书
- **描述**：您可以使用亚信安全服务器深度安全防护系统管理中心将用于代码签名和 SSL 连接的信任证书导入系统。对于 Amazon Web 服务区域的信任证书，必须使用 `dsm_c` 命令行工具。
- **使用管理中心导入**：
  1. 转至“管理 > 系统设置 > 安全”。
  2. 单击“查看证书列表”。
  3. 单击“从文件导入...”启动导入证书向导。
- **使用 `dsm_c` 导入**：
  ```shell
  dsm_c -action addcert -purpose PURPOSE -cert CERTFILE
  ```
  - `PURPOSE`：证书将要用于的连接类型。例如：`AWS - Amazon Web 服务`、`DSA - 代码签名`、`SSL - SSL 连接`。
  - `CERTFILE`：包含待导入证书的文件的路径。例如：`/path/to/cacert.pem`
- **注意事项**：在 Linux 环境下运行亚信安全服务器深度安全防护系统管理中心时，需要以 root 用户身份运行 `dsm_c` 命令。

#### 查看信任证书
- **使用管理中心查看**：
  1. 转至“管理 > 系统设置 > 安全”。
  2. 单击“查看证书列表”。
- **使用 `dsm_c` 查看**：
  ```shell
  dsm_c -action listcerts [-purpose PURPOSE]
  ```
  - `-purpose PURPOSE`：可选参数。如果省略该参数，可查看所有证书的列表。如果指定了值，将只显示用于指定用途的证书。
- **注意事项**：在 Linux 环境下运行亚信安全服务器深度安全防护系统管理中心时，需要以 root 用户身份运行 `dsm_c` 命令。

#### 移除信任证书
- **使用管理中心移除**：
  1. 转至“管理 > 系统设置 > 安全”。
  2. 单击“查看证书列表”。
  3. 选择要移除的证书并单击“删除”。
- **使用 `dsm_c` 移除**：
  ```shell
  dsm_c -action removecert -id ID
  ```
  - `ID`：亚信安全服务器深度安全防护系统管理中心为要删除的证书分配的 ID 值。例如：`3`
- **注意事项**：在 Linux 环境下运行亚信安全服务器深度安全防护系统管理中心时，需要以 root 用户身份运行 `dsm_c` 命令。
