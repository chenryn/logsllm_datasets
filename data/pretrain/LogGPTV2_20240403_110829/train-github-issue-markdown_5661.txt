# Environment
    Windows build number: Microsoft Windows [Version 10.0.18362.900]
    PowerToys version: v0.19.1
    PowerToy module for which you are reporting the bug (if applicable): FancyZones
# Steps to reproduce
Try to attach a window to a FancyZone that borders the Windows Taskbar and the
window is resized behind the taskbar.
Noticed this happening after PowerToys update when Windows Explorer had to
restart and also after Sleep/Hibernation.
# Expected behavior
Windows don't go behind the taskbar
# Actual behavior
Windows do go behind the taskbar
