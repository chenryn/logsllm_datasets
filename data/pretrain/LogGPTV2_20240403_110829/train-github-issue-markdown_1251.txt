### What happened
On Airflow 2.4.3
When using a deferrable operator the log disappears from the webpage while the
operator is deferred and appears when it is running/finished. This doesn't
happen consistently over all DAG runs.
The error has been reproduced in the new log panel on the grid page & the
"standalone" log page.
As an example, I've tried running a simple DAG (just executes `sleep 10`). The
logs are for the 3rd retry and I am using the "Auto-refresh" button on the
grid page.
There are no visible errors on the web console when this happens, but the web
server logs look like Airflow doesn't know it is the 3rd try while the
operator is deferred and gets the 2nd try.
