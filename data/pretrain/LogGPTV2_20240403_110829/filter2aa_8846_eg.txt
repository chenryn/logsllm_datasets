### 不分密级与混合环境

#### 1. 不分密级
在这种环境中，数据的隔离仅基于内容，而不是密级。每个域代表一个单独的隔间，并且有独立的权限管理。

#### 2. 混合环境 (Hybrid Environment)
在混合环境中，数据既按密级也按内容进行隔离。这种管理方式使得客体管理的粒度更细，但同时也增加了管理的复杂性。

### 域 (Domain) 的概念

“域”这一术语早在 Microsoft 创立之前就已经存在，并在 CISSP 中多次出现。通常情况下，人们会将域理解为由一台运行 Microsoft 软件的服务器（称为域控制器）控制的一组计算机和设备。实际上，域是指某个主体可用的一组资源。主体可以是用户、进程或应用程序。在操作系统内部，每个进程都有一个域，即该进程在执行任务时可访问的一组系统资源，如内存段、硬盘空间、操作系统服务以及其他进程。在网络环境中，域是一组可用的物理和逻辑资源，包括路由器、文件服务器、FTP 服务、Web 服务器等。

#### 安全域 (Security Domain)
安全域是在普通域的基础上，增加了相同安全策略下的资源管理和统一的管理组。例如，网络管理员可能会将所有会计人员及其相关资源放入一个域中，而将所有管理人员及其相关资源放入另一个域中。这些不同的容器不仅实现了相似的业务功能，更重要的是它们具有相同的信任级别，从而允许单个安全策略对多个实体进行管理。

### 域的结构与通信
不同域之间通过逻辑边界（如防火墙、目录服务器）进行分隔。域可以采用层次化结构，说明不同域之间的关系及主体间的通信方式。主体能够访问相同或更低信任级别的域中的资源。通信信道由安全代理（如防火墙、路由器 ACL、目录服务）控制，不同的域使用特定的子网掩码地址进行隔离。

### 单点登录 (Single Sign-On, SSO)
AIO 教材中提到的域概念与单点登录相关，因为目前有多种技术可用于定义和实施这些域及其对应的安全策略，如 Windows 环境中的域控制器、企业资源管理 (ERM) 产品、Microsoft Passport（现为 Windows Live ID）以及提供 SSO 功能的各种产品。这些技术的目标是使用户只需登录一次即可访问不同的域，无需重新输入其他凭证。

### 自主访问控制 (Discretionary Access Control, DAC)

1. **自主访问控制 (DAC)**:
   - 由客体的所有者、创建者或数据保管者自行控制和定义主体对该客体的访问。
   - 通常通过客体的访问控制列表 (ACL) 来实现。
   - 在通用准则 (Common Criteria, CC) 中属于 C 级。
   - 自主访问控制有时也被称为基于身份的访问控制 (Identity-based Access Control)，其中系统识别用户身份并分配相应资源。

2. **非自主访问控制 (Nondiscretionary Access Control, NDAC)**:
   - 管理员集中管理不可任意支配的访问控制。
   - 包括基于规则、基于角色和基于格子的访问控制模型。
   - 实际考试中，非自主访问控制通常指的是基于角色的访问控制，其特点是通过中央授权来决定主体可以访问哪些客体。

### 预防与减缓访问控制攻击

#### 一、黑客与攻击者
- 黑客、骇客和攻击者的区别：早期，骇客指干坏事的攻击者，黑客指不干坏事的技术高手。现在，所有的攻击者统称为黑客或攻击者。
- 相关攻击技术和案例很多，不再赘述。

#### 二、高级持续性威胁 (Advanced Persistent Threat, APT)
- APT 是由国家或政府支持的、高技术水平的、长期持续的网络空间作战威胁。
- 经典案例包括 Target 公司 POS 机用户信用卡信息被窃事件和“震网”病毒。

#### 三、针对访问控制的攻击
1. **访问聚合攻击 (Access Aggregation Attacks)**:
   - 通过收集多个非敏感信息块并结合以获得敏感信息。
   - 网络侦察 (Reconnaissance) 也是一种访问聚合攻击，通过结合多种工具识别系统的多个元素。
   - 应结合严格访问控制、“需知”和最小特权原则来预防此类攻击。

2. **密码攻击 (Password Attacks)**:
   - **字典攻击 (Dictionary Attacks)**: 使用预先编好的字典进行快速尝试。
   - **暴力攻击 (Brute-Force Attacks)**: 尝试所有可能的字母、数字和符号组合。
   - **生日攻击 (Birthday Attack)**: 利用生日悖论来破解散列值。
   - **彩虹表攻击 (Rainbow Table Attacks)**: 使用预先计算好的字符串和散列值映射数据库加速密码破解。
   - **嗅探攻击 (Sniffer Attacks)**: 通过网络捕捉和分析流量。

3. **电子欺骗攻击 (Spoofing Attacks)**:
   - 包括 IP 欟骗、邮件欺骗、电话欺骗等。

4. **社会工程学攻击 (Social Engineering Attacks)**:
   - 包括网络钓鱼 (Phishing)、鱼叉式钓鱼 (Spear Phishing)、捕鲸 (Whaling) 和语音钓鱼 (Vishing)。

5. **智能卡攻击 (Smartcard Attacks)**:
   - **故障生成攻击 (Fault Generation Attacks)**: 通过操纵智能卡环境组件引入错误，反向工程加密过程。
   - **旁路攻击 (Side-Channel Attacks)**: 通过分析功耗、电磁发射和计时来获取敏感信息。
   - **微区探查 (Microprobing)**: 通过去除智能卡外部保护材料直接访问 ROM 芯片。

6. **拒绝服务攻击 (Denial-of-Service Attacks, DoS)**:
   - 通过大量请求使目标系统无法正常服务。

7. **网址嫁接 (Pharming)**:
   - 将受害者重定向到伪造的 Web 站点。

### 访问控制的安全防护方法汇总
1. 物理上保护好核心设备。
2. 使用加密、散列和加盐技术。
3. 设置强密码。
4. 输入密码时隐藏显示。
5. 使用多因素认证。
6. 使用账户锁定技术防止暴力破解。
7. 显示最后一次登录的时间和地点。
8. 进行安全意识培训。

### 管理身份与访问供给生命周期
1. **提供/开通/指派 (Provisioning)**:
   - 新用户账户的初始创建。
2. **审查/评审 (Review)**:
   - 检查账户以确保不活跃账户被禁用，并防止特权过度和特权蠕变。
3. **撤销 (Revocation)**:
   - 及时禁用离职员工的账户，并根据需要注销。

### 安全评估与测试
1. **安全测试 (Security Testing)**:
   - 验证安全防护措施是否正常运行。
2. **安全评估 (Security Assessments)**:
   - 对系统进行全面评价，识别漏洞并提出修复建议。
3. **安全审计 (Security Audits)**:
   - 由独立审计员执行，展示控制的有效性。
4. **漏洞评估/漏洞测试 (Vulnerability Assessments)**:
   - 通过漏洞扫描和渗透测试发现系统弱点。

### 有关角色
1. **系统工程师和安全专家**:
   - 与赞助商共同创建或论证测试和评估战略。
   - 提出风险管理方法，监控测试过程并提出改进建议。
   - 作为测试团队的顾问帮助指定相关计划。
