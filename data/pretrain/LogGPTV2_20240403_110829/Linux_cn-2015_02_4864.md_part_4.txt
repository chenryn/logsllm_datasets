### 为什么 Mac 用户不选择 Linux

Linux 和 Mac 用户至少有一个共同点：他们都不太喜欢使用 Windows。然而，除了这一点外，两者在其他方面几乎无法达成共识，从而分道扬镳。那么，为什么 Mac 用户不选择 Linux 呢？是什么因素导致了这种选择？

Datamation 对此问题进行了调查，并试图给出答案。Datamation 的结论是，所有原因都归结于应用和工作流程，而非操作系统本身。

> 某些事例表明，尝试用新应用替代现有应用并不实际——无论是对工作流程还是整体实用性来说都是如此。不幸的是，苹果在这方面做得非常好。因此，在几乎不可能改变这些事实的情况下，想要拉拢那些忠实的 Mac 用户是一个巨大的挑战。
>
> 老实说，除了 Web 开发者，我还没有见过 Mac 用户仅仅为了避免升级到 OS X Yosemite 而集体尝试变更他们的工作流程。确实，经历过 Yosemite 更新后，考虑过权限的用户会发现这方面变得非常令人讨厌。尽管 OS X 在 UI 方面有一些微小的变化，但其核心工作流程在最大程度上保持了原样。
>
> 我相信 Linux 在未来将继续保持多样化的特点。Linux 会继续成长，但绝不会一成不变。

我大体上同意 Datamation 关于应用和工作流程重要性的结论。在选择操作系统时，这两方面确实是必须考虑的因素。但我认为，对于 Mac 用户来说，选择 Mac 有比这两方面更重要的因素。我相信不同的心态造就了 Linux 和 Mac 用户，这才是为什么 Mac 用户不选择 Linux 的真正原因。

#### 控制权才是 Linux 用户最看重的地方

Linux 用户倾向于控制电脑上的每一个细节，他们会尽一切努力使操作系统变成他们想要的样子。然而，这种方式并不适用于 OS X 及其他苹果产品。如果你使用苹果的产品，就意味着你必须按照苹果预先设定的模式来使用它们。

对于 Mac（以及 iOS）用户来说，这并没有什么问题，因为他们似乎并不介意生活在苹果那围墙高筑的花园里，仅使用苹果提供的标准和选择。但这对绝大多数 Linux 用户来说是完全不能接受的。大多数 Linux 新用户来自 Windows，正是在那里，他们开始厌恶那些限制操作系统权限的东西。

自从他们尝到自由的 Linux 系统带来的甜头之后，他们再也不会回到苹果或微软的“监牢”中去了。即使在他们死后，从他们冰冷僵硬的手指中撬出 Linux 也比让他们接受苹果或微软为他们定制的操作系统更容易。

而大多数 Mac 用户则没有这样的决心。对他们来说，当苹果升级 OS X 时放弃现有的习惯方式是非常容易的。即使在苹果那围墙高筑的花园里不满意苹果的变化，他们也会迅速接受。

因此，对控制权的渴望是 Mac 用户与 Linux 用户的最大不同。但我并未将其视为一个问题，尽管这反映出两类用户截然不同的态度。

#### Mac 用户离不开苹果的技术支持

Linux 用户与 Mac 用户的区别还体现在技术支持方面。Linux 用户并不介意亲自维护自己的电脑。虽然维护电脑及控制操作系统都是很大的责任，但 Linux 用户愿意独自承担，通过自己的力量使系统更高效，并且深入了解操作系统是每一位 Linux 用户乐衷的事情。

当 Linux 用户遇到问题时，他们会迅速尝试自己解决问题。如果这不奏效，他们会在网上搜索其他 Linux 用户的解决方案，并不断尝试直到问题解决。

但 Mac 用户却不大会这样做。这也许是为什么苹果零售店如此火爆，为什么许多 Mac 用户在购买新 Mac 时会选择购买苹果维护服务的原因。Mac 用户会很轻易地带着他们的电脑去苹果零售店，走进天才吧并要求苹果的工作人员为其查看和修复电脑。

大多数 Linux 用户连想都不会想这种事情。谁会愿意让一个你不认识的人碰你的电脑并维修它呢？

因此，对 Mac 用户来说，很难抛弃过去可以从苹果那里得到的技术支持，转而使用 Linux。这种选择会让某些 Mac 用户觉得自己的电脑变得脆弱、易受攻击，如同离开母亲怀抱的婴儿般无助。

#### Mac 用户喜爱苹果的硬件

Datamation 发表的文章主要研究了软件方面的原因，但我认为硬件因素同样对 Mac 用户有很大影响。大部分 Mac 用户非常喜欢苹果的硬件。他们购买 Mac 不仅仅是为了 OS X，苹果精美的工艺设计也是他们购买时的重要考虑因素。Mac 用户愿意支付高价购买电脑，因为他们认为这是物有所值的。

另一方面，Linux 用户似乎不太考虑这些。我认为他们更关注电脑的成本，而不是外观和设计。对他们来说，花费最少的钱获取尽可能好的硬件才是最重要的。他们不像 Mac 用户那样热衷于电脑的外观，因此这一点并不是他们在购买电脑时的主要考虑因素。

我认为对于硬件的不同观点并没有高低之分。这只是与用户的需求有关，只会影响他们在购买电脑时的选择，或者对某些 Linux 用户来说，只是因为他们想要自己组装电脑而已。两种观点只是因为出发点不同，对电脑真正价值的理解不同罢了。

当然，在 Mac 上运行 Linux 是可能的，直接运行或者通过虚拟机运行。因此，真心喜欢苹果硬件的 Mac 用户可以选择在 Mac 上安装 Linux。

#### Linux 发行版太多，难以选择

另一个让 Mac 用户无法选择 Linux 的原因是，要从众多 Linux 发行版中选择一个实在太困难了。在大多数 Linux 用户并不抗拒的多元化发行版时代，没有相关知识的 Mac 用户会对如何选择感到困惑。

我认为，随着时间的推移，Mac 用户可以学习并找出最适合自己的发行版。但在短时间内，尤其是长期使用 OS X 之后，这是一个艰巨的任务。我不认为这个问题是无法克服的，但有必要在这里提一下。

当然，我们可以提供资源，推荐参考 DistroWatch 和我们的博客 Desktop Linux Reviews，这都有助于 Mac 用户找到适合的 Linux 发行版。再说一条，网上有很多诸如“最好的 Linux 发行版”等类似的文章，当 Mac 用户想要寻找适合自己使用的发行版时可以参考一下。

但苹果顾客购买 Mac 的原因之一是苹果硬件和软件协调统一的简便性和易用性。所以我不确定有多少 Mac 用户愿意花费时间找出适合自己的 Linux 发行版。也许是否要使用 Linux 会让他们考虑一阵子。

#### Mac 用户是苹果，Linux 用户是橘子

我认为 Mac 用户与 Linux 用户分道扬镳并没有什么不妥。我们只是在谈论两类完全不同的人群，这是一件好事，因为两类人群都在按自己喜欢的方式使用操作系统和软件。让 Mac 用户和 Linux 用户各自沉浸在 OS X 和 Linux 中吧，希望他们都能高兴并对自己的电脑满意。

也许 Mac 用户会偶然走入 Linux 的世界并开始转向 Linux，但我认为大多数时候，两类人都愿意呆在不同的世界并不与对方接触。通常来说，我并不会随意比较二者，尤其是你已经自己拿定主意的时候，况且这只不过是选苹果还是选橘子的问题罢了。
