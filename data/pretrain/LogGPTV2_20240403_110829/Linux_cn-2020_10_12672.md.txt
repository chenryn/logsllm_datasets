#### 泄露的 Windows XP 代码已成功编译成可运行的操作系统

上周，Windows XP 和 Windows Server 2003 的源代码在 4chan 论坛上被泄露。一位名叫 NTDEV 的用户从数百万下载该代码的人中脱颖而出，他决定亲自验证这些代码的真实性。根据他在网上分享的视频，NTDEV 成功地在周末编译了 Windows XP 代码，并于次日完成了 Windows Server 2003 代码的编译。

来源：[zdnet](https://www.zdnet.com/article/windows-xp-leak-confirmed-after-user-compiles-the-leaked-code-into-a-working-os/)

> 拍一拍：这种方式确实令人惊叹！

#### Khronos 发布新版 OpenCL SDK 及 OpenCL 3.0 规范

Khronos Group 最近发布了 OpenCL 3.0 标准及其配套的 SDK。相较于前一版本，OpenCL 3.0 将许多原先强制性的功能转变为可选项，从而使得这一标准能够适应更广泛的设备和环境需求。此外，新版本还引入了几项扩展功能，包括允许获取特定 OpenCL 驱动或设备上的唯一标识符（UUID），以及增强异步直接内存访问（DMA）的能力，以更好地支持嵌入式处理器的应用场景。

来源：[cnbeta](https://www.cnbeta.com/articles/tech/1035659.htm)

> 拍一拍：值得一提的是，OpenCL 是首个面向异构系统通用并行计算而设计的开放且免费的标准，提供了一个统一的编程框架。

#### 摩托罗拉 68000 系列处理器的 LLVM 支持提案再现

尽管摩托罗拉 68000 系列处理器早在 1980 年代就随着早期苹果 Macintosh 电脑而广为人知，但直至今日它仍受到复古计算机爱好者及 DIY 爱好者的青睐。最近，有开发者再次提出了将对 M68k 架构的支持纳入 LLVM 编译器基础设施上游项目中的请求。这并不是首次尝试将 M68k 引入 LLVM，但鉴于其历史意义和技术价值，此次努力仍然值得我们关注。

来源：[phoronix](https://www.phoronix.com/scan.php?page=news_item&px=LLVM-Motorola-6800-Series-2020)
