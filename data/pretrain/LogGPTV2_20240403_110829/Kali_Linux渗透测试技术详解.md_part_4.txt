### Win32 Disk Imager

**图1.27 Win32 Disk Imager启动界面**

- **Image File**: 选择镜像文件
- **Device**: 选择目标设备 [G:\]
- **Copy mS Hash**: 复制哈希值
- **Progress**: 显示进度
- **Version**: 0.9
- **Cancel**: 取消操作
- **Read**: 读取数据
- **Write**: 写入数据
- **Exit**: 退出程序

### 使用Win32 Disk Imager写入Kali Linux镜像

1. 在启动界面，单击“文件”图标，选择 `kali-linux-1.0.6a-rpi.img` 镜像文件。此时将显示如图1.28所示的界面。
   
   **图1.28 添加映像文件**
   
2. 单击“Write”按钮，将显示如图1.29所示的确认界面。

   **图1.29 确认写入数据的磁盘**

3. 该界面会提示是否确定要将镜像写入到设备 [G:]。选择“Yes”，将显示如图1.30所示的写入进度界面。

   **图1.30 开始写入数据**

4. 数据写入完成后，将显示如图1.31所示的成功界面。

   **图1.31 完成写入数据**

5. 单击“OK”按钮，返回到图1.28所示的界面。然后单击“Exit”按钮，关闭Win32 Disk Imager工具。

6. 将SD卡从Windows系统中弹出，并插入树莓派。连接显示器、网线、鼠标、键盘和电源。几秒后，树莓派将启动Kali Linux操作系统。使用默认用户名 `root` 和密码 `toor` 登录。

### 远程登录到Kali Linux

#### 使用PuTTY进行SSH连接

1. 下载并安装PuTTY的Windows版本。
2. 启动PuTTY工具，显示如图1.32所示的界面。

   **图1.32 PuTTY工具**

3. 在“HostName (or IP address)”文本框中输入Kali系统的IP地址（例如：192.168.6.101），选择“Connection type”为“SSH”。单击“Open”按钮，将显示如图1.33所示的警告信息。

   **图1.33 警告信息**

4. 该警告信息是为了确认是否信任该服务器。单击“Yes”按钮，将显示如图1.34所示的登录界面。

   **图1.34 登录到Kali系统**

5. 输入Kali系统的默认用户名 `root` 和密码 `toor` 登录到系统。现在可以在命令行下运行任何命令。

#### 使用Xming远程连接图形界面

1. 从 [http://sourceforge.net/projects/xming/](http://sourceforge.net/projects/xming/) 下载Xming软件。
2. 启动下载的Xming软件，显示如图1.35所示的欢迎界面。

   **图1.35 欢迎界面**

3. 单击“Next”按钮，选择安装位置，显示如图1.36所示的界面。

   **图1.36 选择安装位置**

4. 单击“Next”按钮，选择安装组件，显示如图1.37所示的界面。

   **图1.37 选择安装组件**

5. 选择“Don't install an SSH client”组件，单击“Next”按钮，完成安装。

### 使用VMware创建虚拟机

1. 启动VMware，选择“新建虚拟机向导”，显示如图1.45所示的界面。

   **图1.45 新建虚拟机向导**

2. 选择“典型”方式安装虚拟机，单击“下一步”按钮，显示如图1.46所示的界面。

   **图1.46 安装客户机操作系统**

3. 选择“稍后安装操作系统”，单击“下一步”按钮，显示如图1.47所示的界面。

   **图1.47 选择客户机操作系统**

4. 选择Linux操作系统，版本为“其他Linux 2.6.x内核”，单击“下一步”按钮，显示如图1.48所示的界面。

   **图1.48 命名虚拟机**

5. 为虚拟机命名并设置安装位置，单击“下一步”按钮，显示如图1.49所示的界面。

   **图1.49 指定磁盘容量**

6. 设置磁盘容量（建议至少8GB），单击“下一步”按钮，完成虚拟机创建。

希望这些步骤能够帮助您顺利地使用Win32 Disk Imager写入Kali Linux镜像，并通过PuTTY和Xming远程连接到Kali Linux系统。
