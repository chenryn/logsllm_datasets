### Docker 简介

Docker 是一款开源软件，提供了一个开放平台，用于应用的开发、交付和运行。它允许用户将应用程序从基础设施中独立出来，封装成更小、更易于管理的单元（即容器），从而加速软件的部署与分发过程。

#### 容器 vs. 虚拟机

尽管 Docker 容器与虚拟机在某些方面相似，但它们的工作机制存在本质区别：
- **容器**通过操作系统级别的虚拟化技术实现隔离，共享宿主机的操作系统内核；
- **虚拟机**则通过模拟完整的硬件环境来运行独立的操作系统实例。
因此，相较于虚拟机而言，Docker 容器更加轻量级且高效，在资源利用上也更为灵活便捷。

#### 架构概览

根据 Docker 官方提供的架构图，整个系统主要由以下几个关键组件构成：
- **Docker 客户端**：用户用来与 Docker 交互的工具；
- **宿主机**：运行 Docker 容器的实际物理或虚拟服务器，其中包括了 Docker 守护进程 (`dockerd`) 以及本地存储的容器和镜像；
- **Docker 镜像仓库**：集中存储 Docker 镜像的地方，便于开发者们分享及获取所需的镜像资源。

特别地，`dockerd` 作为核心服务之一，负责监听来自客户端或其他系统的 API 请求，并对诸如容器创建、启动等操作进行统一管理和调度。
