### 错误代码及说明

#### 设备分类器 (Device Classifier) 错误
- **DC-3-FILE_READ_ERROR: 读取配置文件 [chars] 时出错**
  - 无法读取配置文件内容。错误文件中的配置文件将无法加载到设备分类器中。
- **DC-3-FILE_CHECKSUM: 配置文件 [chars] 的校验和 [chars] 失败**
  - 配置文件的校验和计算或验证失败。错误文件中的配置文件将无法加载到设备分类器中。
- **DC-3-SM_GET: 无法获取 [chars]**
  - 与会话管理器 (Session Manager) 通信时发生内部错误。这可能导致 `show` 命令无法显示设备分类结果。
- **DC-3-DISABLE_FAIL: 无法禁用 DC**
  - 尝试禁用设备分类器时发生内部错误。
- **DC-3-ENABLE_FAIL: 无法启用 DC**
  - 尝试启用设备分类器时发生内部错误。
- **DC-3-HRPC_SYNC: 成员交换机 [dec] 上的错误，错误代码 = [dec]**
  - 尝试同步设备分类器状态时发生内部错误。
- **DC-4-FILE_OPEN_WARNING: 无法打开 [chars]**
  - 设备分类器未找到默认配置文件或文件可能已损坏。设备分类器将使用内置配置文件工作。
- **DC-3-PATTERN_NOT_FOUND: [chars]**
  - 设备配置文件解析失败。这表明文件可能已损坏。如果嵌入式配置文件失败，设备分类器将处于操作中断状态。
- **DC-3-INCOMPATIBLE_PROFILE_VERSION: 在 [chars] 中检测到不兼容的配置文件版本。期望版本: [dec]，找到的版本: [dec]**
  - 检测到不兼容的配置文件版本。设备分类器功能将处于操作中断状态。
- **DC-3-BUILTIN_INIT_FAIL: [chars]**
  - 设备分类器内置数据库初始化失败。因此，设备分类器将处于操作中断状态。
- **DC-6-DEFAULT_INIT_INFO: [chars]**
  - 默认配置文件未加载，相关设备将不会被分类。设备分类器将继续使用内置配置文件工作。
- **DC-3-SM_INIT_FAIL: 设备分类器已禁用。[chars] 初始化失败**
  - 设备分类器在会话管理器 (SM) 中注册失败。因此，设备分类器将处于操作中断状态。
- **DC-3-OUI_INIT_FAIL: [chars] 的 OUI 表初始化失败**
  - 设备分类器中的 OUI 数据库加载失败。
- **DC-3-CHECK_INIT_FAIL: [chars]**
  - 设备分类器中的检查数据库加载失败。
- **DC-3-DB_INIT_FAIL: 设备分类器已禁用。[chars] 初始化失败**
  - 设备分类器内部数据库初始化失败。因此，设备分类器将处于操作中断状态。
- **DC-3-MAX_RULES_EXCEEDED: [chars]**
  - 设备配置文件中定义的规则数量超过了最大限制。如果嵌入式配置文件失败，设备分类器将处于操作中断状态。
- **DC-3-MAX_PROFILES_EXCEEDED: [chars]**
  - 设备配置文件中定义的配置文件数量超过了最大限制。如果嵌入式配置文件失败，设备分类器将处于操作中断状态。
- **DC-3-MAX_OUI_EXCEEDED: [chars]**
  - 设备配置文件中定义的 OUI 代码数量超过了最大限制。如果嵌入式配置文件失败，设备分类器将处于操作中断状态。
- **DC-3-NO_MEM: [chars]**
  - 系统中没有可用内存进行分配。如果嵌入式配置文件失败，设备分类器将处于操作中断状态。
- **DC-3-DC_DISABLED: [chars]**
  - 由于在初始化或解析配置文件时遇到错误条件，设备分类器已被禁用。
- **DC-3-SM_UPDATE: 无法更新 [chars]**
  - 设备分类过程中发生内部错误。这可能导致设备状态未知。
- **DC-3-SM_API: [chars] 返回 [chars]**
  - 会话管理器 API 返回内部错误。

  - 来自 DFP 管理器的消息具有未知类型。
- **DFP-4-ENQUEUE: 进程入队失败: [chars]**
  - 将 DFP 元素入队时发生意外错误。
- **DFP-4-BAD_POINTER: [chars] 应用程序未设置指针**

- **DFP-4-NUM_TOO_LARGE: [chars] 应用程序发送了太多值**
  - 应用程序向 DFP 子系统发送了太多值。
- **DFP-4-AGENT_NOT_FOUND: [chars] 服务未在 DFP 中注册**
  - 无此服务在 DFP 子系统中注册。
- **DFP-4-BAD_LISTEN: 服务 [chars] - 监听失败**

- **DFP-4-NO_PROCESS_START: 无法启动服务 [chars] 的进程**

- **DFP-4-STRING_TOO_LONG: 字符串 [chars] 超过 15 个字符 - 对于服务名称太长**
  - 用户输入的服务字符串超过 15 个字符。
- **DFP-4-PACKET_TOO_SMALL: 来自管理器 [inet] 服务 [chars] 的 DFP 包太小**
  - 接收到的 DFP 包太小。
- **DFP-4-REPEAT_SERVICE_INIT: 服务 [chars] 已经初始化，无法重新初始化**
  - 应用程序试图重新初始化一个已经初始化的服务。
- **DFP-4-NO_QUEUE_REQUEST: 无法为服务 [chars] 的 [chars] 排队请求**
  - 无法为特定操作向 DFP 进程排队请求。
- **DFP-4-SUBSYS_NOT_UP: 服务 [chars] - 在 DFP 子系统启动之前尝试注册**
  - 子系统初始化顺序出现问题。
- **DFP-4-SECURITY_FAIL: 管理器 [inet] 服务 [chars] 的 CASA 包中的安全信息 [chars]**
  - 安全检查失败。

  - 套接字绑定调用出现问题。

#### DHCP 错误
- **DHCP-6-ADDRESS_ASSIGN: 接口 [chars] 分配了 DHCP 地址 [inet] 掩码 [inet] 主机名 [chars]**
  - 接口通过 DHCP 分配了一个地址。

  - DHCP 正在重启接口。
- **DHCP-3-HA_INIT_FAIL: DHCP 客户端中的函数 [chars] 初始化失败: [chars]**
  - DHCP 客户端中的高可用性相关函数未能正确初始化。如果您使用的是具有冗余处理器或盒对盒冗余性的平台，DHCP 客户端设置可能在主处理器故障时无法保留，从而强制使用备用处理器。
- **DHCPD-2-NO_PROCESS: DHCP 无法启动其 [chars] 进程**
  - DHCP 无法启动其某个进程。
- **DHCPD-2-SOCKET_ERROR: DHCP 无法打开 UDP 端口 [dec]**
  - DHCP 无法打开 UDP 端口 67 (bootps)。
- **DHCPD-2-MALLOC_ERROR: DHCP 服务内存不足**
  - DHCP 无法分配必要的数据结构。
- **DHCPD-2-RECVMSG_ERROR: Recvmsg 意外失败。DHCP 服务已终止**
  - 失败原因未知。
- **DHCPD-4-RELAY_ONLY: DHCP 将仅作为中继代理运行**
  - DHCP 无法启动其数据库进程。服务器组件已禁用。只有中继代理是活动的。
- **DHCPD-3-READ_ERROR: DHCP 无法从 [chars] 读取绑定**

- **DHCPD-2-READ_DEADLOCK: DHCP 检测到死锁条件 - 读取失败过多**

- **DHCPD-3-WRITE_ERROR: DHCP 无法将绑定写入 [chars]**
  - DHCP 服务器无法将绑定写入指定的数据库代理。
- **DHCPD-4-PING_CONFLICT: DHCP 地址冲突: 服务器 ping 了 %I**
  - DHCP 服务器检测到地址冲突。另一台主机正在使用指定的 IP 地址。解决冲突后，请执行 `clear ip dhcp conflict` 命令。
- **DHCPD-4-DECLINE_CONFLICT: DHCP 地址冲突: 客户端 [chars] 拒绝了 %I**
  - DHCP 客户端检测到地址冲突。另一台主机正在使用指定的 IP 地址。解决冲突后，请执行 `clear ip dhcp conflict` 命令。
- **DHCPD-4-HA_CONFIG_MISMATCH: 活动 DHCP 池 [chars] 配置与备用不匹配**
  - 失败原因未知。
- **DHCPD-6-HIGH_UTIL: 池 \\[chars]\\ 处于高利用率状态，已使用 [dec] 个地址，总共 [dec] 个。阈值设置为 [dec]%**
  - 池的利用率已超过用户配置的高水位线。

  - DHCP 客户端发送了格式错误的选项。

  - DHCP 服务器中的高可用性相关函数未能正确初始化。如果您使用的是具有冗余处理器或盒对盒冗余性的平台，DHCP 服务器数据可能在主处理器故障时无法保留，从而强制使用备用处理器。
