### Linux 4.0 发布——代号“Hurr durr I'ma sheep”

**作者：Linus Torvalds**
**类别：新闻**
**标签：内核, Linux**

就在周日，Linus Torvalds 如期发布了 Linux 4.0。这个版本的代号被称为“[Hurr durr I'ma sheep](https://imasheep.hurrdurr.org/)”。

#### 发布详情
- **日期**: 2015/4/12 15:41:30 周日 -0700
- **主题**: Linux 4.0 发布
- **来自**: 我是一只羊

#### 发布说明
根据 Linus 的邮件列表公告，他决定按照正常的计划发布 4.0 版本，因为没有出现什么问题，并且他下周会去一个大学访问。他希望这不会对合并窗口产生太大影响。

从 `linux-next` 分支和最终大小来看，Linux 4.0 是一个相对较小的发布版本。尽管如此，它仍然包含了超过一万个非合并提交。预计未来的 `linux-next` 4.1 版本会更大。

#### 稳定性与功能
4.0 版本的主要目标是保持稳定性，而不是引入大量新功能。这一版本完全符合“v4.0 应该是一个稳定的发布版本”的要求。Linus 表示，他更喜欢按时间进行发布，而不是等到有重大功能才发布。

尽管 4.0 版本没有特别有趣的新功能，但它在 git 统计上达到了新的里程碑：
- 超过了 50 万个提交数
- 达到了 400 万个 git 对象数量

这些数字显示出开发工作的显著增长。例如，在使用 BitKeeper (BK) 时，提交数用 16 位二进制存储，限制了 65000 次提交。而自切换到 Git 以来，开发速度大大加快。

#### 主要更新
虽然 4.0 版本没有特别突出的新功能，但还是有一些值得关注的亮点：
- **Live Patching**: 无需重启即可为内核打补丁的机制。
- **parallel NFS (pNFS)**: 子系统支持 FlexFile 布局。
- **ubifs 文件系统**: 支持多队列块层。
- **Btrfs 文件系统**: 更新支持 Intel Quark X1000 SoC 和基于 MIPS32 Release 6 的处理器。
- **VirtIO 1.0**。
- **可信计算 TPM 2.0 支持**。
