### 规则更新警报
当亚信安全规则更新已下载并可使用超过三十分钟，但电脑仍未进行更新时，系统将触发警报。

### 代理服务器配置
如果中继必须通过代理服务器访问互联网（包括亚信安全更新服务器），您可以在管理中心的“管理 >> 系统设置 >> 代理服务器”选项卡下的“代理服务器”区域定义代理服务器设置。

### 安全更新
#### 启动全系统安全更新
1. 转至“管理 >> 更新 >> 安全”。
2. 单击“检查更新并下载”按钮。

#### 对特定客户端/设备执行安全更新
1. 在“计算机”页面的计算机列表中选择目标客户端/设备。
2. 右键单击并选择“操作 >> 下载安全更新”。

#### 预设常规安全更新任务
1. 转至“管理 >> 预设任务”。
2. 创建一个新的“检查安全更新”类型的任务。

### 中继组管理
#### 将已启用中继的客户端组织成中继组
1. 至少需要一个已启用中继的客户端。
2. 激活后，该客户端将被添加到名为“默认中继组”的组中。
3. 您可以将新的已启用中继客户端从一个中继组移动到另一个中继组。
4. 查看或创建新的中继组，请转至“管理 >> 更新 >> 中继组”。

#### 分配更新源
- 默认情况下，中继组从“管理 >> 系统设置 >> 更新”选项卡上指定的更新源获取更新。
- 可以将中继组配置为从其他中继组获取更新，从而创建中继组的层次结构。
- 已启用中继的客户端必须从层级较高的中继组或已配置的安全更新源处获取更新。

#### 将中继组分配给客户端/设备
1. 转至“计算机”页面。
2. 右键单击目标计算机并选择“操作 >> 分配中继组...”。
3. 从显示的可用中继组列表中选择所需的中继组。

### 特殊环境中的更新配置
#### 在气隙环境中配置更新
1. 至少配置一个已启用中继的客户端，该客户端可以从亚信安全更新服务器下载更新。
2. 其他已启用中继的客户端或设备连接到该客户端以分发更新。
3. 如果不允许已启用中继的客户端通过互联网连接，则可以将更新包导入已启用中继的客户端以分发更新。

#### 生成和导入更新包
1. **生成更新包**：
   - 在已启用中继客户端上的命令行中输入：`dsa_control -b`
   - 命令行输出将显示生成的 .zip 文件的名称和位置。

2. **导入更新包**：
   - 将生成的 .zip 文件复制到已启用中继客户端的安装目录（Windows: "C:\Program Files\AsiaInfo Security\Deep Security Agent"；Linux: "/opt/ds_agent"）。
   - 当启动安全更新下载时，如果客户端无法从配置的更新源获取更新，它将自动检查安装目录中的 .zip 文件并导入更新。

#### 配置气隙中继客户端的更新源
1. 转至“管理 >> 系统设置 >> 更新 >> 主安全更新源”。
2. 选择“其他更新源”并输入 `https://localhost:4122`。
3. 单击“确定”。

### 软件更新
#### 管理和分发软件更新
1. 管理中心定期连接到亚信安全更新服务器检查更新。
2. 接收到新软件可用警报后，将软件导入管理中心。
3. 导入步骤：
   - 转至“管理 >> 更新 >> 软件”。
   - 选择需要的软件包并单击“导入”。
   - 当“已导入”列中出现绿色复选标记时，表示软件包已成功下载。
   - 升级特定客户端或所有客户端：
     - 右键单击目标客户端并选择“操作 >> 升级客户端/设备/中继软件”。
     - 或者，转至“管理 >> 更新 >> 软件”，然后单击“升级客户端/设备软件”。

### 虚拟设备扫描缓存
#### 简介
- 扫描缓存用于提高虚拟机防恶意软件扫描和完整性监控扫描的效率。
- 缓存包含已扫描文件和其他扫描目标的列表，避免重复扫描相同内容。
- 通过共享扫描结果，克隆虚拟机或类似虚拟机的扫描性能得到提升。

#### 配置
- **查看现有缓存配置**：
  - 转至“管理 >> 系统设置 >> 高级 >> 扫描缓存配置”并单击“查看扫描缓存配置...”按钮。
- **到期时间**：
  - 默认设置：手动/预设恶意软件扫描为一天，实时恶意软件扫描为 15 分钟，完整性监控扫描为一天。
- **使用 USN (仅限 Windows)**：
  - 指定是否利用 Windows NTFS 更新序列号来记录文件更改。
- **包括和排除文件**：
  - 使用正则表达式和通配符来确定哪些文件应包含在缓存中或从中排除。

#### 设置
- **最大并发扫描数**：
  - 确定虚拟设备同时执行的扫描数量，默认建议为 5。
- **最大缓存条目数**：
  - 确定不同类型扫描的最大记录数，例如恶意软件手动/预设扫描、实时扫描和完整性监控扫描。

### 用户管理
#### 用户、角色和联系人
- **用户**：具有唯一用户名和密码的帐户持有者，分配有角色。
- **角色**：权限集合，用于在管理中心查看数据和执行操作。
- **联系人**：不具有用户帐户，但可以接收电子邮件通知和预设报告。

#### 基于角色的访问权限和编辑权限
- 访问权限和编辑权限附加到角色，而不是用户。
- 更改单个用户的权限，需重新分配角色或编辑角色本身。

#### 角色的权限设置
- **完全访问权限角色**：具有所有计算机和策略的查看、编辑、删除等权限。
- **审计员角色**：具有所有计算机和策略的查看权限，但无编辑和删除权限。
- **新建角色**：默认具有基本的查看权限，但无编辑和删除权限。
