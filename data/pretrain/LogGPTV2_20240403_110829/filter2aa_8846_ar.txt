**30. 拒绝服务攻击的一种物理防范方式是拔掉服务器的网线。**

**31. 关于安全性的一句名言是：“如果攻击者无限制地对计算机进行物理访问，那么攻击者就可以完全控制它”（If an attacker has unrestricted physical access to a computer, the attacker owns it.）。例如，攻击者可以对身份认证服务器进行物理访问，并在短时间内盗取密码并进行离线破解。**

**34. “知其所需”（need to know）和“最小特权”（least privilege）经常被混淆。唯一的区别在于，“最小特权”还包括在系统上的操作权限，而“知其所需”主要体现在业务语境下，最小特权则更侧重于技术权利。**

**OSG 测试：**

1. **字典攻击、暴力攻击、中间人攻击和虚假登录页面都是对访问控制机制的威胁，但拒绝服务攻击和钓鱼攻击不是。**

2. **CSRF（跨站请求伪造），也被称为“One Click Attack”或Session Riding，通常缩写为CSRF或XSRF，是一种针对网站的恶意利用。尽管听起来与跨站脚本（XSS）类似，但它与XSS不同。XSS利用站点内的信任用户，而CSRF则是通过伪装成受信任用户的请求来利用受信任的网站。**

3. **例如，一个网站用户Bob可能正在浏览聊天论坛，同时另一个用户Alice也在该论坛中，并且Alice发布了一条包含Bob银行链接的图片消息。假设Alice编写了一个提交取款表单的链接，并将其作为图片的src属性。如果Bob的银行在cookie中保存了他的授权信息且该cookie未过期，那么当Bob的浏览器尝试加载图片时，将提交这个取款表单和他的cookie，从而在未经Bob同意的情况下授权了这次交易。**

4. **XSS 是实现 CSRF 的一种途径，通常通过 XSS 实现的 CSRF 被称为 XSRF。CSRF 的全称是“跨站请求伪造”，而 XSS 的全称是“跨站脚本”。它们都属于跨站攻击，即不直接攻击服务器端而是攻击正常访问网站的用户。CSRF 通过伪造请求冒充用户在站内的正常操作。大多数网站通过 cookie 等方式识别用户身份（包括使用服务器端 Session 的网站，因为 Session ID 通常也存储在 cookie 中），因此要伪造用户的正常操作，最好的方法是通过 XSS 或链接欺骗等途径，让用户在拥有身份 cookie 的浏览器端发起未知请求。**

5. **XSS 更偏向于方法论，而 CSRF 更偏向于一种形式。只要是伪造用户发起的请求，都可以视为 CSRF 攻击。**

6. **在生物识别中，如果错误地将指纹识别为另一个客户，这是类型2错误；如果错误地拒绝了一个合法客户，这是类型1错误。**

7. **带外身份验证是指使用电话或短信等方式来验证用户身份。**

8. **OpenID 是一种去中心化的网上身份认证系统。用户只需在一个 OpenID 身份提供商（IdP）的网站上注册。任何网站都可以使用 OpenID 作为用户登录的方式，也可以成为 OpenID 身份提供商。你的网站地址（URI）就是你的用户名，而你的密码安全地存储在一个 OpenID 服务网站上（你可以自己建立一个 OpenID 服务网站，也可以选择一个可信赖的 OpenID 服务网站进行注册）。**

9. **登录支持 OpenID 的网站非常简单。只需输入你注册的 OpenID 用户名，然后登录的网站会跳转到你的 OpenID 服务网站，在那里输入密码（或其他需要填写的信息）验证通过后，你会回到登录的网站并成功登录。除了“一处注册，到处通行”之外，OpenID 还为所有支持它的网站带来了共享用户资源的价值。**

10. **OpenID 强调的是身份认证（authentication），而 OAuth 强调的是授权（authorization）。**

11. **LDAP 的 SASL 模式提供了多种身份验证类型的支持，包括安全的方法。**

**领域 6 - 安全评估和测试**

**第 15 章 安全评估和测试**

1. **安全测试（Testing）包括自动扫描、渗透测试和手动测试。**

2. **安全评估（Assessment）是对系统、应用或其他测试环境的综合评价。安全评估不仅包括使用安全测试工具（如漏洞扫描和渗透测试），还包括彻底审查威胁环境、当前和未来面临的风险以及目标环境的价值。安全评估的产物是一份评估报告，以非技术语言描述评估结果，并提出具体建议。**

3. **安全审计类似于安全评估，但必须由独立的审计员执行。测试和评估主要是内部使用的，而审计的目的则是向第三方展示，阅读对象可能是董事会、政府监管人员和其他第三方。**

4. **审计分为两种：内部审计和外部审计。**
   - **内部审计：首席审计执行官直接向总裁、CEO 或董事会汇报。**
   - **外部审计：由外部审计公司执行，这些审计具有较高的外部效度（即通用程度较高）。**

5. **信息安全专家通常会被要求参与内部和外部审计，主要职责是提供相关信息、接受面谈和提供文档。审计员有权了解所有信息，主管安全的工作人员应予以配合，如有需要可以咨询管理层。**

6. **漏洞扫描有三种类型：网络发现、网络漏洞扫描和 Web 漏洞扫描。**

7. **网络发现：**
   - **TCP SYN 扫描（半开放扫描）**
   - **TCP 连接扫描（全连接扫描）**
   - **TCP ACK 扫描**
   - **Xmas 扫描（打开 FIN、PSH 和 URG 标志位）**

8. **常见的网络扫描工具是 nmap，扫描结果包括：开放、关闭和过滤（无法确定状态，因为防火墙干扰）。**

9. **网络漏洞扫描可能会出现假阳性（FP）和假阴性（FN）。可以使用 Nessus 漏洞扫描器工具。**

10. **Web 漏洞扫描会检测 SQL 注入、XSS、XSRF（跨站请求伪造）等漏洞。仍然可以使用 Nessus。新系统上线前和新代码进入生产之前都应该进行 Web 漏洞扫描。PCI DSS 要求每年进行一次 Web 漏洞扫描，或者安装 WAF。**

11. **渗透测试常常使用 Metasploit 工具，该工具使用脚本语言允许自动执行常见的攻击。**
    - **白盒测试：提供目标系统的详细信息，使攻击者节省很多侦查步骤。**
    - **黑盒测试：不提供任何信息。**
    - **灰盒测试：提供部分信息。**
    - **渗透测试报告应注意对漏洞数据的保存和发送。**

12. **最正式的代码评审过程称为 Fagan 检查，包括以下六个步骤：**
    - **计划（Planning）**
    - **概览（Overview）**
    - **准备（Preparation）**
    - **检查（Inspection）**
    - **返工（Rework）**
    - **后续行动（Follow-Up）**

13. **正式的 Fagan 检查通常只存在于高度受限的环境中，其中代码如果有缺陷可能会产生灾难性影响。**

14. **大多数公司不使用如此严格的方法，他们通常采用以下做法进行代码审查：**
    - **在会议上，开发人员过一遍代码，其他成员进行评审。**
    - **高级开发人员手工审查并在生产前签字确认。**
    - **在生产前使用自动审查工具进行检查。**

15. **动态测试包括使用综合事务（synthetic transactions）验证系统的性能，这些是已知预期结果的脚本事务。比较结果有偏差就表明存在 Bug。**

16. **模糊测试（Fuzz Testing）是一种专门的动态测试技术，通过生成大量不同类型输入来压测系统极限，发现以前未被发现的缺陷。模糊测试监控程序的性能、崩溃、溢出或其他不可预测的结果。**

17. **模糊测试有两种：**
    - **变异模糊测试（Mutation/Dumb Fuzzing）：主要是在以前的输入基础上做一些变化，改变字符或添加字符串。**
    - **智能模糊测试（Generational/Intelligent Fuzzing）：开发数据模型并创建新的模糊输入。**
    - **zzuf 工具可以自动化变异模糊测试，自动对文本进行一些变化。**

18. **接口测试主要包括以下三种：**
    - **API 测试（对外公开的应用编程接口的服务需要测试，如 RESTful 接口）**
    - **用户界面（UI）测试，包括图形界面和命令行界面（CLI）**
    - **物理接口测试**

19. **对于一些已知的风险，如黑客通过更改银行账户 ID 来获取他人账号信息的情况，软件测试人员会使用误用案例测试（Misuse Case Testing）或滥用用例测试（Abuse Case Testing）来评估软件对这些已知风险的脆弱性。**

20. **测试覆盖率 = 已经测试的用例数 / 总用例数。准确计算覆盖率很难，因为总用例数需要穷举所有可能情况。解释覆盖率时需要了解这些数字是如何得来的。**

21. **信息安全管理人员应定期审查日志，以确保特权用户不滥用特权。尤其是信息安全团队的人，很有可能接触到用户隐私。**

22. **账户管理审核可能是信息安全管理人员或内部审计师的职责，通常会对高特权账户进行审核。一种方法是：**
    - **管理人员要求管理员提供特权用户列表及其被授予的特权，管理人员会在旁边监督，以防管理员做手脚。**
    - **管理员要求特权批准机构提供授权用户的列表。**
    - **对比两个列表发现问题。**

23. **由于没有时间对所有用户进行检查，此时可以抽样。如果在抽样中没有发现问题，则认为这些样本代表所有账户。注意：只有随机抽样才是有效的！**

24. **管理人员应定期检查数据备份的有效性，检查方法包括：审核日志、检查散列值、实际恢复系统或文件。**

25. **安全管理人员需持续监视关键性能指标和风险指标，如漏洞数量、漏洞修复时间、受攻击次数等。识别这些关键安全度量后，管理人员希望开发一个仪表盘来显示并时常查看这些指标。**

**OSG 测试：**

1. **突变测试（Mutation Testing），也称为突变分析或程序突变，用于衡量软件测试的质量。突变测试通过对程序的源代码做小改动，如果测试代码没有觉察到这种小改动带来的错误，就说明这个测试有问题。**
    - **示例：正常程序：`if (a < 0) Assert.fail();`**
    - **突变测试：`a=-1; if (a < 0) Assert.fail();`**
    - **执行代码后如果没有抛出断言失败，表明测试存在问题。**

2. **Nikto 是扫描 Web 服务器的最佳选择。**
    - **THC Hydra：用于密码暴力破解。**
    - **Ettercap：用于中间人攻击。**

3. **OpenAVS 是开源的漏洞扫描工具，Nessus 是闭源的；Nmap 主要用于端口扫描；MBSA 是微软的基线安全分析器（Microsoft Baseline Security Analyzer）。**

4. **服务组织报告 SSAE-16 SOC 在 2010 年取代了 SAS-70 报告。**
    - **SOC Type 1 报告仅涵盖某一时间点。**
    - **SOC Type 2 报告至少覆盖 6 个月。**
    - **SSAE 基于 ISAE 3402（国际鉴证业务准则第 3402 号）。**

5. **SOC 1 仅涉及财务控制方面的内容，SOC 2 向业务合作伙伴、监管机构提供组织的机密性、完整性和可用性（CIA）报告，SOC 3 则向更广泛的受众提供关于组织服务运营的 CIA 报告。**
    - **SOC 1 报告涵盖财务报告控制。**
    - **SOC 2 报告涵盖安全、可用性、完整性和隐私，适用于业务合作伙伴、监管机构及其他类似组织。**
    - **SOC 3 报告旨在广泛传播，通常带有网站印章，支持组织对其提供完整性、可用性和保密性的声明。**

6. **WPA2 的企业模式使用 RADIUS 进行身份验证，多次尝试后可能会被锁定，因此使用 aircrack-ng 进行口令攻击很难成功。**
    - **Aircrack 是破解 WEP/WPA/WPA2 加密的主要工具之一，目前排名第一。**
    - **Aircrack-ng 套件包含捕获数据包、握手验证等工具，可用于暴力破解和字典攻击。**

7. **Windows 默认不支持 syslog。**

8. **合成监测（Synthetic Monitoring），又称综合测试或主动监测，是一种主动监控系统性能的方法。**
