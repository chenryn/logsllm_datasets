### 问题描述

我们在将大量用户的主目录迁移到运行 Windows Storage Server 2008 R2 的新服务器上。对于大多数用户，迁移过程顺利，组策略成功地将用户的“我的文档”文件夹重定向到新的网络路径（`\\server\Users\%username%\Work`）。

然而，在少数几台运行 Windows 7 的 PC 上，文档库没有自动添加用户重定向的“我的文档”路径。当我尝试手动添加该路径时，系统提示：“此网络位置无法包含，因为它未被索引。”

尽管 Windows Search 服务已安装在服务器上，并且 Users 目录已启用索引，但仍然出现上述错误。即使我手动解决了这个问题并将用户的主目录添加到文档库中，Windows 仍会显示“由于不支持的库位置，某些库功能不可用”。

### 解决方案

为了使 Windows Search 服务能够索引某个文件夹/共享，服务器上的本地 SYSTEM 账户需要具有 NTFS 权限。

在设置这些权限后，组策略能够成功将用户的主目录添加到文档库中，我也能够手动完成这一操作，而不会再收到“此网络位置无法包含，因为它未被索引”的消息。

### 建议

如果能够在系统事件日志中记录一条错误信息，提示 Windows Search 服务没有所需的权限来索引某个文件夹，那将会非常有帮助。这样可以更快速地诊断和解决问题。
