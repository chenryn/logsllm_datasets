两年前，微软做出了一个令人意想不到的决定：宣布将其 .Net 开发框架开源，包括 .Net Core 和 ASP.Net。这一公告引起了广泛关注，因为长期以来，微软一直对开源项目持谨慎态度，担心它们会破坏其软件生态系统。

随着新任CEO萨提亚·纳德拉提出“微软爱Linux”的口号，微软开始在开源领域频繁动作。不仅在GitHub上的多个仓库表现出活跃态势，还积极与其他开源厂商展开合作。

![](/data/attachment/album/201612/19/201801d54yyzqajmpay0aj.png)

无论微软拥抱开源背后的动机是什么，无论是为了促进软件销售、云服务的推广，还是吸引更多开发者加入其平台，目前看来，这一策略已经取得了成功。.Net 和 ASP.Net 开源后，微软的开源战略已经显现出了价值。

微软将 .Net Core 定位为一个跨平台开发框架，不仅支持 Windows，还扩展到了 MacOS 和 Linux 平台。.Net Core RC1 在 2015 年 11 月发布到 GitHub 上，而 2016 年 6 月就发布了 1.0 版本。开发者对此表示认可，微软合作伙伴总监项目经理 Scott Hunter 表示：

> “40% 的 .Net Core 用户是新进入该平台的用户，这就是我们开源的目的——吸引新用户。”

由于 .Net Core 的开源，过去一年中 .Net 开发者数量增加了 61%。Hunter 在 2016 年 11 月的一次演讲中提到，GitHub 上 .Net 的开发活跃度增长迅速。

尽管 .Net Core 并没有直接为微软带来收入，但它间接地通过 Azure 服务和微软开发者工具增加了公司的收益。Rob Sanfilippo 说：“可以说，这项技术间接地通过 Azure 服务和微软开发者工具增加了微软的收入。”

### 程序员们积极参与

.Net 程序员和博客作者 Matt Warren 称 .Net 的开源为“成功”。他指出，社区积极参与了微软在 GitHub 上开源的多个仓库的活动。

“开源社区报告了问题和建议，并通过发送拉取请求（PR）实际贡献了代码。随着时间的推移，这些代码被纳入产品中，社区的贡献量还在持续增长。”Warren 说，“我积极跟踪和参与了 CoreCLR 和 .Net Core Lab 这些仓库的讨论，因此获得了社区贡献的第一手资料。”

虽然微软的举措是否完全成功尚不能定论，但这毕竟是从封闭走向开放的重要一步。他说：“我的意思是，他们不仅仅是把源代码放在那里，而是努力让社区参与进来。”

### Visual Studio for Mac 带来的利好

微软最近发布的 Visual Studio for Mac 也为 .Net Core 带来了新的机遇。“这是 Visual Studio IDE 首次发布到非 Windows 平台上（Visual Studio Code 是不同的技术，且不算作 IDE），它是基于微软收购的 Xamarin 技术，重点关注 .Net Core 开发。”Sanfillippo 说，“这次发布进一步推动了 .Net Core 的发展。”

微软还计划在即将推出的 Visual Studio 2017 IDE 中增强 .Net Core 工具，简化 .Net Core 项目文件的格式。

微软开源 .Net 改变了外界对其的看法，Warren 说：“现在感觉（微软）更加开放和平易近人了。”

虽然开源社区对微软仍存有一定疑虑，但显然，微软已经不再是开源社区最敌视的公司之一。
