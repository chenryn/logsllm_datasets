虽然 Linux 相对于 Windows 10 在安全性方面具有一定的优势，但 Linux 并非绝对安全。根据美国国家标准技术研究院（National Institute of Standards and Technology, NIST）的国家漏洞数据库（National Vulnerability Database, NVD）最新数据，这一点得到了证实。

### Windows 中的安全缺陷较少

数据显示，在 1999 至 2019 年期间，Debian Linux 是所有操作系统中存在最多安全漏洞的一个。在此期间，Debian Linux 报告了 3067 个漏洞，Ubuntu 报告了 2007 个漏洞。相比之下，Windows 7 受到 1283 个漏洞的影响，而 Windows 10 则仅有 1111 个漏洞。

尽管历史上 Windows 并未特别强调安全性设计，但从 Windows XP 开始，微软逐渐增加了许多安全功能和强大的防火墙以应对日益增长的安全威胁。此外，微软还不断为 Windows 添加新的安全性和隐私特性。然而，Windows 的主要目标仍然是满足大多数个人和商业计算机的需求。

![](/data/attachment/album/202003/10/102307vvuc3cxwsddun77r.jpg)

值得注意的是，对于早期版本如 Windows 7 而言，其在 2019 年发现的安全缺陷（250 个）实际上少于同期 Windows 10 的数量（357 个）。

重要的是要认识到，计算机上的许多漏洞源于硬件组件（例如芯片组和驱动程序），而非操作系统本身。同时，很多针对 Windows 的攻击是面向企业的，因此某些漏洞可能对普通用户影响不大。

不过，值得指出的是，Debian Linux 社区通常能在几天内修复已知漏洞，而 Windows 用户有时则需等待一个月才能收到补丁。

![](/data/attachment/album/202003/10/094211nx61ihw32iygsf41.jpg)

### 整体而言微软产品线存在最多安全缺陷

回顾过去二十年的数据可以提供更加全面的视角。如果考虑整个公司的产品线，微软产品的安全缺陷数量超过任何其他公司。自 1999 年以来，共发现了 6814 个与微软相关的产品漏洞。以下是排名前五的企业及其对应的安全缺陷数量：

1. 微软：6814 个
2. 甲骨文：6115 个
3. IBM：4679 个
4. 谷歌：4572 个
5. 苹果：4512 个

![](/data/attachment/album/202003/10/101952dv8vooq2u7y3ota1.jpg)

### Android 的安全缺陷数量持续领先

近年来，在所有技术产品中，谷歌的 Android 操作系统连续多年保持较高的安全缺陷数量。据报告，Android 在 2019 年、2017 年及 2016 年均是所有操作系统中安全缺陷最多的。仅在 2018 年，Debian GNU/Linux 的安全缺陷超过了 Android。2019 年，Android 以 414 个安全缺陷位居榜首，Debian Linux 以 360 个紧随其后，而 Windows 10/Windows Server 2016&2019 则以 357 个位列第三。

![](/data/attachment/album/202003/10/102042qh3lrzk6lqxrhs1o.jpg)

令人担忧的趋势是，随着操作系统及其他软件产品变得越来越复杂，近二十年来报告的技术漏洞数量显著增加。1999 年仅记录了 894 个技术漏洞；而到了 2019 年，这一数字激增至 12174 个，增幅超过 14 倍。

对此，一位 Android 发言人表示：
> 
> “我们致力于提高透明度，并每月发布关于 Android 已解决的安全问题的公共安全公告，以增强生态系统的安全性。我们认为，衡量操作系统中已解决的安全问题的数量并不能准确反映平台的安全性。这实际上是 Android 生态系统开放性的体现。”
>
