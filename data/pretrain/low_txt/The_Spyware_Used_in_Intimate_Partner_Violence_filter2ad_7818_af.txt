#### A. 查询滚雪球、种子查询和初始过滤
我们使用查询滚雪球技术来查找全面的搜索词，利用搜索引擎提供的推荐API。在图8中，我们给出了滚雪球方法的伪代码。该算法输入一组种子查询和最大返回查询数。我们为Google和Google Play使用不同的种子查询，因为它们的查询推荐API性质不同。前者建议语义相关的查询，而后者仅提供查询补全。图9列出了两种搜索的初始种子查询。

两个推荐API都返回大量不相关的搜索词，Google比Google Play更甚。例如，“cheating”被扩展为游戏“cheating tom”，或者“cheating husband”被扩展为“cheating husband quotes”，这些都不是我们想要的结果。随着我们反复扩展搜索结果，不相关的查询会将滚雪球引向不相关术语。因此，我们过滤掉包含图9（右表）列出的任何单词（正则表达式）的建议查询。

```plaintext
QSnowball(Qinit, limit):
    Q <- empty set
    while Qinit is not empty:
        q <- pop(Qinit)
        Q <- Q union {q}
        if |Q| >= limit then return Q
        Y <- recommend(q)
        Y_filtered <- filter(Y) \ Q
        Qinit <- Qinit union Y_filtered
    return Q
```

图8：查询滚雪球方法。这里`recommend`表示一个查询推荐API，它接受一个查询字符串并返回一组推荐查询；`filter`使用正则表达式启发式方法删除不相关的搜索短语。`Qinit`是一组初始查询短语，`limit`是查询滚雪球`Q`的大小限制。

#### B. iTunes App Store中的IPS应用
在第III节的Google搜索中，我们发现了几个在Apple App Store上列出的双用途应用程序。所有发现的非商店应用程序都支持Android和iOS平台。为了进一步寻找IPS应用程序的证据，我们决定将我们的测量管道应用于iOS平台，使用我们在第III节中的爬虫管道，并进行一些修改，如下所述。

有趣的是，与Play Store API不同，iTunes App Store（Apple的官方市场，也称为App Store）提供的搜索API在许多与IPS相关的搜索词（如“catch your spouse cheating”）上没有返回任何搜索结果或查询建议。然而，通过Google的站点特定搜索（通过在搜索词前添加前缀“site: itunes.apple.com”进行搜索），我们在App Store中找到了几个未通过直接搜索找到的应用程序链接。此外，我们发现iTunes查询补全API返回了许多完全不相关的查询，与IPS无关。因此，我们选择使用通过Google Play商店搜索建议获得的搜索词滚雪球来搜索iTunes和Google以查找双用途iOS应用程序。

通过直接在iTunes上搜索以及在Google上进行站点特定搜索，我们找到了2,724个应用程序。手动检查从这个列表中随机抽取的500个应用程序的描述后，我们发现97个应用程序（20%）具有IPS功能。我们在Apple上找到的店内IPS应用程序的比例低于Android（28%）。在Google Play上列出的双用途应用程序中，我们发现有9个IPS相关的应用程序也在iTunes App Store中列出（具有非常相似的应用标识符）。然而，许多跨平台列出的应用程序在iOS上的功能有所不同，这是由于其更严格的权限模型和对开发者政策的更严格执行。例如，TrackView应用程序在iOS上不会隐藏其应用图标或秘密使用摄像头，而在Android上则会（如第四章B节所述）。
