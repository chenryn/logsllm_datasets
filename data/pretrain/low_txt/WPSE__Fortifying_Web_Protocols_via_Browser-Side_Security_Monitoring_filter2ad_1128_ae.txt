5. XML 规范文档：
   - [36] A Sample XML Specification. Figure 5 shows the XML specification of the OAuth 2.0 automaton in Figure 2. The protocol is enclosed within `<protocol>` tags and describes the flow as a sequence of requests and responses. For every message, we detail its pattern, possibly specifying the endpoint and a list of parameters for requests or a list of headers for responses. Identifiers can be introduced in the protocol flow specification by adding the `id` attribute to the tag of the message component of interest. Additional identifiers can be defined within `<identifier>` tags, where the value associated with the new identifier is the part of the message matching the regular expression. If the regular expression contains a capturing group, denoted by parentheses, only the string matching the group is selected. The syntax `${id}` can be used to refer to the value bound to the identifier `id`. Security policies are defined within `<secrecy>` and `<integrity>` tags. The secrecy policy specifies that the value in `<secrecy>` must be sent only to the enumerated origins. The integrity policy specifies that the value in `<integrity>` must match the content of `<value>`, which can possibly be a regular expression.
