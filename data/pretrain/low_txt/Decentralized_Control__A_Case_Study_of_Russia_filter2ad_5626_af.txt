#### A. 验证俄罗斯黑名单

为了验证我们在 [63] 中的历史黑名单来源，我们通过几个匿名渠道获得了少量由 Roskomnadzor 数字签名的黑名单。为了从 Zapret 来源中获取与匿名提供的黑名单日期和时间戳最接近的历史黑名单，我们搜索了 Zapret 源。两个来源之间的日期和时间戳没有完全匹配，这使我们相信 Zapret 信息的来源与我们从匿名渠道获得的小量黑名单不同。

使用最接近的 Zapret 源版本，我们对两个黑名单的内容进行了预处理。这包括提取每个文件中的所有 IP 地址和域名，从而生成要比较的 IP 地址和域名集。

我们使用 Jaccard 相似性指数来比较两个来源的 IP 地址和域名集。Jaccard 指数是通过取两个集合交集的大小并除以两个集合并集的大小来计算的。Jaccard 指数是一个介于 0.0 和 1.0 之间的数字，其中 0.0 表示没有相似性，1.0 表示完全相似的集合。

| 日期       | 仅 IP 地址 | 仅域名    | IP 和域名   |
|------------|-------------|-----------|--------------|
| 2017-06-13 | 1.0         | 0.99994   | 0.99999      |
| 2018-04-27 | 0.99733     | 0.99996   | 0.99805      |
| 2018-05-13 | 0.99998     | 0.99867   | 0.99996      |
| 2018-11-08 | 0.99999     | 0.99998   | 0.99997      |

表 VI：使用 Jaccard 指数计算 Zapret 提供的黑名单与匿名提供的由 Roskomnadzor 签名的黑名单之间的相似性，按列名中提到的类别进行分类。

将 Jaccard 指数应用于来自不同来源的黑名单（由 Roskomnadzor 签名），我们发现 Zapret 黑名单与签名的黑名单非常相似。我们的结果如表 VI 所示。我们分析了 IP 地址集、域名集以及所有 IP 地址和域名组合的整体集的相似性。所有采样的黑名单在任何给定内容类型（IP、域名或 IP 和域名）上的相似性均大于 0.99。基于这些发现，我们得出结论，Zapret 黑名单来源代表了 Roskomnadzor 生成的列表，并且是正确和完整的，因此足以用于本文的分析。

#### B. RUBL 分析

图 8a 显示了每天添加的独特 IP 地址数量超过了删除的独特 IP 地址数量，进一步显示了 RUBL 的快速增长。此外，时间序列图显示了显著的波动性。重大事件（如法院裁决限制某项服务）会导致添加的 IP 地址数量激增。另一方面，特定附带损害实例的媒体报道会导致删除的 IP 地址数量激增。尽管如此，许多天没有任何显著活动，导致图表中出现多次下降。由于特定站点被列入黑名单不需要法院裁决，因此 IP 地址的添加很少降至零。相反，地址移除经常降至零，因为内容所有者启动正式移除程序所需的时间和难度很大。类似于图 2，图 8a 也显示了 2019 年独特 IP 地址的添加增加和删除减少，表明最近对黑名单的处理更加谨慎。我们在图 8b 中观察到类似的域名趋势，尽管有更多的变异性。

#### D. RUBLsub 测量

正如我们在第五部分 A 节中提到的，RUBLsub 包含 39 个子网，范围从 /16 到 /24。在这 39 个子网中，有 31 个子网至少有一个 IP 地址可以被我们的控制节点访问。剩下的八个子网完全无法从我们的控制节点访问，其中七个属于 Telegram，且全部位于莫斯科。

图 9 显示了在我们的控制节点可访问的 31 个 RUBLsub 子网中每个子网的封锁百分比，其中封锁百分比是指无法访问的 IP 地址数量占可访问子网中总 IP 地址数量的比例。两个子网，即 Subnet 16 和 Subnet 27，在大多数探针处的封锁率明显较低。这两个子网分别属于美国托管提供商 Cloud South 和英国托管提供商 UK2。其他几个子网属于 DigitalOcean 等提供商，尚不清楚为什么这两个子网的住宅封锁较少，但这可能与封锁它们时产生的附带损害有关。另一个有趣的发现是，封锁似乎与子网的大小相关：较大的子网被封锁更多，无论是通过探针还是 VPS。
