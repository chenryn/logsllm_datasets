### Authentication Factors

1. **Knowledge Factors**
   - Password
   - Passphrase
   - Personal Identification Number (PIN)
   - Challenge Response (the user must answer a question)

2. **Ownership Factors**
   - ID Card
   - Security Token
   - Software Token
   - Phone
   - Cell Phone

3. **Inherence Factors**
   - Fingerprint
   - Retinal Pattern
   - DNA
   - Signature
   - Face
   - Voice
   - Unique Bio-electric Signals or Other Biometric Identifiers

### Authentication Methods and Tokens

- **ID Card**:
  - Fake Card
  - High Security Protection

- **Security Token**:
  - High Security Protection
  - Export Token

- **Software Token**:
  - Export Token

- **Phone**:
  - Fake Phone Number

- **Cell Phone**:
  - Fake Phone Number and Phone ID

- **Password Types**:
  - Static Password
  - Synchronous Dynamic Password
  - Asynchronous Password

- **Challenge Response**:
  - Digital Signature

- **Tokens**:
  - Disconnected Tokens
  - Connected Tokens
  - Smart Card
  - Contactless Tokens
  - Bluetooth Tokens
  - Single Sign-On Software Tokens

### Token Functionality

- **Fire and Forget**:
  - For Time-based or Event-based Authentication
  - No Input Data Confirm/Check Function

- **Auto Response**:
  - For Data Confirm/Check Function
  - Most Connected Tokens Designed as Auto Response Type
  - One PIN Code Protection
  - Support Data Confirm/Check Function

- **Auto Response Token**:
  - Do Not Use Fixed Input Data Format
  - Add Trap for Protection
  - Add Dynamic PIN for System Level

- **Fire and Forget Token**:
  - Unique ID
  - No Entry for Attack Path
  - Mixed Mode with Auto Response and Fire and Forget

This structured format provides a clear and organized overview of the different authentication factors, methods, and token functionalities.
