### 附录

#### A. RSSI 检测机制

为了检测在存在强信号反射的 BLE 网络中的 RSSI 值异常，我们采用了两分量高斯混合模型。因为在噪声丰富的环境中，RSSI 值的分布可以用两个正态分布来建模 [35]。RSSI 值的概率分布函数可以表示为：

\[ f_r(y_i) = w \cdot \frac{1}{\sigma_1 \sqrt{2\pi}} \cdot e^{-\frac{(y_i - \mu_1)^2}{2\sigma_1^2}} + (1 - w) \cdot \frac{1}{\sigma_2 \sqrt{2\pi}} \cdot e^{-\frac{(y_i - \mu_2)^2}{2\sigma_2^2}} \]

其中，\(\mu_1\) 和 \(\mu_2\) 表示两个分量的均值，\(\sigma_1\) 和 \(\sigma_2\) 表示两个分量的标准差，\(w\) 表示权重参数，\(y_i\) 表示一个 RSSI 样本。在 BlueShield 中，监视器利用回溯窗口中的 \(N_l\) 个 RSSI 值通过传统的期望最大化（EM）算法 [29] 学习 \(\mu_1\)、\(\mu_2\)、\(\sigma_1\)、\(\sigma_2\) 和 \(w\) 的值。然后，监视器计算观察窗口中广告包的 RSSI 值 \((y_i, \forall i \in [1, N_o])\) 来自公式（1）给出的模型的负对数似然，即：

\[ L_r = \frac{1}{N_o} \sum_{i=1}^{N_o} -\log f_r(y_i) \]

最后，如果负对数似然大于 RSSI 检测阈值 \(\tau_r\)，则监视器检测到异常，即 \(L_r > \tau_r\)。这里，\(\tau_r\) 是 BlueShield 中的一个设计参数，将在第6节中进一步讨论。

#### C. INT 检测分析

从 INT 下限的角度进行理论分析，我们说明了 INT 检测机制在检测欺骗攻击时几乎不会出现漏检。

在 BlueShield 中，INT 下限 \(T_{lb}\) 计算为 \(T_{lb} \in [T_p - \Delta, T_p]\)，其中 \(T_p\) 是设备的广告周期，\(\Delta = 10\text{ms}\)。在欺骗攻击中，当设备和攻击者同时广播广告包时，监视器可以在两个连续的良性广告包之间观察到一个欺骗广告包（见图16）。我们注意到原始的 INT 被分为两个观察到的 INT 值 \(t_1\) 和 \(t_2\)。如果 \(t_1\) 或 \(t_2\) 中至少有一个小于 \(T_{lb}\)，则可以检测到这种欺骗攻击。显然，当 \(t_1 < T_{lb}\) 时，攻击可以被检测到。当 \(t_1 \geq T_{lb}\) 时，如果 \(t_2 \geq T_{lb}\)，INT 检测可能会漏检。这种情况下的漏检条件如下：

\[ T_p \leq 3\Delta \]
\[ t_1 + t_2 \leq T_p + \Delta \]
\[ 2T_{lb} \leq T_p + \Delta \quad (\text{因为 } T_{lb} \leq t_1 \text{ 和 } T_{lb} \leq t_2) \]
\[ 2T_p - 2\Delta \leq T_p + \Delta \]
\[ T_p \leq 3\Delta \]

上述表达式表明，当 \(\Delta = 10\text{ms}\) 时，如果 \(T_p \leq 30\text{ms}\)，攻击者可以绕过 INT 检测。我们强调，根据 BLE 规范 [5] 第1322页，\(T_p \geq 20\text{ms}\)。在我们的实验（表2）中，由于九个测试设备中 \(T_p\) 的最小值为 100ms，因此我们没有观察到这种漏检。

然而，BLE 网络中的数据包丢失率可能会影响 BlueShield 中的 INT 检测漏检情况。具体来说，当收集器由于严重的干扰而未接收到良性广告包，但接收到攻击者的欺骗广告包时，INT 检测可能无法检测到广告间隔的异常。在这种情况下，当良性广告包的数据包丢失率为 \(\rho\) 时，攻击者发送 \(n\) 个欺骗广告包而不被 INT 检测发现的概率为 \(\rho^n\)。因此，结果的漏检概率是可忽略不计的。例如，当 \(\rho = 10\%\)（这是我们实验设置中观察到的典型数据包丢失率，如表6所示），在两次广告事件中发送 \(n = 6\) 个欺骗广告包而不被检测到的概率为 0.0001%。我们强调，在实际环境中，六个是非常小的欺骗攻击成功所需的数据包数量。

#### D. 多设备监控的影响

我们评估了 BlueShield 在监控多个 BLE 设备方面的有效性，主要考虑两个指标：（1）数据包丢失率，定义为收集器接收到的数据包数量与 BLE 设备传输的数据包数量之比；（2）数据包延迟，定义为监视器从收集器检索并检查数据包特征所需的时间。如表6所示，我们的结果显示，在监控一个 BLE 设备和九个 BLE 设备之间，平均数据包丢失率（9.42% 对比 10.21%）和平均数据包延迟（10.56ms 对比 10.70ms）没有显著差异。此外，在我们的实验中，BlueShield 能够轻松发现办公室环境中的 30 个 BLE/蓝牙设备。这表明 BlueShield 可以有效监控至少 30 个 BLE 设备，涵盖了大多数 BLE 使用场景。
