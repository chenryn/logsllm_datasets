### Now on Tap 的失败与谷歌助手的改进

Now on Tap 功能被广泛认为是一个不成功的尝试。它的激活方式——长按 Home 键——使其成为了一个隐蔽且难以发现的功能，很容易被用户遗忘。我们推测这个功能的使用率非常低。即使有用户发现了 Now on Tap，它也经常无法准确理解用户的意图，导致许多用户在几次尝试后选择放弃。

随着 2016 年 Google Pixel 系列手机的发布，谷歌似乎承认了 Now on Tap 的失败。他们将 Now on Tap 重新命名为“屏幕搜索”，并将其降级为谷歌助手的辅助功能。新的谷歌助手接管了 Home 键的手势，并将其与语音系统的激活手势结合。谷歌还从 Now on Tap 的可发现性问题中吸取了教训，在 Home 键上添加了一组带有动画效果的彩色点，以帮助用户发现和记住这一功能。

### 应用权限系统

安卓 6.0 终于引入了应用权限系统，使用户能够更细粒度地控制应用可以访问的数据。

#### 权限管理

- **安装时权限请求**：应用在安装时不再一次性展示长长的权限列表。在棉花糖版本中，应用安装时不会请求任何权限。当应用需要特定权限（如位置、摄像头、麦克风或联系人列表）时，会在实际需要时向用户请求。
- **实时权限请求**：在应用运行过程中，如果需要新权限，会弹出一个“允许”或“拒绝”的对话框。一些应用在启动时会请求关键权限，其他权限则在实际需要时再提示。这种做法更好地解释了为什么需要这些权限，例如，应用需要摄像头权限是因为用户点击了拍照按钮。

#### 权限设置界面

除了实时的权限请求对话框，棉花糖还增加了一个权限设置界面。这个界面通过复选框列表让用户可以浏览每个应用拥有的权限。用户不仅可以通过应用查询权限，还可以通过权限类型查询。例如，用户可以查看所有具有麦克风访问权限的应用。

#### 早期试验与改进

谷歌在之前的版本中已经试验过应用权限管理，但棉花糖的权限系统代表了一个有序的过渡。安卓 4.3 中的 App Ops 系统从未计划公开给用户，因此开发者对此并不熟悉。在 4.3 中拒绝某个权限常常会导致奇怪的错误信息或应用崩溃。相比之下，棉花糖的权限系统是默认启用的，适用于针对棉花糖 SDK 开发的应用。这表明开发者已经准备好处理权限请求，并且可以在权限被拒绝时与用户沟通，指导他们在需要时打开相关权限。

### 指纹 API

在棉花糖之前，少数厂商推出了自己的指纹识别解决方案，作为对苹果 Touch ID 的回应。然而，在棉花糖中，谷歌终于引入了生态级别的指纹识别 API。新系统包括指纹注册界面、指纹验证锁屏以及允许应用将内容保护在指纹扫描或锁屏验证之后的 API。

#### 支持与合作

Play 商店是最早支持该 API 的应用之一，用户可以使用指纹购买应用而无需输入密码。Nexus 5X 和 6P 是首批支持指纹 API 的设备，配备了背面指纹读取硬件。指纹 API 的推出标志着安卓生态系统中的罕见合作，所有具备指纹识别功能的手机都采用了谷歌的 API，大多数银行和购物应用也很好地支持了这一功能。
