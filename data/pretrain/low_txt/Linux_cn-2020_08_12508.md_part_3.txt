### 微软与开源：从对抗到合作

**00:18:30 - Steven Vaughan-Nichols**：
即使是像微软这样规模庞大的公司，也难以与数千名致力于开发 Linux 及其他大型开源项目的开发者竞争。长期以来，微软一直不愿意与开源社区合作。前微软首席执行官史蒂夫·鲍尔默对 Linux 的态度极其反感，甚至将其视为一种“癌症”，因为 Linux 采用了 GPL 许可证。然而，随着鲍尔默的离任，新的微软领导层开始意识到，试图阻止开源潮流是徒劳无功的。他们决定转变策略，与 Linux 合作而非对抗。

**00:19:30 - Steven Vaughan-Nichols**：
2017 年的微软已经不再是史蒂夫·鲍尔默或比尔·盖茨时代的微软。这是一家全新的公司，拥有全新的理念。一旦拥抱开源，就再也无法回到过去。开源已经渗透到了整个技术领域。尽管许多人可能从未听说过 Linux，但他们在使用 Facebook、进行谷歌搜索或使用 Android 手机时，实际上都在运行 Linux。Linux 已经无处不在，微软无法阻止这一趋势，更不可能通过某种方式完全掌控它。

**00:20:00 - Saron Yitbarek**：
开源支持者曾担心微软会像混入羊群的狼一样，但开源软件的本质保护了它，使其不会被任何一家公司完全控制。Greg Kroah-Hartman 是 Linux 基金会的一员，他认为每个公司和个人都出于自身利益对 Linux 做出贡献。这种自私的行为反而让所有人都受益，因为他们会将代码回馈给社区，从而惠及每一个人。

**00:21:00 - Saron Yitbarek**：
微软已经认识到，在即将到来的云计算战争中，与 Linux 对抗就像与空气作战一样毫无意义。如今，微软已成为 Linux 基金会的白金会员，并且是 GitHub 上最大的开源项目贡献者之一。2017 年 9 月，微软还加入了开源促进联盟（Open Source Initiative, OSI）。现在，微软在开源许可证下发布了大量代码。John Gossman 描述了微软开源 .NET 框架的经历。起初，他们并没有期望得到太多外部贡献，但三年后，超过 50% 的 .NET 框架库的贡献来自微软以外的开发者。三星为 .NET 提供了 ARM 支持，Intel 和 ARM 等芯片厂商也为 .NET 贡献了特定于其处理器的代码，以及其他大量的 bug 修复和性能改进。

**00:22:00 - Saron Yitbarek**：
几年前，这样的微软几乎是不可想象的。我是 Saron Yitbarek，这里是《代码英雄》。我们已经见证了为了赢得数百万桌面用户而进行的激烈竞争，以及开源软件在专有软件巨头的背后悄然崛起并攫取巨大市场份额的过程。我们看到了一批批编程英雄如何塑造了今天的编程世界。现在，大企业正在吸收开源软件，每个人都能从中受益。

**00:23:00 - Saron Yitbarek**：
在技术领域，借鉴和进化一直存在。苹果受到施乐的启发，微软受到苹果的启发，Linux 受到 UNIX 的启发。开源软件不再仅仅是大卫，也不再是歌利亚。它已经成为了一个战场，新的战争正在云计算和开源领域加剧。Steven Levy 指出，目前有四到五家公司正努力成为全方位的平台，包括人工智能领域。这些公司在智能助手方面展开竞争，例如苹果的 Siri、微软的 Cortana、谷歌助手、三星的 Bixby 和亚马逊的 Alexa。

**00:24:00 - James Allworth**：
规模优势在过去是线性的，但现在变成了指数型。一旦某个公司走在前面，新玩家要想赶上就变得越来越难。互联网时代尤其如此，无论是因为规模还是数据带来的竞争优势。客户往往会选择领先者，因为他们有更好的产品和服务。

**00:25:00 - Saron Yitbarek**：
这个故事始于史蒂夫·乔布斯和比尔·盖茨等非凡人物，但科技的进步已经呈现出一种众包和自组织的特点。林纳斯·托瓦兹在发明 Linux 内核时甚至没有一个明确的计划，但他无疑是一位才华横溢的年轻开发者。变革是不可避免的。据估计，用传统专有的方式创建一个 Linux 发行版将花费超过 100 亿美元，这说明了开源的力量。成功的公司必须保持开放，这是最终的教训。

**00:26:00 - Saron Yitbarek**：
未来的优秀程序员无论来自何方，只要能够访问代码，他们就能构建下一个大项目。这就是操作系统战争的故事，这场战争塑造了我们的数字生活，从桌面转移到服务器机房，最终进入云计算领域。过去的敌人成为了盟友，众包的未来让一切都变得开放。
