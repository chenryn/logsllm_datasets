### SQL Firewall and Web Application Security

- **[IronBee](https://github.com/ironbee/ironbee)**: An open-source project for building a universal web application security sensor, providing a framework for developing a web application firewall (WAF).
- **[Curiefense](https://github.com/curiefense/curiefense)**: Adds a suite of automated web security tools, including a WAF, to Envoy Proxy.

### Scanning and Penetration Testing
- **[Spyse](https://spyse.com/)**: An OSINT search engine that provides real-time data on the entire web, including IPv4 hosts, subdomains, whois, ports, banners, protocols, technologies, OS, AS, and SSL/TLS databases.
- **[sqlmap](http://sqlmap.org/)**: An open-source penetration testing tool that automates the detection and exploitation of SQL injection flaws. It includes features for database fingerprinting, data fetching, file system access, and command execution.
- **[ZAP (Zed Attack Proxy)](https://www.owasp.org/index.php/OWASP_Zed_Attack_Proxy_Project)**: An easy-to-use integrated penetration testing tool for finding vulnerabilities in web applications. It is suitable for users with varying levels of security experience.
- **[OWASP Testing Checklist v4](https://www.owasp.org/index.php/Testing_Checklist)**: A comprehensive list of controls to test during a web vulnerability assessment. A Markdown version is available [here](https://github.com/amocrenco/owasp-testing-checklist-v4-markdown/blob/master/README.md).
- **[w3af](http://w3af.org/)**: A Web Application Attack and Audit Framework designed to help secure web applications by identifying and exploiting vulnerabilities.
- **[Recon-ng](https://github.com/lanmaster53/recon-ng)**: A full-featured web reconnaissance framework written in Python, similar in look and feel to the Metasploit Framework.
- **[PTF (Penetration Testers Framework)](https://github.com/trustedsec/ptf)**: A modular framework for up-to-date penetration testing tools.
- **[Infection Monkey](https://github.com/guardicore/monkey)**: A semi-automatic penetration testing tool for mapping and testing networks, simulating human attackers.
- **[ACSTIS](https://github.com/tijme/angularjs-csti-scanner)**: A tool for scanning web applications for AngularJS Client-Side Template Injection (CSTI) vulnerabilities.
- **[padding-oracle-attacker](https://github.com/KishanBagaria/padding-oracle-attacker)**: A CLI tool and library for executing padding oracle attacks, supporting concurrent network requests and an elegant UI.
- **[is-website-vulnerable](https://github.com/lirantal/is-website-vulnerable)**: A tool that identifies publicly known security vulnerabilities in a website's frontend JavaScript libraries.
- **[PhpSploit](https://github.com/nil0x42/phpsploit)**: A full-featured C2 framework that persists on web servers via an evil PHP one-liner, featuring stealth persistence, privilege escalation, and post-exploitation capabilities.
- **[Keyscope](https://github.com/SpectralOps/keyscope)**: An extensible key and secret validation tool built in Rust for checking active secrets against multiple SaaS vendors.
- **[Cyclops](https://github.com/v8blink/Chromium-based-XSS-Taint-Tracking)**: A Chromium-based web browser with XSS detection features, used to find flows from a source to a sink.
- **[Scanmycode CE (Community Edition)](https://github.com/marcinguy/scanmycode-ce)**: A code scanning and static analysis tool that supports multiple languages and checks, including PHP, Java, Scala, Python, Ruby, JavaScript, GO, and more.
- **[recon](https://github.com/rusty-ferris-club/recon)**: A fast Rust-based CLI that uses SQL to query over files, code, or malware, with content classification and processing for security experts.

### Runtime Application Self-Protection (RASP)
- **[Sqreen](https://www.sqreen.io/)**: A RASP solution that instruments and monitors applications, reporting suspicious activities and blocking attacks at runtime without modifying code or redirecting traffic.
- **[OpenRASP](https://github.com/baidu/openrasp)**: An open-source RASP solution maintained by Baidu, featuring context-aware detection with minimal false positives and low performance overhead.
