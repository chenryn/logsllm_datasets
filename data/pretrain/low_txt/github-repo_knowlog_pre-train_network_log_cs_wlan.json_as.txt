### 802.1X 相关消息
- **DOT1X-4-INFO_EAPOL_PING_RESPONSE**:
  - **消息**: "接口 %s 上的 802.1x 客户端 MAC 地址为 %s"
  - **描述**: 发送 EAPOL ping 到该端口以确定连接的客户端是否支持 802.1x。设备（如 PC）已收到 EAPOL 响应。

- **DOT1X-5-IGNORE_LARGER_EAPOL**:
  - **消息**: "EAPOL 数据包大小 (%d) 超过了接口 %s 的 MTU 大小 (%d)，忽略发送 EAPOL 数据包"
  - **描述**: 待发送的 EAPOL 数据包大小超过了客户端连接端口的 MTU 大小，因此忽略该数据包的发送。

- **DOT1X-5-SUCCESS**:
  - **消息**: "接口 %s 上的客户端 (%s) 认证成功，审计会话 ID: %s"
  - **描述**: Dot1x 认证成功。

- **DOT1X-5-FAIL**:
  - **消息**: "接口 %s 上的客户端 (%s) 认证失败，审计会话 ID: %s"
  - **描述**: Dot1x 认证失败。

- **DOT1X-5-RESULT_OVERRIDE**:
  - **消息**: "接口 %s 上的客户端 (%s) 认证结果被覆盖，审计会话 ID: %s"
  - **描述**: 认证结果被覆盖。

- **DOT1X-3-PROC_START_ERR**:
  - **消息**: "Dot1x 无法启动"
  - **描述**: 系统未能创建 Dot1x 进程。

- **DOT1X-3-UNKN_ERR**:
  - **消息**: "发生未知的操作错误"
  - **描述**: 由于内部系统错误，Dot1x 进程无法运行。

- **DOT1X-4-MEM_UNAVAIL**:
  - **消息**: "内存不足以执行 802.1X 操作，审计会话 ID: %s"
  - **描述**: 系统内存不足，无法执行 Dot1x 认证。

### EAP 相关消息
- **EAP-2-PROCESS_ERR**:
  - **消息**: "EAP 关键错误 %s"
  - **描述**: 路由器无法执行与 EAP 相关的操作，这是一个关键问题。

- **EAP-6-MPPE_KEY**:
  - **消息**: "IP=%s MPPE_RECV_KEY_LENGTH=%d"
  - **描述**: 路由器收到了指定主机的 MPPE 密钥。

- **EAP-3-BADPKT**:
  - **消息**: "IP=%s HOST=%s"
  - **描述**: 路由器从指定主机收到了无效或格式错误的 EAP 数据包。

- **EAP-6-FIPS_UNSUPPORTED_METHOD**:
  - **消息**: "EAP 配置文件 %s 中的方法 %s 在 FIPS 模式下不被允许"
  - **描述**: 指定的方法在 FIPS 模式下不受支持。

### EPM 相关消息
- **EPM-6-POLICY_REQ**:
  - **消息**: "IP %s|| MAC %s|| 审计会话 ID %s|| 事件 %s"
  - **描述**: EPM 收到了策略应用/移除请求。

- **EPM-6-POLICY_APP_SUCCESS**:
  - **消息**: "客户端 [%s] MAC [%s] 审计会话 ID [%s] 的策略类型 [%s] 策略名称 [%s] 应用成功"
  - **描述**: 显示的策略已成功应用于客户端。

- **EPM-4-POLICY_APP_FAILURE**:
  - **消息**: "客户端 [%s] MAC [%s] 审计会话 ID [%s] 的策略类型 %s 策略名称 %s 应用失败，原因 %s"
  - **描述**: 显示的策略无法应用于客户端。

- **EPM-6-AAA**:
  - **消息**: "策略 %s|| 事件 %s"
  - **描述**: 已成功发送/下载指定 dACL 的下载请求。

- **EPM-4-ACL_CONFIG_ERROR**:
  - **消息**: "ACL 名称 '%s' || ACE 序号 %d || 结果失败 || 原因 %s"
  - **描述**: 由于配置错误，忽略了向指定 ACL 添加 ACE。

- **EPM-6-IPEVENT**:
  - **消息**: "IP %s|| MAC %s|| 审计会话 ID %s|| 事件 %s"
  - **描述**: 指定了 IP 事件等待/释放/分配。

- **EPM-6-AUTH_ACL**:
  - **消息**: "策略 %s|| 事件 %s"
  - **描述**: 应用或移除了 AUTH_DEF_ACL 或 AUTH_DEF_ACL_OPEN。

  - **消息**: "接口 %s 上的安全违规，新 MAC 地址 (%s) 被检测到，审计会话 ID: %s"
  - **描述**: 指定接口上的主机试图进入网络或以不支持的方式进行认证，被视为安全违规，端口已被禁用。

  - **消息**: "接口 %s 上的客户端 (%s) 已尝试所有认证方法，审计会话 ID: %s"
  - **描述**: 所有可用的认证方法都已尝试过。

- **SESSION_MGR-5-MACMOVE**:
  - **消息**: "MAC 地址 (%s) 从接口 %s 移动到接口 %s 审计会话 ID: %s"
  - **描述**: 客户端在未注销前一个接口的情况下移动到新接口。

- **SESSION_MGR-5-MACREPLACE**:
  - **消息**: "接口 %s 上的 MAC 地址 (%s) 被 MAC (%s) 替换，审计会话 ID: %s"
  - **描述**: 新客户端触发违规，导致现有客户端被替换。

- **SESSION_MGR-4-UNAUTH_MOVE**:
  - **消息**: "%s MAC 地址 (%s) 从 %s 移动到 %s"
  - **描述**: 在旧 IDB 上未认证的情况下移动。

- **SESSION_MGR-4-ILLEGAL_TRACE_REQ**:
  - **消息**: "组件 ID %d 未注册"
  - **描述**: 组件 ID %d 未注册。

- **SESSION_MGR-0-CONFIG_CORRUPT**:
  - **消息**: "检测到校验和接口配置损坏"
  - **描述**: 接口配置字段被意外修改。
