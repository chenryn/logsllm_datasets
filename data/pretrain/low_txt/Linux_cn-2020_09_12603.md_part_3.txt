### 探讨开源社区对工作的促进作用

我与 Shannon Crabill 就此话题进行了深入交流。Shannon 白天是一名电子邮件开发者，晚上则致力于学习前端开发。她对社区的价值有着切身的体会。

**00:17:28**：
去年，Shannon 参加了一个为期一个月的开源活动——Hacktoberfest，该活动旨在吸引更多人参与开源贡献。当时，Shannon 完全是一个开源新手。

**00:17:44 - Shannon Crabill**：
在十月份的时候，我觉得自己手头的工作不多，还有很多像我这样的新手可能找不到可以做的事情。因此，我提出了一些相对简单的任务，这样他们就可以开始尝试和学习，逐渐熟悉 Git 和 GitHub 的使用。

**00:18:04**：
我认为最困难的部分在于习惯工作流程并付诸实践。如何推送代码库？如何分享项目？如何处理 PR（Pull Request）及相关内容？通过帮助他人进行开源贡献，我感到非常惊喜和满足。

**00:18:21 - Saron Yitbarek**：
对于新手来说，即使拥有代码库，走出自己的舒适区，融入社区并与他人互动，确实是一项挑战。你必须与贡献者交谈，审查他们的代码并提供反馈。这听起来确实有些吓人。

**00:18:42 - Shannon Crabill**：
最初的反应是“哇，这太酷了”，但紧接着就是“天哪，我把自己卷入了什么境地？”我意识到自己只合并过自己的代码，从未处理过别人的 PR。因此，我不得不从零开始学习如何合并 PR。总的来说，将复杂的 PR 合并起来仍然是我需要努力解决的问题。

**00:19:09**：
这种感觉就像是一场旋风，“这很酷，但我不知道该怎么做。”幸运的是，每个人都很友好。我也尽量保持友好和真诚，例如说：“嘿，我现在有点不知所措，但我明天会处理你们提交的 PR。”大家对此反应良好。

**00:19:26 - Saron Yitbarek**：
当你维护一个项目时，尤其是作为新手开发者，你是否觉得自己必须成为整个代码库中最聪明的人？你实际上在评估、评判并审查其他人的代码。你是否遇到过自己懂的东西不如提交 PR 的人多的情况？你是如何应对的？

**00:19:55 - Shannon Crabill**：
这是一个好问题。我认为，认为自己必须成为最聪明的开发者的想法会成为一种障碍。我很幸运，在进入这个领域时没有这种想法，所以我能够抱着开放的心态去尝试，并期待未来的发展。

**00:20:12**：
你不需要成为一个有 20 年经验的高级开发者。你只需要有一个好的想法，知道如何使用相应的软件，并且愿意学习自己不懂的东西。

**00:20:22**：
确实有几个 PR 为我的项目添加了一些很酷的功能，坦白说，如果这些功能崩溃了，我也不知道该如何修复。我可能会看着代码想：“是的，它崩溃了。”但我不知道该怎么做才能从头构建它们。

**00:20:34**：
这就是开源的魅力所在。如果只有我一个人做贡献，项目可能只会做一些基本的工作，而无法达到那些经验丰富的人所贡献的水平。

**00:20:45 - Saron Yitbarek**：
作为一个维护者，在使项目更易于访问、更加友好和易于贡献的过程中，你学到了什么？

**00:20:55 - Shannon Crabill**：
我认为最有帮助的一件事是尽可能建立模板和文档。我在 README 文件中添加了很多内容，并认为在项目开始时有一个详细的 README 文件是非常重要的一步。这意味着你在告诉其他人：“嘿，请查看我们的贡献指南。”

**00:21:07**：
我创建了 PR 模板、议题模板，并提供了“单击这里查看当前议题”的链接，以避免重复提交相同的内容。

**00:21:31**：
让项目尽可能简单和用户友好，这是作为维护者需要迈出的重要一步。

**00:21:38 - Saron Yitbarek**：
绝对是这样。README 文件的重要性不言而喻。你应该如何构建它，使其真正吸引希望做出贡献的人呢？

**00:22:00 - Shannon Crabill**：
我在我的 README 文件中放了很多 GIF 动图。

**00:22:03 - Saron Yitbarek**：
很好。

**00:22:05 - Shannon Crabill**：
我不仅有 GIF，还有链接。

**00:22:07 - Saron Yitbarek**：
从一开始我就知道 Shannon 已经认识到了合作的重要性。她明白，当人们被邀请来协作时，项目就会变得更好，大家也会因此度过美好的时光。

**00:22:20 - Shannon Crabill**：
有人用开源项目做出了很棒的事情，我也觉得这很有趣。比如有个有趣的项目：“嘿，我制作了这些很酷的蝙蝠，每次你点击页面时，它们都会随机生成。”

**00:22:33 - Saron Yitbarek**：
我喜欢人们做不同的事情。如果你喜欢艺术性的东西，你可以做蝙蝠生成功能。如果你想让项目整洁有序，你也可以这样做。如果你想专注于贡献文档，为其他贡献者创造一个干净舒适的环境，这也是很好的选择。

**00:22:56 - Shannon Crabill**：
是的。我已经明确表示，无论你想贡献什么，都是受欢迎的。如果你发现了一个拼写错误并想要纠正它，很好。如果你发现某个链接损坏并想要修复它，也很好。如果你只是想帮着注释代码，使其更易读和理解，那也非常有帮助。

**00:23:12 - Saron Yitbarek**：
你在社区中获得积极反馈真的很棒，因为我听说过很多并不那么美好的故事。网络上的人们并不总是友好和热情，尤其是对那些可能会问出比他们想象中更简单问题的新手。

**00:23:33**：
你认为是什么使得你的社区比其他社区更加友好呢？

**00:23:41 - Shannon Crabill**：
因为在我们的社区，贡献是一件很随意的事。如果你想做出贡献，你可以；如果你不想，也没关系。有些人认为开源是一件令人恐惧的大事，需要具备丰富的经验和复杂的技术知识，但实际上并非如此。

**00:24:03 - Saron Yitbarek**：
确实是这样。参加 Hacktoberfest 对你现在对开源的看法有什么影响？

**00:24:11 - Shannon Crabill**：
它对我产生了积极的影响。正如我说的，我有过很棒的经历，我希望每个参与我项目的人也能有同样的体验。这无疑促使我想尝试更多类似的事情，即使它们没有进一步的发展。现在看来，这些项目变得更加平易近人了。

**00:24:32 - Saron Yitbarek**：
真是个好消息。

**00:24:34**：
数百家公司的数千人以及一些不在公司工作的人为 Linux 内核做出了贡献。这意味着运行互联网的 Linux 是由一群日常英雄在维护的。如果你渴望为开源做出第一次贡献，或许你会想了解有关 Fedora 社区的更多信息。请查阅 [redhat.com/commandlineheroes](http://redhat.com/commandlineheroes) 以获取更多信息。

**00:25:20 - Saron Yitbarek**：
快速提醒一下，这一季我们将构建我们自己的开源《代码英雄》游戏。欢迎你以对你来说合适的方式来做出贡献。快来了解如何成为其中一员吧。我们希望你可以通过 [redhat.com/commandlineheroes](http://redhat.com/commandlineheroes) 和我们一起开发这款游戏。

**00:25:42 - Saron Yitbarek**：
下一集，我们将讨论残酷的达尔文式错误以及开源开发中失败的美丽之处——它如何困扰我们、指导我们，并使我们变得更好。

**00:25:57 - Saron Yitbarek**：
《代码英雄》是红帽的原创播客。你可以在 Apple Podcast、Google Podcast 或其他你喜欢的平台上免费收听。我是 Saron Yitbarek，坚持编程，下期再见。
