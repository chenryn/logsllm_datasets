### 问题描述

在我们的域控制器中，有一个或多个GPO的Sysvol权限与基线域中的GPO权限不同步。我最近安装了第二个域控制器，除了组策略外，所有复制都正常工作。在Windows Server 2012 R2中，通过新的组策略管理工具，当我点击“立即检测”时，结果显示ACLs与基线域不同步。

**环境:**

- DC1: Windows Server 2008 R2
- DC2: Windows Server 2012 R2
- 森林和域功能级别: Windows Server 2003
- 复制类型: FRS (文件复制服务)

我已经运行了`dcdiag`、检查了事件日志，并使用了`repadmin /showrepl`等命令，一切看起来都很正常，但组策略无法同步。我还检查了两个DC上的Sysvol ACL，它们似乎具有相同的权限。此外，组策略中央存储（即Sysvol）也已正确复制。

我在一个博客上找到了类似的问题及解决方案（[链接](http://sysadminconcombre.blogspot.com.au/2014/06/microsoft-dfs-r-problem-sysvol.html)），该方案建议重启DFSR服务。但由于我的域功能级别是2003，所以使用的是FRS而不是DFSR。

**问题：**

是否有一种方法可以在不迁移到DFSR的情况下解决这个问题？或者我应该先迁移到DFSR？许多资料表明，在迁移之前，复制必须完美无缺地工作。

任何建议都非常感谢！

### 更新

我最终通过手动应用两个服务器上的Sysvol ACL来解决了这个问题。不知为何，我需要为每个位于`sysvol\policies`下的策略添加域管理员组的完全控制权限，之后它就能正常同步了。现在一切都正常工作，我会在以后升级域功能级别时再考虑迁移到DFSR。谢谢大家！

对于遇到同样问题的人，如果你只有少量的策略，可以考虑备份设置，删除所有策略后再重新添加，这可能会更快解决问题。
