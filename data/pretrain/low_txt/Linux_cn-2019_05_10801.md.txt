昨日，推特用户“EmacsTW”发布了一条推文，建议那些考虑使用 Emacs 的人转而选择现代化的编辑器如 VSCode。这一举动在技术圈引发了广泛讨论，特别是在 Emacs 用户群体中。

![](/data/attachment/album/201905/01/124829s1ts92vh800i201n.jpg)

以下是部分用户的反应：

* “中出了个叛徒”
* “虽然我现在用 VSCode 比 Emacs 多，但说 Emacs 害人有点过了”
* “哼哼哼… VSCode 又不能煮咖啡、不能玩游戏”
* “太突然了。作为一个 Emacs 使用者觉得他说的很有道理，我也从来不向他人推荐 Emacs，太浪费时间了。”
* “我司如果只看 Vim vs. Emacs 的话，Vim 完胜，但是大部分还是 VSCode……”

### 一个 Emacs 忠实粉丝眼中的 Emacs 背刺事件

由于笔者本人并不是 Emacs 的粉丝（更喜欢 Vim），因此采访了 LCTT 翻译组内的 Emacs 铁粉 DarkSun（lujun9972），询问他的看法。

当我问起这个事件时，DarkSun 的第一反应是：

> 
> 欢迎背叛，舒服就成
> 

在我表现出不解后，DarkSun 进一步解释了他的观点：

> 
> 如果只是把 Emacs 当作写代码的工具，那确实不如用 IDE。我把 Emacs 当作信息交流的平台，类似于终端。Emacs 插件可以看作 UNIX 工具，它们之间可以通过文本或 Lisp 对象进行交流。例如，我可以使用 Emacs 查看 PDF 文档，遇到不懂的单词可以用有道词典翻译，并将结果存入 Anki 作为学习材料。在这个过程中，Emacs 本身起到信息交流平台的作用，其他功能则由第三方程序或插件实现。
>
> 一百个人有一百种使用 Emacs 的方法，这种多样性才是 Emacs 的魅力所在。
>

了解他对 Emacs 的理解后，再想到“Emacs 煮咖啡”的梗，我对他的观点深感赞同。

> **Emacs 是什么？**
> 
> 我们熟悉的 Emacs 是由自由软件领袖 Richard Matthew Stallman (RMS) 于 1984 年开发并维护至 2008 年。
> 
> Emacs 追求全面的哲学，希望在一个编辑器中完成所有功能，通过快捷键和宏实现一切。因此，Emacs 常被戏称为“**被误认为是操作系统的编辑器**”。

### 新旧技术的更迭

近年来，VSCode 和 Sublime Text 3 的出色表现吸引了越来越多的开发者。对于一些重型语言（如 C/C++、Java、Golang 等），IDE 成为首选。相比之下，Emacs 似乎不再那么受欢迎。

然而，我认为 Emacs 不仅仅是一个工具，它代表了一种生活方式。在 Emacs 中，你可以完成任何事情，它融入你的生活，改变你的习惯。从这个角度来看，Emacs 无可替代。

### 被钉在耻辱柱上的社群背叛者 —— azazabc123

不论 Emacs 和 VSCode 的优劣，该推特账号的行为是不妥当的。这不仅损害了 Emacs TW 社群的形象，还破坏了社群的信任。作为社群对外窗口的维护者，发布与社群理念相悖的内容显然不合适。这种行为使 Emacs 的背叛从个人上升到群体层面，对整个社群造成了负面影响。

笔者非常反对这种公器私用的行为。尽管可能出于吐槽或任性，但这种无脑行为只会损害社群的整体信用和个人品行评价。

### 英雄迟暮？

诞生于 1984 年的 Emacs 已经走过 35 个年头。尽管 Emacs 依然好用，但时代已经变了。对于现代黑客来说，工具的价值更为重要。在这种观念下，学习成本极高的 Emacs 难免不受欢迎，简单易用的 VSCode 和 Sublime 成为必然选择。

无论如何，我们欢迎新技术和变化，但对该推特维护者的这种让社区蒙羞的行为表示遗憾和谴责。
