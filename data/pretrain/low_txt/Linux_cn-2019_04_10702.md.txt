### Git 十四周年：你最喜欢 Git 的哪一点？

Git 自 Linus Torvalds 开发以来，已经过去了十四年。在这段时间里，它对软件开发的影响是其他工具难以企及的。根据 [StackOverflow 2018 年开发者调查](https://insights.stackoverflow.com/survey/2018/#work-_-version-control)，87% 的受访者表示他们使用 Git 作为项目的版本控制工具。显然，Git 在版本控制系统（SCM）中的地位无人能撼动。

为了庆祝 Git 的十四周年纪念日，我询问了一些爱好者他们最喜欢 Git 的哪些方面。以下是他们的回答：

> “我对 Git 感到非常不满，无论是其复杂的术语还是分布式特性。只有通过 Gerrit 等插件，它才能达到像 Subversion 或 Perforce 这样的集中式仓库管理器的一半好用程度。不过，既然问题是‘你喜欢 Git 的什么？’，我还是会说：Git 使得复杂的源代码树操作成为可能，并且它的回滚功能使得解决需要多次修改的问题变得简单。”
>
> — [Sweet Tea Dorminy](https://github.com/sweettea)

> “我喜欢 Git 是因为它不会强制我采用特定的工作流程。开发团队可以自由地选择适合自己的方式来进行协作，无论是拉取请求、电子邮件传递差异文件，还是赋予所有人推送权限。”
>
> — [Andy Price](https://www.linkedin.com/in/andrew-price-8771796/)

> “我从 2006 年或 2007 年开始使用 Git。我喜欢 Git 是因为，无论是一个小项目还是大型的团队合作项目，Git 都适用。它允许我从几乎所有错误提交中回滚到先前版本，这大大减轻了我在版本管理方面的压力。”
>
> — [Jonathan S. Katz](https://opensource.com/users/jkatz05)

> “我很欣赏 Git 的底层命令和高层命令的理念。用户可以高效地分享任何形式的信息，而不需要了解其内部工作原理。好奇的人可以通过表层命令发现支持许多代码共享平台的内容定位文件系统。”
>
> — [Matthew Broberg](https://opensource.com/users/mbbroberg)

> “我喜欢 Git 是因为它几乎涵盖了浏览、开发、构建、测试和提交代码的所有工作。它经常激发我参与开源项目的积极性。”
>
> — [Daniel Oh](https://opensource.com/users/daniel-oh)

> “Git 是我使用的第一款版本控制工具。多年来，它从一个令人恐惧的工具变成了一个友好的工具。我喜欢它在修改代码时给我带来的自信，因为它能保证主分支的安全（除非你强制推送一段考虑不周的代码）。你可以检出之前的提交来撤销更改，这一点也很棒。”
>
> — [Kedar Vijay Kulkarni](https://opensource.com/users/kkulkarn)

> “我喜欢 Git 是因为它淘汰了一些其他的版本控制工具。没有人再使用 VSS，Subversion 可以与 git-svn 一起使用（如果必要），BitKeeper 和 Monotone 只为老一辈所知。当然，还有 Mercurial，但在我几年前为 Firefox 添加 AArch64 支持时，我觉得它还不完善。有些人可能会提到 Perforce、SourceSafe 或其他企业级解决方案，但它们在开源世界中并不流行。”
>
> — [Marcin Juszkiewicz](https://github.com/hrw)

> “我喜欢内置的 SHA1 化对象模型（commit → tree → blob）的简易性。我也喜欢它的高层命令。我还将其作为 JBoss/Red Hat Fuse 的补丁机制，效果很好。此外，我还喜欢 Git 的 [三棵树的故事](https://speakerdeck.com/schacon/a-tale-of-three-trees)。”
>
> — [Grzegorz Grzybek](https://github.com/grgrzybek)

> “我喜欢 [自动生成的 Git 说明页](https://git-man-page-generator.lokaltog.net/)，虽然这个页面听起来像是关于 Git 的，但实际上没有实际意义，但它总是给人一种像是真的 Git 页面的感觉……”
>
> — [Marko Myllynen](https://github.com/myllynen)

> “Git 改变了我作为开发者的生活。它使 SCM 问题从世界上消失得无影无踪。”
>
> — [Joel Takvorian](https://github.com/jotak)

看完这些爱好者的回答后，轮到你了：你最欣赏 Git 的哪一点？请在评论区分享你的看法！
