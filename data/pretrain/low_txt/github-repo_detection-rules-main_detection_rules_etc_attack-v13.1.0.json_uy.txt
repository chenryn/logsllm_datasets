- **描述**: 在许多云环境中，创建新的实例或虚拟机是常见的操作。事件不应孤立地看待，而应作为可能导致其他活动的行为链的一部分。例如，由新用户帐户创建实例，或在意外创建一个或多个快照后创建实例，可能表示可疑活动。在 AWS 中，CloudTrail 日志通过 RunInstances 事件捕获实例的创建；在 Azure 中，VM 的创建可能被捕获在 Azure 活动日志中；Google 的 Cloud Audit Logs 中的 Admin Activity 审计日志可以用来检测使用 gcloud compute instances create 创建 VM 的行为。
