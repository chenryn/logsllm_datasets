### 通过 WMIC 创建远程进程

**描述**：监控新构建的进程和/或命令行中的 "wmic"。如果在源主机上使用了命令行工具 wmic.exe，则可以通过分析进一步检测到。源主机上的命令行可能类似于 wmic.exe /node:"\\\" process call create "\\\"。还可以通过 IP 地址连接，在这种情况下，字符串 "\\\" 将替换为 IP Address。可以通过 WMI 以几种其他方式远程创建进程，例如更直接的 API 访问或内置工具 PowerShell。

**实现示例**：通过 WMIC 创建远程进程

**检测伪代码**：
```python
processes = filter processes where ((event_id="10" OR event_id="4688") AND exe == "wmic.exe" AND command_line == "* process call create *")
```

**检测说明**：
- 伪代码事件 ID 是针对 Sysmon（事件 ID 10 - 进程访问）和 Windows 安全日志（事件 ID 4688 - 创建了新进程）。
- 除了执行任意进程外，wmic.exe 还可以用于执行存储在 NTFS 替代数据流中的数据 [NTFS 文件属性](https://attack.mitre.org/techniques/T1564/004)。
- 查找包含以下子字符串的 wmic.exe 命令行：
  - process call create
  - /node:
