## 概要
安全标识符 (SID) 是用于唯一标识 Windows 操作系统中的安全主体或安全组的可变长度值。一些已知的 SID 代表通用用户或组，并且在所有操作系统版本中保持一致。了解这些 SID 对于解决安全相关问题和处理 ACL（访问控制列表）编辑器中可能出现的显示问题至关重要。有时，ACL 编辑器会显示 SID 而不是用户或组名称。

## 详细信息
以下是一些常见的 SID 及其对应的名称和说明：

- **SID: S-1-0**
  - 名称：空机构
  - 说明：标识符颁发机构

- **SID: S-1-0-0**
  - 名称：没有人
  - 说明：没有安全主体

- **SID: S-1-1**
  - 名称：世界机构
  - 说明：标识符颁发机构

- **SID: S-1-1-0**
  - 名称：所有人
  - 说明：包括所有用户，甚至是匿名用户和来宾用户。成员资格由操作系统控制。
  - 注意：默认情况下，Everyone 组不再包含运行 Windows XP Service Pack 2 (SP2) 的匿名用户。

- **SID: S-1-2**
  - 名称：本地机构
  - 说明：标识符颁发机构

- **SID: S-1-2-0**
  - 名称：本地
  - 说明：包括所有本地登录的用户组

- **SID: S-1-2-1**
  - 名称：控制台登录
  - 说明：包括通过物理控制台登录的用户组
  - 注意：在 Windows 7 和 Windows Server 2008 R2 中添加

- **SID: S-1-3**
  - 名称：创建者颁发机构
  - 说明：标识符颁发机构

- **SID: S-1-3-0**
  - 名称：创建者所有者
  - 说明：父级可继承的访问控制项 (ACE) 中的占位符。当 ACE 继承时，系统将此 SID 替换为对象创建者的 SID。

- **SID: S-1-3-1**
  - 名称：创建者组
  - 说明：父级可继承 ACE 中的占位符。当继承 ACE 时，系统将此 SID 替换为主要对象创建者的组 SID。仅 POSIX 子系统使用的主要组。

- **SID: S-1-3-2**
  - 名称：创建者所有者服务器
  - 说明：此 SID 在 Windows 2000 中未使用

- **SID: S-1-3-3**
  - 名称：创建者组服务器
  - 说明：此 SID 在 Windows 2000 中未使用

- **SID: S-1-3-4**
  - 名称：所有者权限
  - 说明：表示对象当前所有者的组。当执行此 SID 的 ACE 应用于对象时，系统将忽略隐式 READ_CONTROL 和 WRITE_DAC 权限。

- **SID: S-1-5-80-0**
  - 名称：所有服务
  - 说明：包括系统上配置的所有服务进程的组。成员资格由操作系统控制。
  - 注意：在 Windows Vista 和 Windows Server 2008 中添加

- **SID: S-1-4**
  - 名称：非唯一授权机构
  - 说明：标识符颁发机构

- **SID: S-1-5**
  - 名称：NT 机构
  - 说明：标识符颁发机构

- **SID: S-1-5-1**
  - 名称：拨号
  - 说明：包括通过拨号连接登录的所有用户组。成员资格由操作系统控制。

- **SID: S-1-5-2**
  - 名称：网络
  - 说明：包括通过网络连接登录的所有用户组。成员资格由操作系统控制。

- **SID: S-1-5-3**
  - 名称：批处理
  - 说明：包括通过批队列工具登录的所有用户组。成员资格由操作系统控制。

- **SID: S-1-5-4**
  - 名称：交互式
  - 说明：包括以交互方式登录的所有用户组。成员资格由操作系统控制。

- **SID: S-1-5-5-X-Y**
  - 名称：登录会话
  - 说明：每个登录会话都有不同的 X 和 Y 值。

- **SID: S-1-5-6**
  - 名称：服务
  - 说明：包括作为服务登录的所有安全主体的组。成员资格由操作系统控制。

- **SID: S-1-5-7**
  - 名称：匿名
  - 说明：包括匿名登录的所有用户组。成员资格由操作系统控制。

- **SID: S-1-5-8**
  - 名称：代理服务器
  - 说明：此 SID 在 Windows 2000 中未使用

- **SID: S-1-5-9**
  - 名称：企业域控制器
  - 说明：包括 Active Directory 目录服务林中的所有域控制器。成员资格由操作系统控制。

- **SID: S-1-5-10**
  - 名称：主体自身
  - 说明：在 Active Directory 中的对象可继承的 ACE 中的占位符。当 ACE 继承时，系统将此 SID 替换为持有帐户的安全主体的 SID。

- **SID: S-1-5-11**
  - 名称：已验证的用户
  - 说明：包括身份已通过验证的所有用户组。成员资格由操作系统控制。

- **SID: S-1-5-12**
  - 名称：受限制的代码
  - 说明：保留供将来使用

- **SID: S-1-5-13**
  - 名称：终端服务器用户
  - 说明：包括所有登录到终端服务服务器的用户组。成员资格由操作系统控制。

- **SID: S-1-5-14**
  - 名称：远程交互式登录
  - 说明：包括通过终端服务登录的所有用户组

- **SID: S-1-5-15**
  - 名称：此组织
  - 说明：包括来自同一组织的所有用户组。仅在 Windows Server 2003 或更高版本的域控制器中添加。

- **SID: S-1-5-17**
  - 名称：IIS 用户
  - 说明：默认 Internet Information Services (IIS) 用户使用的帐户

- **SID: S-1-5-18**
  - 名称：本地系统
  - 说明：操作系统使用的服务帐户

- **SID: S-1-5-19**
  - 名称：本地服务
  - 说明：本地服务帐户

- **SID: S-1-5-20**
  - 名称：网络服务
  - 说明：网络服务帐户

- **SID: S-1-5-21-**<域>-**500**
  - 名称：管理员
  - 说明：系统管理员用户帐户，默认情况下是唯一能够完全控制系统的用户帐户。

- **SID: S-1-5-21-**<域>-**501**
  - 名称：来宾
  - 说明：没有单独帐户的用户帐户。此帐户不需要密码，默认情况下被禁用。

- **SID: S-1-5-21-**<域>-**502**
  - 名称：KRBTGT
  - 说明：密钥分发中心 (KDC) 服务使用的服务帐户。

- **SID: S-1-5-21-**<域>-**512**
  - 名称：域管理员
  - 说明：成员被授权管理域全局组。默认情况下，域管理员组是所有加入域的计算机上的管理员组成员，并且是任何新创建对象的默认所有者。

- **SID: S-1-5-21-**<域>-**513**
  - 名称：域用户
  - 说明：全局组，默认情况下包括域中所有用户帐户。在创建用户帐户时，默认将其添加到此组。

- **SID: S-1-5-21-**<域>-**514**
  - 名称：域来宾
  - 说明：全局组，默认情况下只有一个成员，即域的内置来宾帐户。

- **SID: S-1-5-21-**<域>-**515**
  - 名称：域计算机
  - 说明：全局组，包括所有客户端和加入域的服务器。

- **SID: S-1-5-21-**<域>-**516**
  - 名称：域控制器
  - 说明：全局组，包括域中的所有域控制器。默认情况下，新的域控制器都添加到此组。

- **SID: S-1-5-21-**<域>-**517**
  - 名称：证书发布者
  - 说明：全局组，包括所有运行企业证书颁发机构的计算机。证书发布者有权在 Active Directory 中发布证书的用户对象。

- **SID: S-1-5-21-**<根域>-**518**
  - 名称：架构管理员
  - 说明：本机模式域中的通用组；混合模式域中的全局组。该组有权在 Active Directory 中进行架构更改。默认情况下，该组的唯一成员是林根域的管理员帐户。

- **SID: S-1-5-21-**<根域>-**519**
  - 名称：企业管理员
  - 说明：本机模式域中的通用组；混合模式域中的全局组。该组有权在 Active Directory 中进行全林性更改，例如添加子域。默认情况下，该组的唯一成员是林根域的管理员帐户。

- **SID: S-1-5-21-**<域>-**520**
  - 名称：组策略创建者所有者
