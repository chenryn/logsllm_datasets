### 数据库、应用和所涉 DBMS 的一种中性格式来表现数据

数据库、应用程序及其涉及的数据库管理系统（DBMS）通常使用一种中性格式来表示数据。这种格式可以确保不同系统之间的数据互操作性和一致性。

### 安全断言标记语言 (SAML)

在第五域 B.6 章节中，详细介绍了安全断言标记语言 (SAML)。SAML 是一种基于 XML 的标准，用于交换认证和授权信息。SAML 设计允许具有不同身份管理系统的联邦式系统通过简化登录和单点登录进行交互。

尽管 SAML 在设计上具备固有的安全性，但安全架构师必须确保其实施不会削弱这些安全特性。例如，在传递 SAML 断言时，如果系统省略了授权请求的标识符或接收者的身份，攻击者可能会未经授权访问用户账号。

### OpenID Connect

OpenID Connect 是基于 OAuth 2.0 规范的一种可互操作的身份验证协议。它使用 REST/JSON 消息流，旨在“让简单的事情更简单，让复杂的事情变得可能”。

OpenID Connect 与 OpenID 2.0 在架构上有许多相似之处，都解决了类似的问题。然而，OpenID 2.0 使用的是 XML 和自定义消息签名方案，这使得开发人员有时难以实现。因此，OpenID 2.0 的实现有时会出现互操作性问题。相比之下，OpenID Connect 在需要签名时使用标准的 JSON Web 令牌 (JWT)，从而大大提高了互操作性。

如果用户希望使用 Google 账户登录某个网站，可以使用 OpenID。在第五域 B.1 章节关于单点登录的部分，提到了 OpenID。

### 开放 Web 应用安全项目 (OWASP) / 如何确保网站安全

在第八域 B.1 章节中也介绍了 OWASP。OWASP 是一个致力于提高软件安全性的非营利组织，提供了多种免费资源，包括：

- **OWASP 十大项目**：列出并提供解决 Web 应用程序十大常见缺陷的方法。
- **OWASP 指南项目**：提供全面的手册，指导如何设计安全的 Web 应用和服务。
- **OWASP 软件保障成熟度模型 (SAMM)**：为机构提供针对具体风险量身定制的软件开发框架。
- **OWASP 移动项目**：为开发人员和架构师提供资源，以开发和维护安全的移动应用。

鉴于基于 Web 和云的解决方案广泛使用，OWASP 提供了一个全面的框架，帮助确保 Web 应用程序的安全。以下是 OWASP 识别出的十大顶级风险：

- A1: 注入
- A2: 跨站脚本攻击 (XSS)
- A3: 无效的认证和会话管理
- A4: 不安全的直接对象引用
- A5: 跨站请求伪造 (CSRF)
- A6: 安全配置错误
- A7: 不安全的加密存储
- A8: URL 访问限制失败
- A9: 薄弱的传输层保护
- A10: 未经验证的重定向和转发

### Web 应用安全联盟 (WASC)

Web 应用安全联盟 (WASC) 是一个为 World Wide Web 及其相关软件提供最佳安全实践标准的组织。该组织提供了多种资源、工具和信息，帮助了解常见的 Web 软件开发中的安全问题及如何避免这些问题。WASC 还提供了一套 Web 应用程序扫描评估标准，用于评估各种厂商的产品和实践。以下是 WASC 阐述的一些最常见的顶级攻击方法：

- 跨站请求伪造 (CSRF)
- 跨站脚本攻击 (XSS)
- 拒绝服务攻击 (DoS)
- 已知漏洞
- 暴力攻击
- 隐藏参数操作
- 网银木马
- 点击劫持

### 内建安全 (Build Security In, BSI)

美国国土安全部 (DHS) 提供了一系列最佳实践、工具、指南、规则、原则和其他资源，帮助软件开发人员、架构师和安全从业者在软件开发的每个阶段集成安全性。最初被称为 Build Security In (BSI) 的计划与其他组织合作，使整个行业的许多单位能够参与并提供有用的材料。DHS 有一个软件保障计划来维护 BSI。

BSI 提供了一种与过程无关的方法，可用于处理需求、架构和设计、编码、测试、系统、管理和基本原则。BSI 与 MITRE 公司创建的常见缺陷列表 (CWE) 紧密结合，CWE 维护了最危险的顶级软件错误。

### ISO/IEC 27034

ISO/IEC 27034 标准涵盖了应用安全概述和概念、组织规范框架、应用程序安全管理过程、应用安全验证以及特定应用的安全指南。它是 ISO/IEC 27000 系列的一部分，旨在使安全的软件开发过程与 ISO/IEC 的信息安全管理 (ISMS) 模型保持一致。

### 标准选择

上述列举了网络通信、Web 安全和软件开发的标准，各有优长。哪个“最佳实践”或标准最适合你的组织？大多数标准适用于不同的组织和各种软件开发过程，但每种方法都有其特定的关注点。CMMI（能力成熟度模型集成）是一个过程改进模型；WASC 和 OWASP 侧重于将安全集成到 Web 软件开发过程中；BSI 侧重于保护关键基础设施，但也适用于任何软件开发项目；而 ISO/IEC-27034 是一个通用做法，多用于特定行业。

#### 设备安全

- 笔记本电脑等设备最好全系统加密，以防被盗。
- 手机等重要设备丢失时，应能远程擦除或销毁密钥。
- 离开时要锁屏，多次密码错误后锁定设备。

#### 应用安全

- 密钥管理非常重要，建议使用 TPM 芯片。

#### 安全对策

- **全设备加密**：对设备上的存储介质进行加密是一项有价值的功能，但加密不是数据保护的绝对保证，特别是如果设备被盗且存在已知的后门攻击漏洞。
- **远程擦除**：如果设备丢失或被盗，可以通过移动电话服务或互联网连接触发远程擦除或清除。然而，如果设备不联网，远程擦除也无法保证数据安全。
- **锁定**：当用户未能提供凭据并多次尝试失败后，账户或设备将被禁用一段时间，直到管理员解除锁定。
- **锁屏**：防止他人随意使用您的手机或移动设备。但如果黑客通过蓝牙、无线或 USB 电缆等方式连接到设备，屏幕锁不一定能有效保护设备。

#### BYOD 自带设备 (Bring Your Own Device)

BYOD 政策允许员工在工作中使用自己的个人移动设备，并通过公司网络访问业务资源和互联网。虽然 BYOD 可以提高员工士气和工作满意度，但它增加了组织的安全风险。开放式的 BYOD 策略可能导致任何设备连接到公司网络，增加安全风险。因此，BYOD 策略应要求特定设备以减少风险，但这也可能要求公司为无法购买兼容设备的员工购买设备。

- **数据所有权**：个人数据和业务数据应分开并隔离。
- **所有权支持**：明确移动设备损坏后的维修责任。
- **补丁管理**：定义个人移动设备的补丁管理方法和机制。
- **反病毒管理**：规定是否需要在移动设备上安装反病毒、反恶意软件和反间谍软件。
- **取证**：BYOD 策略应解决相关移动设备的取证和调查问题。
- **隐私**：BYOD 策略应解决隐私和监控问题。

### 应用密码学

密码学是实现信息安全的重要手段，被誉为信息安全的基石。考试重点在于如何应用密码学，而不是具体的算法。

#### 密码学的发展史

- **早期（人工）时代**
  - **凯撒密码/替代密码**：最早的密码系统之一，由朱利叶斯·凯撒在征服欧洲时使用。它简单地将字母表中的每个字母替换为其后的第三个字母，称为 C3 密码。另一种替代密码是 ROT13（字母移动 13 个位置）。
- **机械时代**
  - **Enigma 商业编码机**：二战期间德国使用，使用三到六个转子实现复杂的替换密码，加解密需要同原理的编码机。同盟国军方通过代号为 Ultra 的绝密工作成功复原了一台 Enigma 原型机。
  - **一次性密码本 (OTP)**：也称为 Vernam 密码，是唯一宣称只要执行得当就不可破解的密码系统。密码本只能使用一次且必须与明文长度相同，使用后不得重复。一次性密码本不能被频率分析或其他密码攻击手段破解。
  - **多表密码/维吉尼亚密码**：这是一种使用多个表（而非单一字母表）的简单替换密码。教材中常用此算法解释算法、密钥、密钥空间、加密等概念。
- **现代密码时代**
  - 计算机的各种加密算法和长密钥。
- **量子密码学**
  - 传统密码学基于数学理论加密，而量子密码学则利用物理学保护数据安全。量子密码学构建在量子物理学基础上，最重要的理论是非维尔纳·海森堡的不确定原理：一个人不可能无限准确地同时知道一个粒子的位置和动量。量子密码学主要用于密钥分配，一旦密钥交换完成，常规密码算法即可使用。量子密码学提高了安全性，因为窃听行为会干扰光子流动，容易被检测到，从而提供额外的安全保障。
  - 量子密码学内有两个独有的信道：
    - 一条信道用于通过单光子光脉冲传输量子密钥材料。
    - 另一条信道携带所有消息流，包括密码协议和加密的用户通信流。根据量子物理学原理，当一个光子被观察时，它的状态已经改变。

#### 密码学的基础概念

- **密码学的目标**
  - **可用性**：确保信息随时可用。
  - **机密性**：确保信息不被未授权访问。
  - **完整性**：确保信息在传输过程中不被篡改。
  - **不可抵赖性**：防止接收者篡改消息并声称消息的原始状态就是如此。只有公钥（或非对称密钥）密码系统才提供不可否认性。
  - **身份认证**：主要通过密钥控制实现，因为只有掌握密钥的人才能加密消息。
  - **访问控制**：只有有权访问正确密钥的人才能进行访问。

- **术语**
  - **明文 (P)**：未加密的信息。
  - **密文 (C)**：加密后的信息。
  - **Kerckhoff 原则**：算法应当公开，但所有密钥都应当保密。Auguste Kerckhoff 于 1883 年发表论文指出，密码系统唯一需要保密的部分是密钥。他认为算法应当公开，并断言如果安全基于过多的秘密，那么就会有更多的脆弱性可供利用。
  - **单向函数 (One-Way Functions)**：不可能逆向运算出原始值的函数。理论上不存在真正的单向函数，但在实际中有很多函数目前还无法被破解。
  - **随机数 (Nonce)/初始向量 (IV)**：这是一个与消息长度相同的随机比特串，并且与原始信息相异或。在每次使用相同密钥加密相同的消息时，密文总不一样。
  - **零知识证明 (Zero-Knowledge Proof)**：不告诉你密码是什么，但我可以通过开门来证明我拥有这个密码。
  - **分割知识 (Split Knowledge)**：类似于职责分离。获取一个密钥或打开一扇门不能仅靠一个系统或一个人完成，必须由多个系统或人共同工作。这样可以避免密钥被轻易篡改。
  - **工作函数/工作因素 (Work Function/Work Factor)**：即工作量。工作函数或工作因素衡量破解密码所需的工作量。
