**GitHub 的新文本编辑器并不完全开源，但似乎没有人对此表示不满。**

![](/data/attachment/album/201404/03/215000usa82qyna64v1snn.png)

Samuel Greenwald [认为](http://www.cioinsight.com/blogs/open-source-has-changed-everything.html#sthash.ESY4Kc3r.u8rX81Ow.dpuf)，“任何 IT 领袖如果没有开源观念，注定会失败。” 然而，即使开源理念打了折扣，大众也不会过分苛责。特别是在涉及复杂的许可证问题时，即使是开源界的精英也可能被迷惑。

例如，GitHub 刚发布的 [Atom](https://github.com/atom) 文本编辑器虽然获得了广泛赞誉，但实际上并非完全开源。在我看来，它还有很大的差距。

### 某种打折扣的开源

尽管如此，并不是说 GitHub 完全将 Atom 伪装成开源。正如 GitHub 联合创始人 Tom Preston-Werner 所言，只有“Atom 核心”代码将是闭源的，而“其余现有的所有 Atom 代码将永远遵守 MIT 许可证。” 这一决定纯粹出于商业考虑，他解释道：

> 
> Atom 将不会封闭源代码，但它也不会完全开源。它将介于两者之间，这样我们更易于对 Atom 进行掌控，同时人们还可以在许可证的限制下看到它的运行方式。关于这一点，我们还没有最终决定如何具体实施。我们将在充分准备后正式启动。
> 

早在开源初期，我们就有了这个概念。事实上，微软也采用了类似的做法。微软称之为“共享源代码”，并于 2002 年推出。共享源代码是微软为其社区提供的一种方式，用于监测但不触及（或重新分配）微软的源代码。SAP 的大数据主管 Vijay Vijayasankar 提醒我们，这种方法对微软来说效果不佳，但对 GitHub 可能会更好：

> 
> [@dberkholz](https://twitter.com/dberkholz) 我记得 OSI 人士严重批评了微软，称其为营销噱头。但这次 GitHub 可能会做得更好 —— Vijay Vijayasankar (@vijayasankarv) [2014年2月27日](https://twitter.com/vijayasankarv/statuses/438882094429642752)
> 

他可能是对的。

### GitHub 时代神圣不可侵犯

毕竟，微软一直被视为“邪恶帝国”，将开源贬低为“毒瘤”。而 GitHub 在世界各地都是开源项目的养父母。[2013 年](http://octoverse.github.com/)，GitHub 突破了千万代码库，新增 300 万用户，每周活跃量包括 20,000 个问题、50,000 个评论和 250,000 个来自全球贡献者的提交，确保了代码库的进展。

换句话说，GitHub 是零起点的开源项目。

也许正因为如此，GitHub 得到了一个免费通行证。在 Hacker News 的评论中，[少数人似乎过于在意](https://news.ycombinator.com/item?id=7310017)，认为 GitHub 没有真正开源 Atom。作为一个社区，开源已经在很大程度上战胜了免费软件：少教条，更实用。许多所谓的“GitHub 一代”甚至懒得去费心将许可证分配给他们的软件。

这是好事吗？

很难说，甚至很难与 GitHub 的做法争辩。它带给世界一个高品质、低成本的文本编辑器，似乎并没有伤害任何人，潜在地还可能帮助许多人。开源社区是自由意志论者：并不愿意制定严格的许可证，更关心良好的代码和产品。

这就是为什么 GitHub、Atlassian 和 Amazon 的 Web 服务都依赖专有软件或服务来赚钱，同时却受到开源开发者的欢迎。

你了解了吗？
