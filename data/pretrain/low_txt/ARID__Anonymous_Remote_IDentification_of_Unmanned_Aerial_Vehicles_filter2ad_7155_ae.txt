### Annex A: ARID Packets Format

**Table 3: MAVLink Frame and ARID Payload Notation**

| Acronym | Content/Size Description |
|---------|--------------------------|
| SOF     | 0xFE (Start of Frame)    |
| LEN     | 0x00-0xFF (Payload Length) |
| SEQ     | 0x01-0xFF (Sequence number, 0x00 represents the first message) |
| SID     | 0x00-0xFF (System Identification number of the UAV) |
| CID     | 0x00-0xFF (System Identification number of the component transmitting the message) |
| MID     | 0x00-0xFF (Message Type Identification number, set to 0xDE for ARID) |
| Payload | ARID message (see below for details) |
| CRC     | Checksum for integrity check |
| RES     | Reserved Bytes (e.g., Network, System, and Component ID) |
| PID     | UAV Pseudonym \( c_{n,t} \) (size based on selected curve) |
| EKEY    | Encrypted one-time key \( \rho_{n,t} \) (size based on selected curve) |
| ULAT, ULON, UALT | UAV Latitude, Longitude, and Altitude (4 bytes each) |
| UVX, UVY, UVZ | UAV Speed x, y, and z axis (2 bytes each) |
| CLAT, CLON, CALT | Latitude, Longitude, and Altitude of the Ground Station (4 bytes each) |
| TS      | Message Timestamp (4 bytes) |
| ES      | UAV Emergency Status (1 byte) |
