### 代码注入检测

**描述：**
监控 Windows API 调用（这些调用表明各种类型的代码注入）可能会生成大量数据，并且除非在特定情况下针对已知的恶意调用序列进行收集，否则可能对防御没有直接帮助，因为良性使用的 API 函数可能是常见的并且难以与恶意行为区分开来。Windows API 调用如 SuspendThread/SetThreadContext/ResumeThread、QueueUserAPC/NtQueueApcThread 以及那些可用于修改另一个进程内存的 API 调用，如 VirtualAllocEx/WriteProcessMemory，都可能用于此技术。(引用: Elastic Process Injection July 2017)
