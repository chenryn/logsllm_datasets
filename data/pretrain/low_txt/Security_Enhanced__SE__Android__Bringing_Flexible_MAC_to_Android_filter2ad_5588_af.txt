#### A. 示例 SELinux 策略
本附录展示了从 SE Android 策略中提取的示例策略源文件内容。这些内容取自 Android 源码树中的 `external/sepolicy/bluetoothd.te` 文件。该文件定义了 Android 蓝牙守护进程的域。策略使用宏（如初始化守护进程域宏）和策略语言语句（如允许和类型转换规则）编写。

```plaintext
type bluetoothd, domain;
type bluetoothd_exec, exec_type, file_type;

init_daemon_domain(bluetoothd)
allow bluetoothd self:capability { setuid net_raw net_bind_service net_admin };
allow bluetoothd self:socket *;
allow bluetoothd bluetoothd_data_file:dir create_dir_perms;
allow bluetoothd bluetoothd_data_file:file create_file_perms;
unix_socket_connect(bluetoothd, dbus, dbusd)
```

#### B. 示例 seapp 上下文
本附录展示了用于确定如何为由 zygote 创建的应用程序进程以及由 installd 创建的应用程序数据目录分配安全上下文的 `external/sepolicy/seapp contexts` 配置片段。每行指定一组输入选择器（如 isSystemServer 布尔值、用户名、seinfo 字符串和包名）以及一组输出值（如域名、类型名、levelFromUid 布尔值和级别字符串）。应用了一组优先级规则来确定使用哪个条目，更具体的条目优先。

```plaintext
isSystemServer=true domain=system
user=system domain=system_app type=system_data_file
user=bluetooth domain=bluetooth type=bluetooth_data_file
user=nfc domain=nfc type=nfc_data_file
user=radio domain=radio type=radio_data_file
user=_app domain=untrusted_app type=app_data_file levelFromUid=true
user=_app seinfo=platform domain=platform_app type=platform_app_data_file
user=_app seinfo=release domain=release_app type=platform_app_data_file
user=_app seinfo=release name=com.android.browser domain=browser_app type=platform_app_data_file
```

#### D. 中间件 MAC 策略
本附录展示了用于安装时 MAC 机制并为应用程序分配 seinfo 标签的 `external/sepolicy/mac permissions.xml` 配置片段。签名值和权限名称已简化以提高可读性。签名是十六进制编码的 X.509 证书。

可以通过 `signer` 子句为具有给定签名的所有包指定 seinfo 字符串和 Android 权限，或者通过 `package` 子句为特定包细化这些权限。允许的 Android 权限可以通过白名单（`allow-permission`）或黑名单（`deny-permission`）指定，但不能同时使用两者。通过指定 `allow-all`，可以允许具有给定签名或包请求的所有权限而不受任何限制。默认标签用于任何不匹配其他子句的应用程序。可以使用 `setool` 程序从一组 Android 包生成策略子句。

```xml
<permissions>
    <signer signature="...">
        <seinfo>...</seinfo>
        <allow-permission>...</allow-permission>
        <!-- 或 -->
        <deny-permission>...</deny-permission>
    </signer>
    <package name="..." signature="...">
        <seinfo>...</seinfo>
        <allow-permission>...</allow-permission>
        <!-- 或 -->
        <deny-permission>...</deny-permission>
    </package>
    <default>
        <seinfo>...</seinfo>
        <allow-all/>
    </default>
</permissions>
```

希望这些优化能帮助你更好地理解和使用这些文献和配置。
